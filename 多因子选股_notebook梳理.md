# 多因子选股 Notebook 梳理

## 1. 范围说明

本文件只梳理目录 `聚宽有价值策略558` 中与“多因子选股”直接相关的 notebook。

我把相关笔记本分成四层：

- 传统多因子选股
- 机器学习驱动的多因子选股
- 因子研究与因子构造
- 选股辅助工具与数据/模型实验

总计梳理了 `24` 个相关 notebook。

## 2. 总体结论

这批 notebook 的主线非常清晰：

1. 先用财务、估值、动量、质量、成长、换手率等因子做基础选股
2. 再用机器学习方法做因子筛选、特征重要性评估和模型融合
3. 最后把“因子研究”延伸到市场微观结构、行业轮动、ETF、基金、情绪和趋势识别

它们不是互相孤立的单篇研究，而是围绕“如何把多因子模型做得更稳、更可交易、更接近实盘”的连续探索。

## 11. 未来可以做的方法和改进

基于这批 notebook 现有的研究路径，后续最值得推进的方向不是“再找一个新因子”，而是把因子研究、模型训练、回测验证和实盘适配做成一条更稳的流水线。

### 11.1 因子层面的改进

1. 从“单因子有效”升级到“因子簇有效”
   - 不再只看某一个因子本身，而是把估值、质量、成长、动量、情绪、流动性拆成多个子簇。
   - 重点观察同一类因子之间是否冗余，是否存在真正互补。

2. 做因子分市场状态研究
   - 同一个因子在牛市、震荡市、熊市可能表现完全不同。
   - 后续可以把市场宽度、波动率、情绪、拥挤度作为状态变量，研究“什么环境下什么因子更有效”。

3. 引入更细粒度的量价因子
   - 现有 notebook 已经涉及成交量激增、动量、日内收益、换手率。
   - 未来可以进一步扩展到盘口、分时结构、波动聚集、资金流强度等更细层面的特征。

4. 增强因子的稳定性筛选
   - 不只看整体 IC，还要看分年度、分行业、分市值分层后的稳定性。
   - 防止单个因子只在某一小段样本里有效。

### 11.2 模型层面的改进

1. 从静态打分升级为动态权重
   - 传统多因子多是固定权重或固定排序。
   - 可以改成随市场环境变化的动态权重，比如根据波动率、趋势强弱、拥挤度自动调整因子权重。

2. 把机器学习从“分类器”升级为“排序器”
   - 多因子选股的本质是排序，不一定非要预测涨跌二分类。
   - 未来可以尝试学习排序模型、pairwise ranking 或分位数预测。

3. 做模型集成而不是单模型依赖
   - 目前 notebook 里已经出现随机森林、SVM、遗传算法、K-means 等。
   - 后续可以把不同模型输出做 stacking、blending 或投票集成，降低单模型偏差。

4. 加强特征选择和降维
   - 因子数量一多就容易共线、过拟合、样本外失效。
   - 可以用 SHAP、Permutation Importance、L1 正则、稳定性选择等方法做因子筛选。

### 11.3 回测层面的改进

1. 做更严格的样本外验证
   - 不只是简单训练集/测试集切分。
   - 更推荐滚动窗口回测、walk-forward 验证和跨年份验证。

2. 加入真实交易约束
   - 包括滑点、涨跌停、停牌、成交额限制、换手限制、持仓集中度限制。
   - 对小市值或高换手策略尤其重要。

3. 增加交易成本敏感性分析
   - 因子策略往往在低成本条件下效果很好，但一加成本会明显缩水。
   - 应该把手续费、滑点和冲击成本当成核心变量来做压力测试。

4. 分拆收益来源
   - 区分 alpha、行业 beta、风格 beta、市场 beta。
   - 这样才能知道策略究竟是“真正选股有效”，还是只是踩中了某种风格。

### 11.4 风控层面的改进

1. 增加市场状态风控
   - 当市场宽度恶化、波动率放大、情绪极端时，主动降仓或切换到更稳健的子策略。

2. 增加个股风险过滤
   - ST、退市、异常波动、流动性不足、财务恶化等都应该前置过滤。

3. 增加组合层面的约束
   - 单行业暴露上限
   - 单因子暴露上限
   - 单标的权重上限
   - 回撤阈值触发减仓

4. 做“策略失效预警”
   - 当最近 N 期 IC 下降、胜率下降、回撤升高时，自动提示策略可能失效。

### 11.5 工程化层面的改进

1. 建立统一的研究框架
   - 把数据获取、因子计算、回测、评估、报告输出拆成标准模块。
   - 这样每个新策略都能快速复用。

2. 建立因子字典和策略字典
   - 因子字段、含义、方向、计算方式、适用市场都应该标准化记录。
   - 后续做研究时，能快速知道哪些因子已经试过、哪些还有空白。

3. 建立实验记录体系
   - 每次改动记录：改了什么、为什么改、结果如何、是否样本外有效。
   - 这会大幅减少“重复试错但无法复盘”的问题。

4. 把 notebook 变成可运行的策略模块
   - Notebook 适合研究，但不适合长期维护。
   - 建议把成熟策略抽成脚本、配置和测试用例，减少手工操作。

### 11.6 如果要优先落地，建议的三条主线

如果只挑最值得做的三条，我建议优先是：

1. 因子状态切换
   - 研究不同市场环境下因子的有效性差异，并据此动态调权。

2. 严格样本外和交易约束回测
   - 让策略从“研究有效”真正变成“实盘可能有效”。

3. 因子库标准化
   - 先把所有因子、策略、实验结果整理成统一文档，后面迭代效率会高很多。

### 11.7 一句话总结

未来提升空间最大的，不是单纯再加几个因子，而是把“因子发现”升级为“因子治理”，把“模型拟合”升级为“环境自适应”，把“回测漂亮”升级为“实盘可控”。
