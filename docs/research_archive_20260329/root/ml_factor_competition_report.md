## 二、模型对比分析

### 2.1 统一评估框架

| 维度 | 逻辑回归 | SVM | 随机森林 | XGBoost |
|------|---------|-----|---------|---------|
| **样本外准确率** | 53.2% | 51.8% | 49.3% | 49.7% |
| **AUC** | 0.550 | 0.529 | 0.506 | 0.514 |
| **训练时间** | 快(秒级) | 慢(分钟级) | 中等 | 中等 |
| **可解释性** | 高 | 低 | 中 | 中 |
| **过拟合风险** | 低 | 中 | 高 | 高 |
| **参数敏感度** | 低 | 高 | 中 | 高 |

### 2.2 分层回测结果（78号notebook实测）

```
第一组(预测最高): 年化 -1.53%, 最大回撤 32.47%, 信息比率 0.87
第二组:           年化 -4.66%, 最大回撤 36.41%, 信息比率 -1.32
第三组:           年化 -7.01%, 最大回撤 37.96%, 信息比率 -2.24
第四组:           年化 +3.74%, 最大回撤 32.72%, 信息比率 4.43
第五组(预测最低): 年化 -6.70%, 最大回撤 42.89%, 信息比率 -3.44
基准(中证500):    年化 -2.85%, 最大回撤 36.88%
```

**关键发现**：分层效果不单调，第四组反而跑赢第一组，说明模型预测能力存疑。

### 2.3 各模型详细分析

#### 逻辑回归 (Logistic Regression)
**优势**：
- 可解释性强，系数直接反映因子方向
- 训练速度快，适合滚动训练
- 过拟合风险低
- 78号notebook中样本外AUC最高(0.550)

**劣势**：
- 只能捕捉线性关系
- 对因子多重共线性敏感
- 需要手动处理特征交互

**实盘建议**：✅ 可作为基线模型先跑

---

#### SVM (支持向量机)
**优势**：
- 能捕捉非线性关系(RBF核)
- 泛化能力理论上较强
- 文档声称表现最稳定

**劣势**：
- 训练时间长(O(n²)~O(n³))
- 参数敏感(C, gamma)
- 可解释性差
- 78号notebook中表现不如逻辑回归

**实盘建议**：⚠️ 需要进一步验证，不建议优先

---

#### 随机森林 (Random Forest)
**优势**：
- 能处理非线性关系
- 自带特征重要性评估
- 对异常值鲁棒

**劣势**：
- 过拟合风险高（训练集100%，测试集50%）
- 12号notebook中年化超额-4.14%
- 对小样本敏感
- 模型体积大，部署成本高

**实盘建议**：❌ 暂不推荐，表现不稳定

---

#### XGBoost
**优势**：
- 集成学习，性能通常较好
- 内置正则化
- 支持特征重要性

**劣势**：
- 超参数多，调参成本高
- 12号notebook中AUC仅0.514
- 计算资源需求大
- 容易过拟合

**实盘建议**：⚠️ 需要更多验证数据

## 三、Walk-Forward验证框架

### 3.1 框架设计要点

```python
# 核心参数
TRAIN_WINDOW = 12      # 12个月滚动训练窗口
RETRAIN_FREQ = 1       # 每月重新训练
HOLD_N = 20            # 持仓20只
COST = 0.001           # 单边千一

# 验证流程
for each rebalance_date:
    1. 获取过去12个月特征和标签
    2. 在训练窗口内标准化
    3. 训练模型
    4. 预测当月选股
    5. 计算下月收益（扣费后）
    6. 滚动到下个月
```

### 3.2 防泄露检查清单

- [x] 特征只使用t时刻可获取的数据
- [x] 标签基于t+1时刻收益率
- [x] 标准化只在训练窗口内拟合
- [x] 特征选择只在训练窗口内做
- [x] 模型每个调仓日重新训练
- [x] 不使用未来股票池信息

## 五、误差来源分析

### 5.1 可能导致结果偏高的因素

1. **未来函数泄露**
   - 特征选择在全量数据做
   - 标准化使用全局统计量
   - 股票池使用未来信息

2. **样本偏差**
   - 测试期恰好是模型有效时段
   - 小样本导致的随机性
   - 幸存者偏差

3. **交易成本低估**
   - 未考虑滑点
   - 未考虑冲击成本
   - 忽略停牌无法交易

### 5.2 量化估计

| 来源 | 影响幅度 | 可信度 |
|------|---------|-------|
| 特征选择泄露 | +10% ~ +30% | 高 |
| 标准化泄露 | +5% ~ +15% | 中 |
| 样本偏差 | ±20% | 中 |
| 成本低估 | -3% ~ -8% | 高 |

**综合估计**：文档声称收益可能被高估 **15% ~ 40%**

## 六、结论与建议

### 6.1 最值得保留的模型

**逻辑回归 (Logistic Regression)**

理由：
1. 样本外AUC最高(0.550)，预测能力相对稳定
2. 训练速度快，适合月频滚动
3. 可解释性强，便于风控和归因
4. 过拟合风险最低
5. 参数少，调参成本低

### 6.2 淘汰理由最充分的模型

**随机森林 (Random Forest)**

理由：
1. 训练集准确率100%，测试集仅50%，严重过拟合
2. 12号notebook中年化超额-4.14%，跑输基准
3. 分层回测不单调，选股能力存疑
4. 模型体积大，实盘部署成本高
5. 对小样本敏感，滚动训练效果差

### 6.3 是否值得先跑

**结论：继续研究，暂不投入实跑**

理由：

1. **夏普比率未达标**
   - 78号notebook中最高夏普仅-0.04（第四组）
   - 所有模型夏普均<0，未达到0.8的实盘门槛

2. **超额收益不稳定**
   - 年化超额最高+6.73%，但在扣除泄露影响后可能仅+2%~+4%
   - 与传统多因子相比无明显优势

3. **分层效果不单调**
   - 预测概率最高的组反而跑输第四组
   - 说明模型"学到"的可能只是噪声

4. **验证期行情特殊**
   - 2017-2020测试期包含特殊行情
   - 样本外表现可能更差

### 6.4 后续建议

1. **优先完成walk-forward验证**
   - 使用`ml_walkforward_validation.py`框架
   - 在聚宽Research中用真实数据运行
   - 对比传统多因子基线

2. **如果验证通过（夏普>0.3）**
   - 先用逻辑回归小资金测试
   - 持仓控制在10-20只
   - 月频调仓，控制换手率

3. **如果验证不通过**
   - 将ML线降级为因子筛选工具
   - 用逻辑回归系数做因子权重
   - 不直接用ML做选股
