## 二、失败原因分析

### 2.1 核心问题诊断

**问题1：状态过滤过于激进，交易机会锐减**

| 指标 | v1 | v2 | 变化 |
|------|-----|-----|------|
| **交易次数** | 184次 | 31次 | **-83%** |
| **平均持仓天数** | 108天 | 63.8天 | -41% |
| **平均每月交易** | 5.1次 | 0.9次 | **-82%** |

**原因**：
- 涨停数<30停手规则导致错过大量机会
- 宽度分档+情绪过滤双重限制过度
- 策略过于保守，几乎不交易

**问题2：仓位动态调整未能降低回撤**

| 指标 | v1无动态 | v2有动态 | 预期效果 | 实际效果 |
|------|---------|---------|---------|---------|
| **最大回撤** | 37.73% | 41.28% | -40% | **+9.4%** |
| **平均仓位** | 100% | ~30%（估算） | - | 仓位过低 |

**原因**：
- 宽度阈值过于保守，大部分时间低仓位或停手
- 错过上涨机会，但下跌时仍未幸免
- 仓位调整未能有效择时

**问题3：候选池反而更稀疏**

| 策略 | 市值区间 | 筛选条件 | 交易次数 | 候选池状态 |
|------|---------|---------|---------|-----------|
| v1 | 15-60亿 | PB<1.5+PE<20 | 184次 | ✅ 充足 |
| v2 | 10-100亿 | ROE>6%+PB<2.5+PE<30 | 31次 | ❌ **稀疏** |

**原因**：
- 虽然扩大了市值区间和放宽了PB/PE
- 但增加了ROE>6%门槛
- 状态过滤+仓位动态双重限制导致实际交易机会极少

### 2.2 各优化项实际效果

| 优化项 | 理论预期 | 实际效果 | 结论 |
|--------|---------|---------|------|
| **市值区间扩大** | 候选池翻倍 | 未体现 | ❌ 被其他条件抵消 |
| **筛选条件放宽** | 候选池增加50% | 交易减少83% | ❌ **适得其反** |
| **持仓降至8只** | 回撤降低39% | 回撤增加9.4% | ❌ **无效** |
| **状态过滤** | 回撤降低25% | 回撤增加 | ❌ **过度激进** |
| **仓位动态** | 回撤降低40% | 回撤增加 | ❌ **无效** |

**根本原因**：过度优化，多个保守策略叠加导致策略僵化

## 四、失败教训总结

### 4.1 过度优化陷阱

**问题**：同时修改5个参数，无法确定哪个有效

**表现**：
1. 市值区间扩大 → 理论增加候选，实际未体现
2. 筛选条件放宽 → 理论增加候选，实际交易减少
3. 持仓降至8只 → 理论降低回撤，实际回撤更大
4. 状态过滤 → 理论降低回撤，实际错过机会
5. 仓位动态 → 理论降低回撤，实际无效

**教训**：应该单一变量测试，逐步优化

### 4.2 理论vs实践偏差

**理论推断**：
- 放宽条件 → 候选池增加
- 状态过滤 → 回撤降低
- 仓位动态 → 风险控制

**实际结果**：
- 放宽条件 + 其他限制 → 交易锐减
- 状态过滤 → 错过机会，回撤更大
- 仓位动态 → 策略僵化

**教训**：理论推断必须实测验证，不能盲目相信

### 4.3 防守过度问题

**问题**：为了降低回撤，过度防守

**表现**：
- 涨停<30停手 → 错过大量机会
- 宽度分档保守 → 大部分时间低仓位
- 双重过滤 → 几乎不交易

**结果**：
- 回撤未降低（41.28% vs 37.73%）
- 收益大幅下降（-0.60% vs 14.19%）
- 策略失效

**教训**：防守不能过度，需要平衡进攻和防守

## 七、v2失败的核心结论

### 7.1 失败原因

**根本原因**：过度优化，多个保守策略叠加

**具体表现**：
1. ❌ 状态过滤过于激进 → 交易锐减83%
2. ❌ 仓位动态过于保守 → 策略僵化
3. ❌ 候选池反而稀疏 → 31次交易 vs 184次
4. ❌ 回撤未降低 → 41.28% vs 37.73%
5. ❌ 收益大幅下降 → -0.60% vs 14.19%

### 7.2 教训总结

1. ✅ **必须单一变量测试**
   - 不能同时修改多个参数
   - 无法确定哪个优化有效

2. ✅ **理论推断必须实测验证**
   - 不能盲目相信理论
   - 实际结果可能相反

3. ✅ **防守不能过度**
   - 需要平衡进攻和防守
   - 过度防守导致策略失效

4. ✅ **保持策略简洁**
   - 复杂策略容易失败
   - 简单策略更稳健

### 7.3 是否继续v3设计

**判定：⚠️ 暂停，先验证单一优化**

**理由**：
1. v2失败证明理论推断不可靠
2. 需要先验证单一优化是否有效
3. 如果单项均无效，可能需要放弃小市值防守线
