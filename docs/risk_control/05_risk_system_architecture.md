## 二、模块功能说明

### 2.1 数据层模块

**市场数据模块**
- 功能：获取实时和历史市场数据
- 数据源：JoinQuant平台
- 数据内容：
  - 行情数据：涨跌幅、开盘价、收盘价、最高价、最低价
  - 涨跌停数据：涨停家数、跌停家数、涨停股列表、跌停股列表
  - 成交额数据：日均成交额、换手率
  - 板块数据：板块涨跌幅、板块热度

**策略数据模块**
- 功能：管理策略的交易和持仓数据
- 数据源：内部交易系统
- 数据内容：
  - 交易记录：每笔交易的买入价、卖出价、收益、时间
  - 策略净值：每日策略净值、累计收益、年化收益
  - 持仓信息：当前持仓股票、持仓仓位、持仓成本
  - 信号记录：买入信号、卖出信号、信号时间

**外部数据模块**
- 功能：获取大盘和外部市场数据
- 数据源：JoinQuant平台、外部API
- 数据内容：
  - 大盘指数：沪深300、中证1000、上证指数
  - VIX指数：期权隐含波动率
  - 资金流向：主力资金流向

### 2.2 计算层模块

**风控指标计算模块**
- 功能：计算各类风控指标
- 计算内容：
  - 市场风险指标：涨停家数、跌停家数、大盘涨跌幅、板块涨跌幅
  - 流动性风险指标：日均成交额、换手率、涨跌停比
  - 策略风险指标：最大回撤、胜率、盈亏比、夏普比率
  - 尾部风险指标：连续亏损次数、单日亏损、单笔最大亏损
- 输出：指标计算结果（数值）

**风控规则判断模块**
- 功能：判断风控规则是否触发
- 判断内容：
  - 买入前检查：情绪过滤、大盘过滤、流动性过滤、市值过滤、位置过滤
  - 持仓监控：个股止损、组合回撤、仓位限制
  - 熔断判断：连续亏损熔断、组合回撤熔断、单日亏损熔断
- 输出：规则判断结果（触发/未触发）

**风控决策生成模块**
- 功能：生成风控决策
- 决策内容：
  - 允许交易：所有检查通过，允许开仓
  - 限制交易：部分检查未通过，限制仓位或暂停开仓
  - 禁止交易：熔断触发，禁止所有交易
- 输出：风控决策（允许/限制/禁止 + 具体措施）

### 2.3 决策层模块

**风控决策引擎**
- 功能：综合判断并生成最终决策
- 输入：指标计算结果 + 规则判断结果
- 处理：
  - 综合判断：根据所有规则判断结果，综合评估风险等级
  - 决策生成：根据风险等级，生成风控决策
  - 决策优化：根据历史经验，优化决策策略
- 输出：风控决策（允许/限制/禁止 + 具体措施）
- 记录：决策日志（时间、指标、规则、决策、执行）

**风控规则库**
- 功能：存储和管理所有风控规则
- 规则内容：
  - 市场风险规则：情绪过滤（涨停家数50-150）、大盘过滤（涨跌幅>=-3%）
  - 流动性风险规则：成交额过滤（>=5000万）、换手率过滤（>=2%）
  - 策略风险规则：回撤控制（<15%）、胜率控制（>=45%）
  - 熔断规则：连续亏损熔断（>=5笔）、组合回撤熔断（>15%）、单日亏损熔断（>5%）
- 规则管理：规则增删改查、规则优先级、规则参数调整

### 2.4 执行层模块

**风控执行模块**
- 功能：执行风控决策
- 执行内容：
  - 与交易系统对接：将风控决策传递给交易系统
  - 自动执行：自动执行风控决策（禁止开仓、降低仓位、触发止损）
  - 执行结果反馈：执行结果反馈给决策引擎
  - 执行日志记录：记录执行过程和结果

**风控监控面板**
- 功能：展示风控状态和提供人工干预接口
- 展示内容：
  - 实时风险指标：涨停家数、大盘涨跌幅、回撤、胜率
  - 预警信息：预警级别、预警触发时间、预警指标
  - 熔断状态：熔断是否触发、熔断类型、熔断时长
  - 历史风控记录：历史决策、执行记录
- 人工干预：
  - 人工确认：重要决策需人工确认
  - 人工调整：调整规则参数
  - 人工熔断：人工触发熔断

### 2.5 交易系统模块

**信号生成模块**
- 功能：生成买入和卖出信号
- 信号内容：
  - 买入信号：基于策略规则生成买入信号
  - 卖出信号：基于策略规则生成卖出信号
- 与风控集成：信号生成前，需通过风控检查

**仓位管理模块**
- 功能：计算和管理仓位
- 仓位内容：
  - 仓位计算：根据信号和风控规则计算仓位
  - 仓位调整：根据风控决策调整仓位
- 与风控集成：仓位计算需考虑风控限制

**交易执行模块**
- 功能：执行交易
- 执行内容：
  - 订单生成：根据信号和仓位生成订单
  - 订单执行：执行订单（买入、卖出）
  - 执行结果反馈：执行结果反馈给风控系统
- 与风控集成：交易执行需遵循风控决策

## 三、模块间数据流

### 3.1 买入信号流程

```
【买入信号流程】

1. 策略生成买入信号
   └── 信号生成模块 → 买入信号候选

2. 风控买入前检查
   ├── 市场数据模块 → 涨停家数、大盘涨跌幅、成交额
   ├── 策略数据模块 → 持仓信息、仓位限制
   ├── 指标计算模块 → 计算风控指标
   ├── 规则判断模块 → 判断买入前规则
   └── 决策生成模块 → 生成风控决策（允许/禁止）

3. 风控决策反馈
   ├── 决策引擎 → 综合判断 + 最终决策
   ├── 执行模块 → 执行风控决策
   └── 监控面板 → 展示风控状态

4. 交易执行
   ├── 允许交易 → 仓位管理 → 订单生成 → 执行交易
   ├── 限制交易 → 降低仓位 → 订单生成 → 执行交易
   └── 禁止交易 → 不生成订单 → 不执行交易

5. 执行结果反馈
   ├── 交易执行模块 → 执行结果（成交价、成交量）
   ├── 策略数据模块 → 更新持仓信息
   └── 监控面板 → 更新风控状态
```

### 3.2 持仓监控流程

```
【持仓监控流程】

1. 实时监控持仓
   ├── 市场数据模块 → 持仓股票涨跌幅、大盘涨跌幅
   ├── 策略数据模块 → 持仓信息、策略净值
   ├── 指标计算模块 → 计算风控指标（回撤、连续亏损）
   └── 规则判断模块 → 判断持仓规则（止损、熔断）

2. 风控决策生成
   ├── 正常持仓 → 无触发规则 → 允许继续持有
   ├── 止损触发 → 单票亏损>5% → 生成卖出信号
   └── 熔断触发 → 连亏>=5笔、回撤>15% → 生成清仓信号

3. 风控决策执行
   ├── 执行模块 → 执行卖出/清仓
   ├── 交易系统 → 订单生成 → 执行交易
   └── 监控面板 → 展示风控状态

4. 执行结果反馈
   ├── 交易执行模块 → 执行结果
   ├── 策略数据模块 → 更新持仓信息、策略净值
   └── 监控面板 → 更新风控状态
```

### 3.3 熔断流程

```
【熔断流程】

1. 熔断触发
   ├── 指标计算模块 → 连续亏损>=5笔、回撤>15%、单日亏损>5%
   ├── 规则判断模块 → 判断熔断规则
   └── 决策生成模块 → 生成熔断决策

2. 熔断决策执行
   ├── 执行模块 → 执行熔断（暂停交易、清仓）
   ├── 交易系统 → 清仓订单 → 执行清仓
   └── 监控面板 → 展示熔断状态

3. 熔断状态监控
   ├── 监控面板 → 熔断类型、熔断时长、恢复条件
   └── 人工干预 → 人工确认熔断

4. 熔断恢复
   ├── 指标计算模块 → 监控恢复条件
   ├── 规则判断模块 → 判断恢复条件是否满足
   └── 决策生成模块 → 生成恢复决策
   └── 执行模块 → 执行恢复（恢复交易）
```

## 四、与交易系统集成方式

### 4.1 集成架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    风控系统与交易系统集成                          │
└─────────────────────────────────────────────────────────────────┘

【交易系统】
├── 信号生成模块
│   ├── 输入：策略规则、市场数据
│   ├── 输出：买入信号候选、卖出信号候选
│   └── 集成点：信号生成前 → 风控买入前检查
│
├── 仓位管理模块
│   ├── 输入：买入信号、持仓信息
│   ├── 输出：仓位计算结果
│   └── 集成点：仓位计算 → 风控仓位限制检查
│
└── 交易执行模块
    ├── 输入：买入信号、仓位、风控决策
    ├── 输出：订单、执行结果
    └── 集成点：订单生成前 → 风控决策检查
               执行后 → 执行结果反馈

【风控系统】
├── 风控决策引擎
│   ├── 输入：指标计算结果、规则判断结果
│   ├── 输出：风控决策
│   └── 集成点：决策生成后 → 传递给交易系统
│
└── 风控执行模块
    ├── 输入：风控决策
    ├── 输出：执行结果
    └── 集成点：执行风控决策 → 通知交易系统
               执行结果反馈 → 更新风控状态
```

### 4.2 集成接口设计

```python
    """
    买入前风控检查接口

        market_data: 市场数据（涨停家数、大盘涨跌幅、成交额）
        strategy_data: 策略数据（持仓信息、仓位限制）

        decision: 风控决策（允许/限制/禁止 + 具体措施）
    """
    # 1. 计算风控指标
    indicators = calculate_risk_indicators(market_data, strategy_data)
    
    # 2. 判断买入前规则
    rules = check_buy_rules(indicators)
    
    # 3. 生成风控决策
    decision = generate_risk_decision(rules)
    
    return decision
```

**接口2：持仓监控接口**
```python
def risk_monitor_holding(holding_info, market_data, strategy_data):
    """
    持仓监控接口
    输入：
        holding_info: 持仓信息（持仓股票、持仓仓位、持仓成本）
        market_data: 市场数据（持仓股票涨跌幅）

        action: 风控动作（继续持有/止损/清仓）
    """

    # 2. 判断持仓规则

    # 3. 生成风控动作
```

**接口3：熔断检查接口**
```python
def risk_check_circuit_breaker(strategy_data):
    """
    熔断检查接口
    输入：
        strategy_data: 策略数据（连续亏损、回撤、单日亏损）

    # 3. 生成熔断决策
    circuit_breaker = generate_circuit_breaker_decision(rules)

```

### 4.3 集成流程

**买入流程集成：**
1. 策略生成买入信号候选
2. 调用风控买入前检查接口
3. 风控返回决策（允许/限制/禁止）
4. 如果允许，调用仓位管理模块
5. 仓位管理调用风控仓位限制检查
6. 生成订单，调用交易执行模块
7. 执行交易，反馈执行结果给风控

**持仓流程集成：**
1. 风控实时监控持仓
2. 调用持仓监控接口
3. 风控返回动作（继续持有/止损/清仓）
4. 如果止损/清仓，调用信号生成模块
5. 生成卖出信号，调用交易执行模块
6. 执行卖出，反馈执行结果给风控

**熔断流程集成：**
1. 风控实时监控策略状态
2. 调用熔断检查接口
3. 风控返回熔断决策（触发/未触发）
4. 如果触发，调用交易执行模块清仓
5. 执行清仓，反馈执行结果给风控
6. 熔断期间禁止所有交易

## 五、故障处理机制

### 5.1 数据故障处理

**故障类型：**
- 数据源故障：无法获取市场数据
- 数据错误：数据异常（涨停数=9999）
- 数据延迟：数据更新延迟

**处理措施：**
1. 检测故障：监控数据源状态、数据异常值
2. 降级处理：使用历史数据替代
3. 风控应对：禁止交易，避免错误决策
4. 故障恢复：数据源恢复后，重新启动风控

### 5.2 系统故障处理

**故障类型：**
- 计算故障：指标计算错误
- 决策故障：决策生成错误
- 执行故障：风控执行失败

**处理措施：**
1. 检测故障：监控系统状态、执行结果
2. 降级处理：使用简化风控规则
3. 人工干预：人工确认风控决策
4. 故障恢复：系统恢复后，重新启动风控

### 5.3 交易系统故障处理

**故障类型：**
- 交易系统故障：无法生成订单或执行交易
- 网络故障：网络中断
- API故障：API调用失败

**处理措施：**
1. 检测故障：监控交易系统状态
2. 人工干预：人工交易或暂停交易
3. 备用方案：使用备用交易系统
4. 故障恢复：交易系统恢复后，重新启动

### 5.4 极端行情故障处理

**故障类型：**
- 系统熔断：大盘触发熔断机制
- 流动性枯竭：大面积跌停无法卖出
- 数据异常：涨停数异常（过高或过低）

**处理措施：**
1. 检测故障：监控大盘状态、涨停数
2. 立即熔断：禁止所有交易
3. 持仓处理：持仓股票如有风险，提前卖出
4. 故障恢复：极端行情结束后，逐步恢复

## 八、总结

### 8.1 系统架构特点

**特点1：分层架构**
- 数据层：数据获取和存储
- 计算层：指标计算和规则判断
- 决策层：决策生成和规则管理
- 执行层：决策执行和状态监控

**特点2：模块化设计**
- 各模块独立，易于维护和升级
- 模块间通过接口集成，易于扩展
- 模块可替换，易于适应不同交易系统

**特点3：实时监控**
- 实时监控市场风险
- 实时监控策略风险
- 实时监控持仓风险
- 及时发现和应对风险

**特点4：自动化执行**
- 自动执行风控决策
- 自动触发熔断机制
- 自动反馈执行结果
- 减少人工干预

### 8.2 集成方式特点

**特点1：接口化集成**
- 通过接口与交易系统集成
- 接口标准化，易于对接不同系统
- 接口可扩展，易于增加新功能

**特点2：流程化集成**
- 买入流程、持仓流程、熔断流程清晰
- 流程标准化，易于理解和执行
- 流程可优化，易于提高效率

**特点3：反馈化集成**
- 执行结果反馈给风控系统
- 风控状态反馈给监控面板
- 形成闭环，易于监控和优化

### 8.3 故障处理特点

**特点1：分级处理**
- 数据故障、系统故障、交易故障、极端行情故障分级
- 不同故障类型采用不同处理措施
- 确保故障不会扩大

**特点2：降级处理**
- 故障时使用降级方案
- 确保系统继续运行
- 故障恢复后重新启动

**特点3：人工干预**
- 重要决策需人工确认
- 故障时人工干预
- 确保风控决策正确
