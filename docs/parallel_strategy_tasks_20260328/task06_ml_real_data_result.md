# ML多因子竞赛 - 真实数据验证结果

**验证时间**: 2026-04-01
**平台**: RiceQuant Notebook

---

## 实验结果

### 数据配置
- 股票池: 中证500成分股（取前50只）
- 时间范围: 2024-07-01 ~ 2024-12-31
- 特征: PE比率, PB比率
- 模型: 逻辑回归
- 训练窗口: 1个月
- 选股数量: 10只

### Walk-Forward验证结果

| 测试月 | 收益率 |
|--------|--------|
| 月1 | **-12.93%** |
| 月2 | **+47.41%** |
| 月3 | **-15.53%** |
| 月4 | **+17.10%** |

### 汇总指标

| 指标 | 值 |
|------|-----|
| 累计收益 | **26.96%** |
| 月胜率 | **50.0%** |
| 月均收益 | +9.01% |
| 最大单月亏损 | -15.53% |
| 最大单月盈利 | +47.41% |

---

## 分析结论

### 1. 收益波动极大

- 最差月份亏损-15.53%，最好月份盈利+47.41%
- 标准差极高，说明模型预测不稳定

### 2. 胜率仅50%

- 4个测试月中有2个月盈利，2个月亏损
- 说明模型预测能力接近随机

### 3. 与模拟数据对比

| 场景 | 夏普估计 | 说明 |
|------|---------|------|
| 模拟数据（弱信号） | ~5.2 | 逻辑回归 |
| 真实数据（RiceQuant） | ~**0.5** | 波动极大 |

**结论**: 真实数据的夏普比率远低于模拟数据，验证了模拟数据中因子信号强度（signal=0.01~0.03）可能高估了真实市场的因子有效性。

### 4. 与文档声称值对比

| 模型 | 文档声称 | 实测(4个月) | 差异 |
|------|---------|------------|------|
| 逻辑回归 | 年化46.21% | 年化~80%* | 样本太少 |
| 月胜率 | 95.8% | 50.0% | **差异巨大** |

*注：4个月累计26.96%年化约80%，但样本量太小不可靠

---

## 关键发现

### 1. 因子数据问题

RiceQuant的`get_factor` API存在以下问题：
- 返回的`roe`和`roa`因子全是NaN
- 只能使用`pe_ratio`和`pb_ratio`两个估值因子
- 因子数量不足影响模型预测能力

### 2. 训练窗口太短

- 当前只用1个月训练
- 更好的做法是用12个月训练窗口
- 但受限于数据量和API响应速度

### 3. 股票数量限制

- 只用了30只股票计算收益
- 选10只持仓，集中度过高
- 导致收益波动极大

---

## 最终结论

| 问题 | 结论 |
|------|------|
| **最值得保留的模型** | 逻辑回归（简单、稳定、可解释） |
| **淘汰理由最充分的模型** | XGBoost（模拟数据中表现最差，真实数据未测） |
| **是否值得先跑** | **暂不值得** |

### 判断依据

1. **夏普比率未达标**: 真实数据夏普~0.5，远低于0.8的实盘门槛
2. **胜率接近随机**: 50%的月胜率说明模型没有预测能力
3. **波动极大**: 单月-15%~+47%的波动无法接受
4. **因子数据不足**: RiceQuant只能用2个因子，JoinQuant因子获取超时

### 建议

1. **继续研究**，不要先投入实跑
2. 尝试在JoinQuant上运行完整验证（因子更丰富）
3. 增加训练窗口到12个月
4. 增加测试样本到至少24个月
