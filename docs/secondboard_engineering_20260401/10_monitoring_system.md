# 任务10：二板接力监控与衰减报警系统

## 任务定位
**阶段**：工程阶段  
**优先级**：P2  
**依赖**：任务01-09  
**产出**：监控系统设计文档 V1.0

## 背景
策略上线后，需要持续监控：
1. 策略表现是否正常
2. 是否出现衰退迹象
3. 是否触发风控
4. 系统是否稳定

必须建立自动化监控系统。

```
任务：设计二板接力的监控指标体系
```

### 提示词 10.3：报警机制设计
```
任务：设计二板接力的报警机制

报警设计：

1. 报警触发条件
   A. 收益类报警
      - 单日亏损 > 3%：黄色报警
      - 单日亏损 > 5%：红色报警
      - 连续3日亏损：黄色报警
      - 连续5日亏损：红色报警
   
   B. 风险类报警
      - 回撤 > 10%：黄色报警
      - 回撤 > 15%：红色报警
      - 波动率 > 历史均值+2σ：黄色报警
   
   C. 策略类报警
      - 胜率 < 70%（近20日）：黄色报警
      - 胜率 < 60%（近20日）：红色报警
      - 信号命中率 < 50%：黄色报警
   
   D. 系统类报警
      - 数据延迟 > 10分钟：黄色报警
      - 数据延迟 > 30分钟：红色报警
      - 系统异常：红色报警

2. 报警级别
   - 绿色（正常）：无异常
   - 黄色（观察）：需要关注
   - 橙色（警告）：需要处理
   - 红色（紧急）：需要立即处理

3. 报警方式
   - 绿色：仅仪表盘显示
   - 黄色：仪表盘 + 邮件通知
   - 橙色：仪表盘 + 邮件 + 短信通知
   - 红色：仪表盘 + 邮件 + 短信 + 电话通知

4. 报警处理流程
   A. 黄色报警
      - 记录报警信息
      - 发送邮件通知
      - 相关人员关注
      - 24小时内处理
   
   B. 橙色报警
      - 记录报警信息
      - 发送邮件+短信通知
      - 相关人员响应
      - 4小时内处理
      - 触发风控降级
   
   C. 红色报警
      - 记录报警信息
      - 发送邮件+短信+电话通知
      - 相关人员立即响应
      - 1小时内处理
      - 触发熔断机制

5. 报警记录
   - 报警时间
   - 报警类型
   - 报警级别
   - 触发条件
   - 当前状态
   - 处理措施
   - 处理结果
   - 处理人

输出格式：
| 报警类型 | 触发条件 | 报警级别 | 通知方式 | 处理时限 |
|---------|---------|---------|---------|---------|
| 单日亏损 | >5% | 红色 | 邮件+短信+电话 | 1小时 |

给出：
1. 完整报警清单
2. 报警处理流程
3. 报警记录模板
```
