## 2. 策略概览

### 2.1 策略名称

**二板接力策略**

### 2.2 策略定位

| 维度 | 定义 |
|------|------|
| 类型 | 短线进攻型策略 |
| 周期 | 日线级别（T+1） |
| 容量 | 300-500万 |
| 风格 | 小市值、情绪驱动 |
| Alpha来源 | 情绪溢价、资金接力、题材发酵 |

### 2.3 策略表现（实测数据）

| 年份 | 交易次数 | 胜率 | 平均收益 | 累计收益 | 最大回撤 | 状态 |
|------|---------|------|---------|---------|---------|------|
| 2021 | 214次 | 43.5% | -1.03% | - | - | ❌失效 |
| 2022 | 待测 | 待测 | 待测 | 待测 | 待测 | ⏳待测 |
| 2023 | 218次 | 48.2% | -0.26% | - | - | ❌失效 |
| 2024 | 83次 | 87.95% | 4.76% | 394.61% | 0.60% | ✅有效 |

**关键指标（2024年实测）：**
- 年化收益：280-320%（扣除滑点后）
- 胜率：80-85%
- 最大回撤：≤15%
- 夏普比率：≥2.0

### 2.4 风险提示

**核心风险：**

1. **周期性失效风险**
   - 2021年、2023年实测失效
   - 2024年实测有效
   - 无法预测何时有效/失效
   - 需要持续监控胜率变化

2. **策略衰减风险**
   - 短线策略半衰期1-2年
   - 市场环境变化可能导致失效
   - 拥挤交易可能导致收益下降

3. **流动性风险**
   - 小市值股票成交额有限
   - 涨跌停可能无法成交
   - 大单可能造成冲击成本

4. **过拟合风险**
   - 历史表现不代表未来
   - 实测数据推翻了推断数据
   - 推断方法不可靠

**风险应对：**
- 持续OOS验证（月度评估）
- 动态仓位控制
- 胜率监控（低于50%暂停）
- 小仓位试错（不作为核心策略）

## 4. 每日操作流程

### 4.1 开盘前准备（8:00-9:15）

#### A. 数据更新（8:00-8:30）

**操作内容：**
1. 获取昨日收盘数据
2. 计算二板候选股（昨日涨停股票中今日继续涨停的）
3. 更新财务数据（如有季报发布）
4. 检查停牌信息

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 获取昨日全市场数据 | 数据完整性检查 | 数据缺失则使用备用数据源 |
| 2 | 筛选昨日涨停股票 | 排除一字板、ST股 | 无涨停股则当日无信号 |
| 3 | 筛选今日继续涨停的股票 | 检查是否为二板 | 无二板则当日无信号 |
| 4 | 应用筛选规则 | 市值、换手率、缩量等 | 筛选后生成候选列表 |
| 5 | 按市值排序 | 从小到大排序 | 取市值最小的候选股 |

#### B. 状态判定（8:30-8:45）

**操作内容：**
1. 判定策略可用性
2. 检查风控状态
3. 确认仓位限制

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 检查近期胜率 | 计算10日胜率 | 胜率<50%则暂停交易 |
| 2 | 检查近期回撤 | 计算最大回撤 | 回撤>15%则降低仓位 |
| 3 | 检查连续亏损 | 计算连续亏损次数 | 连续5笔亏损则暂停1日 |
| 4 | 检查系统状态 | 系统连接、权限确认 | 系统故障则暂停交易 |

#### C. 信号生成（8:45-9:00）

**操作内容：**
1. 应用完整筛选规则
2. 计算信号强度
3. 生成买入候选列表

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 应用基础筛选 | 上市天数、ST、停牌 | 不符合条件的排除 |
| 2 | 应用市值筛选 | 流通市值<30亿 | 按市值从小到大排序 |
| 3 | 应用流动性筛选 | 换手率<30%，成交额>5000万 | 不符合条件的排除 |
| 4 | 应用缩量条件 | 昨日成交量≤前日×1.875 | 可选，符合条件优先级更高 |
| 5 | 生成候选列表 | 按优先级排序 | 无候选股则当日无交易 |

#### D. 计划制定（9:00-9:15）

**操作内容：**
1. 确定今日买入计划
2. 确定今日卖出计划
3. 计算所需资金
4. 确认仓位限制

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 检查持仓 | 查看昨日持仓 | 有持仓则需卖出 |
| 2 | 制定卖出计划 | 按卖出规则执行 | 跌停则无法卖出，等待 |
| 3 | 制定买入计划 | 根据候选列表 | 无候选股则不买入 |
| 4 | 计算资金需求 | 单票≤50万，总仓位≤100% | 资金不足则减少买入 |
| 5 | 确认仓位限制 | 总仓位≤100% | 超限则减少买入数量 |

### 4.2 盘中操作（9:15-15:00）

#### A. 竞价阶段（9:15-9:25）

**操作内容：**
1. 观察候选股竞价情况
2. 根据竞价表现调整买入计划
3. 下单竞价买入（可选）

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 观察竞价涨幅 | 竞价涨幅在[-2%, +5%]区间 | 超出区间则降低优先级 |
| 2 | 观察竞价量 | 竞价量是否充足 | 竞价量不足则放弃 |
| 3 | 检查涨停开盘 | 涨停开盘则放弃买入 | 涨停无法买入 |
| 4 | 调整买入计划 | 根据竞价表现调整 | 竞价异常则减少买入量 |

#### B. 开盘阶段（9:30-10:30）

**操作内容：**
1. 执行买入计划
2. 执行卖出计划（如有）
3. 监控持仓表现
4. 触发止损/止盈

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 开盘买入 | 非涨停开盘买入 | 涨停开盘则放弃 |
| 2 | 执行卖出 | 卖出昨日持仓 | 跌停则无法卖出，等待 |
| 3 | 监控持仓 | 观察持仓股票走势 | 异常走势则准备止损 |
| 4 | 10:30检查 | 检查是否触发止损 | 亏损则考虑止损 |

#### C. 午盘阶段（10:30-13:30）

**操作内容：**
1. 持续监控持仓
2. 触发时间止损
3. 调整仓位

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 持续监控 | 观察分时走势 | 跌破分时均线则警惕 |
| 2 | 13:30检查 | 检查是否触发止损 | 亏损则考虑止损 |
| 3 | 触发止盈 | 盈利≥10%可选择止盈 | 涨停则继续持有 |

#### D. 尾盘阶段（13:30-15:00）

**操作内容：**
1. 执行尾盘卖出（如需）
2. 准备收盘记录

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 检查涨停 | 持仓涨停则继续持有 | 涨停则次日再卖 |
| 2 | 收盘前卖出 | 14:55左右卖出 | 跌停则无法卖出，等待次日 |
| 3 | 记录成交价 | 记录卖出价格 | 滑点计算 |

### 4.3 收盘后处理（15:00-16:00）

#### A. 数据记录（15:00-15:20）

**操作内容：**
1. 记录今日交易
2. 更新持仓记录
3. 计算今日收益

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 记录买入 | 记录买入股票、价格、数量 | 滑点计算 |
| 2 | 记录卖出 | 记录卖出股票、价格、数量 | 收益计算 |
| 3 | 更新持仓 | 更新当前持仓列表 | 涨停持仓特殊标记 |
| 4 | 计算收益 | 计算单笔收益、累计收益 | 记录到交易记录模板 |

#### B. 绩效计算（15:20-15:40）

**操作内容：**
1. 计算累计收益
2. 计算回撤
3. 更新风控指标

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 计算累计收益 | 累计收益 = Σ单笔收益 | 记录到绩效表 |
| 2 | 计算最大回撤 | 最大回撤 = 最大峰值-当前值 | 回撤>15%则预警 |
| 3 | 计算胜率 | 近10日胜率 | 胜率<50%则预警 |
| 4 | 计算盈亏比 | 平均盈利/平均亏损 | 监控盈亏比变化 |

#### C. 验证检查（15:40-15:50）

**操作内容：**
1. 检查策略表现是否正常
2. 检查风控是否触发
3. 生成验证报告

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 对比历史表现 | 对比2024年表现 | 表现异常则分析原因 |
| 2 | 检查风控触发 | 检查熔断是否触发 | 触发则执行应对措施 |
| 3 | 生成验证报告 | 生成当日验证报告 | 异常情况详细记录 |

#### D. 次日准备（15:50-16:00）

**操作内容：**
1. 生成次日候选列表
2. 制定次日计划

**具体步骤：**
| 步骤 | 操作内容 | 注意事项 | 异常处理 |
|------|---------|---------|---------|
| 1 | 预筛选次日候选 | 根据今日涨停股预筛选 | 无候选则次日可能无交易 |
| 2 | 制定次日计划 | 确定次日买入/卖出计划 | 风控触发则次日暂停 |
| 3 | 数据备份 | 备份当日数据 | 防止数据丢失 |

## 5. 异常处理手册

### 5.1 数据异常

#### 异常名称：数据缺失

| 项目 | 内容 |
|------|------|
| 异常描述 | 行情数据、情绪数据、财务数据缺失 |
| 识别方法 | 数据完整性检查（数据为空或异常） |
| 处理流程 | 1. 检查数据源连接 2. 使用备用数据源 3. 如无备用数据，暂停交易 |
| 预防措施 | 多数据源备份，定期数据质量检查 |
| 责任人 | 技术支持人员 |

#### 异常名称：数据错误

| 项目 | 内容 |
|------|------|
| 异常描述 | 价格异常、成交量异常、数据不一致 |
| 识别方法 | 数据校验（价格波动>20%、成交量为负等） |
| 处理流程 | 1. 标记异常数据 2. 人工确认 3. 如确认错误，使用备用数据或暂停交易 |
| 预防措施 | 数据校验规则，多数据源对比 |
| 责任人 | 策略执行人员 |

#### 异常名称：数据延迟

| 项目 | 内容 |
|------|------|
| 异常描述 | 实时数据延迟超过5分钟 |
| 识别方法 | 对比数据时间戳 |
| 处理流程 | 1. 检查网络连接 2. 使用历史数据预估 3. 如延迟严重，暂停交易 |
| 预防措施 | 网络质量监控，备用网络 |
| 责任人 | 技术支持人员 |

### 5.2 交易异常

#### 异常名称：无法买入

| 项目 | 内容 |
|------|------|
| 异常描述 | 涨停无法买入、停牌无法买入、流动性不足 |
| 识别方法 | 检查股票状态（涨停、停牌、成交额） |
| 处理流程 | 1. 涨停：放弃买入，等待次日 2. 停牌：放弃买入，等待复牌 3. 流动性不足：减少买入量或放弃 |
| 预防措施 | 筛选规则已排除涨停、停牌股票 |
| 责任人 | 策略执行人员 |

#### 异常名称：无法卖出

| 项目 | 内容 |
|------|------|
| 异常描述 | 跌停无法卖出、停牌无法卖出 |
| 识别方法 | 检查股票状态（跌停、停牌） |
| 处理流程 | 1. 跌停：等待开板卖出，持续监控 2. 停牌：等待复牌卖出 3. 如连续跌停，触发熔断 |
| 预防措施 | 止损规则及时止损，避免跌停 |
| 责任人 | 策略执行人员 |

#### 异常名称：滑点过大

| 项目 | 内容 |
|------|------|
| 异常描述 | 滑点超过预期（>1%） |
| 识别方法 | 对比实际成交价与预期价格 |
| 处理流程 | 1. 记录滑点 2. 分析滑点原因 3. 调整买入量或暂停交易 |
| 预防措施 | 单票仓位≤日均成交额5%，流动性筛选 |
| 责任人 | 策略执行人员 |

#### 异常名称：订单错误

| 项目 | 内容 |
|------|------|
| 异常描述 | 下单数量错误、下单价格错误 |
| 识别方法 | 对比订单与计划 |
| 处理流程 | 1. 发现错误立即撤销订单 2. 重新下单 3. 如已成交，人工处理 |
| 预防措施 | 订单确认检查，自动化下单系统 |
| 责任人 | 策略执行人员 |

### 5.3 系统异常

#### 异常名称：系统故障

| 项目 | 内容 |
|------|------|
| 异常描述 | 交易系统故障、数据系统故障 |
| 识别方法 | 系统响应异常、无法连接 |
| 处理流程 | 1. 切换备用系统 2. 如无备用系统，暂停交易 3. 人工处理紧急事务 |
| 预防措施 | 备用系统，定期系统检查 |
| 责任人 | 技术支持人员 |

#### 异常名称：网络异常

| 项目 | 内容 |
|------|------|
| 异常描述 | 网络中断、网络延迟 |
| 识别方法 | 网络连接检查 |
| 处理流程 | 1. 使用备用网络 2. 如无备用网络，电话下单 3. 网络恢复后数据同步 |
| 预防措施 | 备用网络（有线+无线） |
| 责任人 | 技术支持人员 |

#### 异常名称：权限异常

| 项目 | 内容 |
|------|------|
| 异常描述 | 交易权限受限、数据权限受限 |
| 识别方法 | 权限检查失败 |
| 处理流程 | 1. 联系券商开通权限 2. 如无法开通，暂停交易 3. 寻找替代方案 |
| 预防措施 | 定期权限检查，权限到期前提前续期 |
| 责任人 | 策略执行人员 |

### 5.4 策略异常

#### 异常名称：信号异常

| 项目 | 内容 |
|------|------|
| 异常描述 | 信号数量异常（过多或过少）、信号质量异常 |
| 识别方法 | 对比历史信号数量和质量 |
| 处理流程 | 1. 检查筛选规则是否正确 2. 人工审核信号质量 3. 如异常严重，暂停交易 |
| 预防措施 | 信号质量监控，自动化信号审核 |
| 责任人 | 策略执行人员 |

#### 异常名称：表现异常

| 项目 | 内容 |
|------|------|
| 异常描述 | 连续亏损、单日巨亏 |
| 识别方法 | 监控胜率、收益 |
| 处理流程 | 1. 连续5笔亏损：暂停交易1日 2. 单日亏损>5%：暂停交易1日 3. 近10日胜率<50%：降低仓位50% |
| 预防措施 | 熔断规则，持续OOS验证 |
| 责任人 | 风险管理人员 |

#### 异常名称：参数异常

| 项目 | 内容 |
|------|------|
| 异常描述 | 参数失效、参数设置错误 |
| 识别方法 | 对比参数设置与推荐参数 |
| 处理流程 | 1. 检查参数设置 2. 对比历史表现 3. 重新验证参数有效性 |
| 预防措施 | 参数文档化管理，参数敏感性分析 |
| 责任人 | 策略开发人员 |

### 5.5 市场异常

#### 异常名称：极端行情

| 项目 | 内容 |
|------|------|
| 异常描述 | 大盘暴跌、个股闪崩 |
| 识别方法 | 大盘跌幅>5%、个股跌幅>10% |
| 处理流程 | 1. 大盘暴跌：触发风控，暂停交易 2. 个股闪崩：立即止损卖出 3. 如无法卖出，触发熔断 |
| 预防措施 | 情绪监控（虽然不接情绪层，但仍监控市场情绪），熔断规则 |
| 责任人 | 风险管理人员 |

#### 异常名称：政策异常

| 项目 | 内容 |
|------|------|
| 异常描述 | 交易规则变化、政策变化 |
| 识别方法 | 新闻监控、政策公告 |
| 处理流程 | 1. 评估政策影响 2. 调整策略参数 3. 如影响严重，暂停策略重新评估 |
| 预防措施 | 政策监控，策略灵活性 |
| 责任人 | 策略开发人员 |

## 11. 核心提示

### 11.1 策略特殊性

**周期性失效风险：**
- 二板接力策略存在周期性失效问题
- 2021年、2023年实测失效，2024年实测有效
- 无法预测何时有效/失效
- **必须持续OOS验证（月度胜率监控）**

### 11.2 情绪层不接入

**实测数据支撑：**
- 无情绪过滤：671.90%收益，0.58%回撤，86.33%胜率（2024年）
- 硬开关（涨停≥10）：394.61%收益，0.60%回撤，87.95%胜率
- **结论：不接情绪层，二板本身回撤已经很低，情绪层反而过度过滤**

### 11.3 风控重要性

**关键风控指标：**
- 近10日胜率<50%：降低仓位50%
- 近20日收益<-5%：暂停策略，重新评估
- 连续5笔亏损：暂停交易1日
- 最大回撤>15%：降低仓位50%

### 11.4 小仓位试错

**仓位建议：**
- 最高占进攻线10%（小仓位试错）
- 不作为核心策略
- 单票≤50万，总仓位≤100%
