## 任务定位
**阶段**：集成阶段

## 背景
二板接力策略已知对情绪状态敏感，需要：
1. 集成情绪开关
2. 集成市场广度开关
3. 设计状态路由器
4. 测试择时效果

### 提示词 8.1：情绪开关设计
```
任务：设计二板接力的情绪开关

设计步骤：

1. 情绪指标选择
   测试以下指标的单独效果：

1. 情绪指标推荐
2. 情绪阈值推荐
3. 情绪状态定义
4. 情绪转换规则
```

### 提示词 8.3：状态路由器设计
```
任务：设计二板接力的状态路由器

设计步骤：

1. 状态定义
   - 关闭：完全停止交易
   - 防守：降低仓位或减少信号
   - 正常：正常交易
   - 进攻：增加仓位或放宽信号

2. 状态判定规则
   - 关闭：广度<15% 或 情绪冰点
   - 防守：广度15-25% 或 情绪启动
   - 正常：广度25-35% 或 情绪发酵
   - 进攻：广度>=35% 且 情绪发酵/高潮

3. 每种状态下的行为
   - 关闭：不交易
   - 防守：仓位降至50%，只选最强信号
   - 正常：正常仓位，正常信号
   - 进攻：仓位提升至120%，可放宽信号

4. 状态转换规则
   - 关闭 -> 防守：广度回升至15%以上 且 情绪改善
   - 防守 -> 正常：广度回升至25%以上
   - 正常 -> 进攻：广度>=35% 且 情绪发酵
   - 进攻 -> 正常：广度下降或情绪回落
   - 正常 -> 防守：广度下降至25%以下
   - 防守 -> 关闭：广度下降至15%以下

5. 滞后机制
   - 避免频繁切换
   - 状态转换需连续2日确认
   - 设置最短持续时间

输出格式：
| 状态 | 判定条件 | 仓位 | 信号选择 | 持续时间 |
|------|---------|------|---------|---------|
| 关闭 | 广度<15% | 0% | 无 | 最短1日 |

给出：
1. 状态定义与判定规则
2. 每种状态下的策略行为
3. 状态转换规则
4. 滞后机制设计
```
