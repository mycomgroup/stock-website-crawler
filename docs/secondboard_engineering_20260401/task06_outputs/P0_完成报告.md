## 二、系统功能概览

### 2.1 数据管理系统

**核心功能**：
- SQLite数据库自动初始化
- 行情、情绪、广度、交易记录四类数据管理
- 数据完整性检查
- 数据质量日志记录

**关键方法**：
- `save_price_data()` - 保存行情数据
- `save_trade_record()` - 保存交易记录
- `query_trade_records()` - 查询交易记录
- `check_data_completeness()` - 检查数据完整性

### 2.2 衰退检测系统

**核心功能**：
- 四级衰退定义（轻度、中度、重度、失效）
- 四级预警判定（绿色、黄色、橙色、红色）
- 多指标综合评估（年化收益、胜率、回撤、夏普）
- 统计检验（t检验，p值计算）
- 趋势检测（连续恶化识别）
- 应对措施自动生成

**关键方法**：
- `detect_decay()` - 综合衰退检测
- `statistical_test()` - 统计检验
- `trend_detection()` - 趋势检测
- `generate_decay_report()` - 生成衰退检测报告

### 2.3 自动化验证系统

**核心功能**：
- 日频验证（单日验证）
- 周频验证（近20日滚动窗口）
- 月频验证（近60日滚动窗口）
- 指标自动计算（收益、风险、质量）
- 验证结果持久化存储

**关键方法**：
- `run_daily_validation()` - 执行日频验证
- `run_weekly_validation()` - 执行周频验证
- `run_monthly_validation()` - 执行月频验证

**指标计算**：
- 收益指标：累计收益率、年化收益率
- 风险指标：最大回撤、波动率、下行波动率
- 风险调整收益：夏普比率、卡玛比率
- 交易质量：胜率、盈亏比、交易次数

### 2.4 报告生成系统

**核心功能**：
- 日报生成（交易概览、收益统计、风险监控）
- 周报生成（滚动窗口验证、基准对比）
- 月报生成（详细统计、衰退检测、应对措施）
- Markdown格式输出

**关键方法**：
- `generate_daily_report()` - 生成日报
- `generate_weekly_report()` - 生成周报
- `generate_monthly_report()` - 生成月报

**预期输出**：
```

  预警级别: green
  累计收益: 4.00%
  胜率: 100.00%
```

### 3.2 基本使用示例

```python
from main import OOSValidationSystem

# 初始化系统
system = OOSValidationSystem('my_oos.db')

# 添加交易记录
trade = {
    'signal_date': '2024-01-01',
    'signal_stock_code': '000001.SZ',
    'signal_price': 10.0,
    'execution_date': '2024-01-01',
    'execution_price': 10.1,
    'execution_volume': 1000,
    'sell_date': '2024-01-02',
    'sell_price': 10.5,
    'sell_reason': '高位回落',
    'profit_loss_pct': 0.04
}
system.add_trade_record(trade)

# 执行月频验证
result = system.run_monthly_validation('2024-01-31')

# 输出结果
print(f"预警级别: {result['alert_level']}")
print(f"年化收益: {result['annual_return']:.2f}%")
print(f"夏普比率: {result['sharpe_ratio']:.2f}")

# 关闭系统
system.close()
```

## 六、关键特性

### 6.1 完全自动化

- **数据管理自动化**：自动创建数据库、自动质量检查
- **验证流程自动化**：自动计算指标、自动判定预警
- **报告生成自动化**：自动生成多频次报告

### 6.2 四级预警机制

| 预警级别 | 触发条件 | 应对措施 | 仓位调整 |
|---------|---------|---------|---------|
| 绿色（正常） | 所有指标达标 | 正常执行 | 30-40% |
| 黄色（观察） | 个别指标观察范围，持续≥2周 | 加强监控 | 维持原仓位 |
| 橙色（警告） | 多个指标警告范围，持续≥1个月 | 降仓50% | 15-20% |
| 红色（降级） | 关键指标失效 | 暂停策略 | 0% |

### 6.3 多维度指标体系

**收益指标**：年化收益率、累计收益率  
**风险指标**：最大回撤、波动率、下行波动率  
**风险调整收益**：夏普比率、卡玛比率  
**交易质量**：胜率、盈亏比、交易次数、平均持仓时长

### 6.4 灵活的验证频率

- **日频验证**：每日收盘后，快速监控当日表现
- **周频验证**：每周五，近20日滚动窗口评估
- **月频验证**：每月末，近60日深度验证+衰退检测

## 七、使用注意事项

### 7.1 数据要求

- **交易记录必需字段**：signal_date, signal_stock_code, execution_date, execution_price, profit_loss_pct
- **建议每日更新交易记录**
- **确保数据完整性，定期检查数据质量**

### 7.2 验证频率建议

- **日频**：每日收盘后15:30执行
- **周频**：每周五收盘后15:30执行
- **月频**：每月末收盘后15:30执行

### 7.3 预警响应流程

**黄色预警**：
1. 验证频率提升至日频
2. 每日人工审核交易信号
3. 启动衰退原因分析
4. 准备应对预案

**橙色预警**：
1. 立即降低仓位至50%
2. 暂停新开仓操作
3. 风控部门介入监督
4. 深度分析衰退原因

**红色预警**：
1. 立即暂停策略执行
2. 开始清空所有持仓
3. 全面复盘分析
4. 重新评估策略有效性
