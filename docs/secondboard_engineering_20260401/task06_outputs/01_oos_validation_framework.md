# OOS验证框架设计

## 一、验证框架总览

### 1.1 框架设计理念

OOS验证框架基于以下核心理念：
- **持续性**：验证应贯穿策略生命周期，而非一次性检验
- **多维度**：从收益、风险、交易质量等多角度评估
- **对比性**：建立多维基准，确保相对表现可评估
- **自动化**：设计可自动化执行流程，降低人工干预成本

## 二、验证频率设计

### 2.1 日频验证（T+1模式）

**触发时间**：每日收盘后（15:30-16:30）

**验证目的**：
- 监控当日交易表现
- 识别单日异常情况
- 维护持仓风险控制

**验证内容**：
| 验证项目 | 验证窗口 | 验证指标 | 验证基准 | 验证方法 |
|---------|---------|---------|---------|---------|
| 当日收益 | 当日 | 单笔收益率、组合收益率 | 无策略（0%） | 绝对验证 |
| 累计收益 | 本月累积 | 累计收益、年化收益（月度化） | 沪深300 | 相对验证 |
| 持仓风险 | 当前 | 持仓数量、持仓市值、平均持仓时长 | 无限制 | 绝对验证 |
| 异常标记 | 当日 | 信号数量异常、收益率异常 | 历史平均±2σ | 统计验证 |

**自动化流程**：
1. 15:30 数据更新完成
2. 15:35 执行当日交易记录提取
3. 15:40 计算当日收益指标
4. 15:45 异常检测判定
5. 16:00 生成日报并发送通知

### 2.2 周频验证（滚动窗口）

**触发时间**：每周五收盘后（15:30-17:00）

**验证目的**：
- 评估短期策略表现
- 检测周度趋势变化
- 验证信号质量稳定性

**验证内容**：
| 验证项目 | 验证窗口 | 验证指标 | 验证基准 | 验证方法 |
|---------|---------|---------|---------|---------|
| 本周收益 | 本周 | 周收益率、日均收益 | 沪深300（本周）、中证1000（本周） | 相对验证 |
| 滚动收益 | 近20日 | 累计收益、年化收益 | 沪深300（近20日） | 相对验证、时序验证 |
| 交易质量 | 本周 | 交易次数、胜率、盈亏比 | 历史平均水平 | 时序验证 |
| 信号质量 | 本周 | 信号数量、命中率 | 历史平均±1σ | 统计验证 |
| 风险指标 | 近20日 | 最大回撤、波动率 | 历史平均水平 | 绝对验证 |

**自动化流程**：
1. 15:30 数据更新完成
2. 15:35 提取本周及近20日交易数据
3. 15:50 计算周度及滚动指标
4. 16:10 生成周度趋势分析
5. 16:30 生成周报并发送通知

### 2.3 月频验证（综合评估）

**触发时间**：每月末收盘后（15:30-18:00）

**验证目的**：
- 月度表现综合评估
- 基准对比与超额收益计算
- 衰退检测预警判定

**验证内容**：

|---------|---------|---------|---------|---------|
| 本月收益 | 本月 | 月收益率、年化收益 | 沪深300、中证1000 | 相对验证 |
| 累积收益 | 本年累积 | 累积收益率、年化收益 | 沪深300、中证1000、历史回测（IS） | 相对验证、时序验证 |
| 滚动收益 | 近60日 | 累计收益、年化收益、夏普比率 | 历史平均水平 | 时序验证 |
| 交易统计 | 本月 | 交易次数、胜率、盈亏比、平均持仓时长 | 历史平均水平 | 统计检验 |
| 风险分析 | 本月 | 最大回撤、波动率、下行风险 | 历史平均水平 | 绝对验证 |

1. 15:30 数据更新完成

**触发时间**：每季末收盘后（15:30-18:30）

**验证目的**：
- 季度表现深度评估
- 市场环境变化分析
- 策略有效性长期验证

**验证内容**：
| 验证项目 | 验证窗口 | 验证指标 | 验证基准 | 验证方法 |
|---------|---------|---------|---------|---------|
| 本季收益 | 本季 | 季收益率、年化收益 | 沪深300、中证1000 | 相对验证 |
| 累积收益 | 上线至今 | 累积收益率、年化收益、夏普比率 | IS表现、沪深300 | 相对验证、时序验证 |
| 滚动收益 | 近120日 | 累计收益、年化收益、夏普比率、卡玛比率 | 历史平均水平 | 时序验证 |
| 市场环境 | 本季 | 情绪指标变化、风格因子变化、波动率变化 | 历史平均水平 | 对比分析 |
| 策略评估 | 近120日 | 策略有效性指标、衰减趋势分析 | IS表现 | 统计检验 |

**自动化流程**：
1. 15:30 数据更新完成

## 五、验证基准设计

### 5.1 基准选择原则

1. **相关性原则**：基准应与策略风格相关
2. **代表性原则**：基准应代表市场整体表现
3. **稳定性原则**：基准数据应稳定可靠
4. **可比性原则**：基准应有足够历史数据用于对比

### 5.2 基准体系

| 基准类型 | 基准名称 | 选择理由 | 使用场景 |
|---------|---------|---------|---------|
| 空仓基准 | 无策略（0收益） | 评估策略绝对价值 | 日频验证 |
| 大盘基准 | 沪深300指数 | 代表大盘表现 | 所有频次 |
| 小盘基准 | 中证1000指数 | 代表小盘表现，与策略风格相关 | 所有频次 |
| 历史基准 | IS表现（2021-2023） | 策略回测基准，用于衰退检测 | 月频、季频 |

### 5.3 基准数据获取

| 基准 | 数据来源 | 更新频率 | 存储方式 |
|------|---------|---------|---------|
| 沪深300 | JoinQuant API | 日频 | 本地SQLite |
| 中证1000 | JoinQuant API | 日频 | 本地SQLite |
| IS表现 | 回测结果 | 固定 | 配置文件 |

## 六、验证方法设计

### 6.1 绝对验证

**定义**：指标是否达到预设标准

**应用场景**：
- 回撤是否超标
- 波动率是否异常
- 交易频率是否合理

**判定逻辑**：
```
if indicator >= threshold:
    status = "达标"
else:
    status = "未达标"
    alert_level = 计算偏离程度
```

### 6.2 相对验证

**定义**：策略是否跑赢基准

**应用场景**：
- vs沪深300超额收益
- vs中证1000超额收益

**判定逻辑**：
```
超额收益 = 策略收益 - 基准收益
if 超额收益 > 0:
    status = "跑赢基准"
    excess_level = 超额收益 / 基准收益
else:
    status = "跑输基准"
    alert_level = 计算跑输幅度
```

### 6.3 时序验证

**定义**：当前表现是否与历史表现一致

**应用场景**：
- 与IS表现对比
- 与历史平均水平对比

**判定逻辑**：
```
偏离度 = (当前指标 - 历史平均) / 历史标准差
if abs(偏离度) <= 1:
    status = "正常"
elif abs(偏离度) <= 2:
    status = "异常观察"
else:
    status = "显著异常"
    alert_level = 偏离度
```

### 6.4 统计验证

**定义**：策略收益是否显著为正

**应用场景**：
- 验证策略有效性
- 衰退检测显著性检验

**判定逻辑**：
```
# t检验
t_stat = (平均收益 - 0) / (标准差 / sqrt(样本数))
p_value = t_distribution(t_stat, 样本数-1)

if p_value < 0.05:
    status = "显著为正"
    confidence = "95%"
else:
    status = "不显著"
```
