# 二板接力策略逻辑文档 V1.0

---

## 1. 策略名称与定位

- **名称**：二板接力策略
- **定位**：短线进攻型策略
- **周期**：日线级别（持仓周期：T+1，次日卖出）
- **容量**：单票上限300万元（已验证）
- **风格**：小市值进攻，高风险高收益
- **适用场景**：情绪高涨期（涨停家数≥10）

---

## 2. 核心逻辑

### 2.1 Alpha来源

**主要Alpha来源：情绪溢价与资金接力**

1. **情绪溢价（60%）**
   - 板块热度持续发酵，市场情绪高涨
   - 涨停家数≥10表明市场情绪健康
   - 二板股票享有情绪溢价空间

2. **资金接力（30%）**
   - 首板分歧后，次日资金承接转强
   - 游资接力，形成连板效应
   - 缩量表明首板分歧充分，承接有力

3. **技术形态突破（10%）**
   - 连续涨停突破前期平台
   - 形成加速上攻形态
   - 市值排序优选小盘股（流动性+弹性）

### 2.2 盈利机制

**盈利逻辑链条：**

```
情绪高涨 → 涨停活跃 → 首板分歧 → 缩量承接 → 二板接力 → 情绪溢价 → 次日冲高 → 盈利兑现
```

**关键盈利点：**
1. **买入时机**：非涨停开盘，承接转强时买入
2. **持有周期**：日内持有，次日冲高卖出
3. **盈利幅度**：平均日内收益4.77%（2024实测）
4. **胜率保障**：87.95%胜率，盈亏比21.91

### 2.3 风险来源

**主要风险点：**

1. **情绪风险（40%）**
   - 涨停家数<10时，市场情绪低迷
   - 板块轮动过快，热点切换
   - 情绪周期切换，二板承接失败

2. **流动性风险（30%）**
   - 小市值股票流动性有限
   - 单票仓位过大导致滑点
   - 涨停板无法买入

3. **个股风险（20%）**
   - 深度低开（开盘跌幅>-4%）
   - 高开低走，冲高回落
   - 一字板无法买入

4. **策略衰减风险（10%）**
   - 短线策略半衰期1-2年
   - 市场风格切换
   - 参数过拟合

## 3. 关键定义

### 3.1 首板定义

**聚宽平台标准定义：**

| 定义项 | 标准 | 说明 |
|-------|------|------|
| **涨停判定** | `close == high_limit` | 收盘价等于涨停价 |
| **涨幅阈值** | ≥ 9.9% | A股涨停标准 |
| **一字板** | `low == high_limit` | 最低价=涨停价，开盘即涨停 |
| **T字板** | `low < high_limit` 且 `close == high_limit` | 有小幅波动后涨停 |
| **ST股票** | 排除 | 不参与ST股票 |
| **科创板/创业板** | 排除 | 代码开头为'4'、'8'、'68'排除 |
| **新股** | 排除上市<250天 | 流动性不足 |

**首板筛选条件：**
- ✓ 收盘价 = 涨停价（误差≤1%）
- ✓ 非一字板（开盘价<涨停价）
- ✓ 非ST股票
- ✓ 非科创板/创业板
- ✓ 上市≥250天

### 3.2 二板定义

**二板判定标准：**

| 定义项 | 标准 | 说明 |
|-------|------|------|
| **连续涨停** | 连续2日涨停 | 前日涨停+昨日涨停 |
| **非一字板** | 昨日非一字涨停 | 有成交空间 |
| **换手率** | < 30% | 避免过度活跃 |
| **缩量条件** | 昨日量 ≤ 前日量 × 1.875 | 承接有力，分歧充分 |
| **市值范围** | 流通市值5-50亿 | 小盘股为主 |
| **封板时间** | 不限 | 待验证 |
| **封单量** | 不限 | 待验证 |

**二板筛选条件：**
- ✓ 前日涨停（首板）
- ✓ 昨日涨停（二板）
- ✓ 昨日非一字板
- ✓ 换手率 < 30%
- ✓ 缩量条件：昨日成交量 ≤ 前日成交量 × 1.875
- ✓ 流通市值5-50亿（排序优先小市值）

**核心逻辑：**
```
首板分歧（缩量） → 承接转强 → 二板接力 → 情绪溢价
```

### 3.3 接力定义

**接力含义：**
- 资金从首板分歧到二板承接的转换过程
- 情绪从分歧到一致的过渡状态

**接力条件：**
1. **缩量承接**：昨日成交量 ≤ 前日成交量 × 1.875
   - 表明首板分歧充分，资金承接有力
   - 验证依据：2024实测，缩量条件胜率更高

2. **换手率适中**：换手率 < 30%
   - 避免过度活跃导致的流动性风险
   - 表明筹码稳定，接力有序

3. **情绪环境**：涨停家数 ≥ 10
   - 市场情绪健康，接力有空间
   - 情绪低迷时接力失败率高

**接力信号：**
- ✓ 非涨停开盘（开盘价<涨停价）
- ✓ 竞价量适中（无极端竞价）
- ✓ 开盘后承接转强（分时形态）

**接力失败标志：**
- ✗ 深度低开（开盘跌幅>-4%）
- ✗ 高开低走（开盘涨幅>3%，收盘跌幅）
- ✗ 冲高回落（盘中最高涨幅>5%，收盘跌幅）
- ✗ 一字跌停（无法成交）

## 4. 交易规则概览

### 4.1 买入时机

**买入条件：**
- 时机：开盘后（9:30-9:35）
- 价格：非涨停开盘（开盘价<涨停价）
- 方式：市价单买入
- 滑点：+0.5%（实测）

**买入流程：**
```
1. 盘前筛选：昨日二板股票（非一字板）
2. 条件过滤：换手率<30%、缩量、市值排序
3. 情绪过滤：涨停家数≥10（可选）
4. 开盘观察：开盘价<涨停价
5. 市价买入：立即买入，滑点+0.5%
```

**仓位控制：**
- 单票上限：300万元
- 总仓位：根据情绪动态调整
  - 涨停≥20：100%仓位
  - 涨停10-20：50%仓位
  - 涨停<10：0%仓位（空仓）

### 4.2 卖出规则

**卖出时机：**
- 时间：次日开盘或冲高后
- 方式：市价单卖出

**卖出策略（实测对比）：**

| 卖出时机 | 胜率 | 平均收益 | 建议 |
|---------|------|---------|------|
| **次日冲高+3%** | 58.2% | +2.87% | ✅ **推荐** |
| 次日收盘卖出 | - | +4.77% | ⚠️ 回撤风险大 |
| 次日开盘卖出 | 38.7% | -1.02% | ❌ 不推荐 |

**卖出流程：**
```
1. 次日开盘观察：开盘涨幅
2. 冲高卖出：冲高+3%立即卖出
3. 未冲高：尾盘14:50卖出
4. 止损卖出：开盘跌幅>-4%立即止损
```

### 4.3 风控规则

**止损规则：**
- 深度低开：开盘跌幅>-4%立即止损
- 高位回落：从最高价回落>5%立即止损
- 连亏停机：连续亏损3笔停机3天

**仓位熔断：**
- 情绪低迷：涨停家数<10空仓
- 连续亏损：连亏3笔仓位减半
- 回撤控制：最大回撤>5%暂停策略

**风险监控：**
- 每日记录实际胜率
- 监控连续亏损次数
- 跟踪实际滑点与回测偏差

### 6.2 推荐定义及理由

**推荐采用本项目实测定义：**

```
二板 = 连续2日涨停 + 非一字板 + 缩量 + 换手率<30% + 市值排序
```

**理由：**
1. ✓ 实测验证有效（胜率87.95%，年化407.65%）
2. ✓ 缩量条件提升承接质量
3. ✓ 换手率过滤避免流动性风险
4. ✓ 市值排序优选小盘股（流动性+弹性）
5. ✓ 排除一字板保证可成交

**vs 聚宽标准定义差异：**
- + 缩量条件（承接验证）
- + 换手率过滤（流动性控制）
- + 市值排序（优先小盘）
- + 情绪开关（可选，涨停≥10）

### 12.2 一字板判定

```python
def get_hl_count_df(hl_list, date, watch_days):
    """计算涨停数与一字涨停数"""
    df = get_price(hl_list, end_date=date, frequency='daily', 
                   fields=['low','close','high_limit'], count=watch_days, 
                   panel=False, fill_paused=False, skip_paused=False)
    df.index = df.code
    
    hl_count_list = []
    extreme_hl_count_list = []
    
    for stock in hl_list:
        df_sub = df.loc[stock]
        # 涨停天数
        hl_days = df_sub[df_sub.close==df_sub.high_limit].high_limit.count()
        # 一字涨停天数（最低价=涨停价）
        extreme_hl_days = df_sub[df_sub.low==df_sub.high_limit].high_limit.count()
        hl_count_list.append(hl_days)
        extreme_hl_count_list.append(extreme_hl_days)
    
    # 创建df记录
    df = pd.DataFrame(index=hl_list, 
                      data={'count':hl_count_list, 
                            'extreme_count':extreme_hl_count_list})
    return df
```

### 12.3 连板计算函数

```python
def get_continue_count_df(hl_list, date, watch_days):
    """计算连板数"""
    df = pd.DataFrame()
    for d in range(2, watch_days+1):
        HLC = get_hl_count_df(hl_list, date, d)
        CHLC = HLC[HLC['count'] == d]  # 筛选连板数为d的股票
        df = df.append(CHLC)
    
    stock_list = list(set(df.index))
    ccd = pd.DataFrame()
    
    for s in stock_list:
        tmp = df.loc[[s]]
        if len(tmp) > 1:
            M = tmp['count'].max()
            tmp = tmp[tmp['count'] == M]
        ccd = ccd.append(tmp)
    
    if len(ccd) != 0:
        ccd = ccd.sort_values(by='count', ascending=False)
    return ccd
```
