## 二、核心指标详解

### 2.1 市场宽度 (BIAS)

**定义**: 各行业/个股收盘价在20日均线之上的比例

**计算方法**:
```python
# BIAS = (Close - MA20) / MA20
breadth = (close > ma20).mean()  # 比例在0-1之间
```

**阈值划分**:
| 区间 | 状态 | 操作建议 | 仓位建议 |
|------|------|----------|----------|
| **≥0.40** | 正常偏强 | 满仓运行 | 70-80% |
| **0.25-0.40** | 弱势震荡 | 减仓防守 | 50-60% |
| **0.15-0.25** | 底部试错 | 轻仓试探 | 30-40% |
| **<0.15** | 极端弱势 | 空仓观望 | 0-15% |

**历史验证**:
- 2022-04 市场宽度跌至0.13,触发极端底部信号
- 2024-01 市场宽度跌至0.18,触发底部试错
- 2026-03-28 当前市场宽度 **0.230**,处于 **弱势震荡** 区间

**项目应用**:
- RFScore PB10 四档仓位规则已基于市场宽度验证
- 红利小盘策略已验证熊市宽度<0.25时的防守效果

### 2.2 FED指标 + 格雷厄姆指数 (大周期估值)

**定义**: 
- **FED模型**: 沪深300盈利收益率(1/PE) - 10年期国债收益率
- **格雷厄姆指数**: 沪深300 PE × 全市场PE / 国债收益率

**计算方法**:
```python
# FED = 1/PE_median - 10年期国债收益率
fed = 100 / pe_median - bond_10y_yield

# 格雷厄姆指数 = (指数PE × 全市场PE) / 国债收益率
graham = (index_pe * market_pe) / bond_yield
```

**阈值划分**:
| FED值 | 格雷厄姆值 | 市场状态 | 操作建议 |
|-------|-----------|----------|----------|
| **>2%** | >2.0 | 极低估 | 积极建仓 |
| **1-2%** | 1.5-2.0 | 低估 | 正常配置 |
| **0-1%** | 1.0-1.5 | 合理 | 维持仓位 |
| **<0%** | <1.0 | 高估 | 减仓防御 |

**历史验证**:
- 2018-12 FED=4.5%, 格雷厄姆=2.8, 极低估,历史大底
- 2022-04 FED=3.2%, 格雷厄姆=2.3, 低估,熊市底部
- 2026-03 当前 FED≈1.8%, 格雷厄姆≈1.6, **低估偏合理**

**项目应用**:
- 验证报告 `11_macro_regime_router_validation.md` 已确认有效性
- 当前状态: **低估区间,可正常配置**

### 2.3 RSRS择时 (支撑阻力强度)

**定义**: 基于最高价-最低价回归斜率的趋势强度指标

**计算方法**:
```python
# RSRS = (High vs Low)回归斜率 β
# 标准分 = (β - β_mean_M天) / β_std_M天

# 改进版: 成交量加权右偏标准分
vol_rsrs_right = zscore * beta * r2
```

**阈值划分**:
| RSRS得分 | 状态 | 操作建议 |
|-----------|------|----------|
| **>0.85** | 强趋势向上 | 满仓持有 |
| **0.7-0.85** | 趋势偏强 | 维持仓位 |
| **-0.7-0.7** | 无趋势 | 观望或减仓 |
| **<-0.85** | 强趋势向下 | 空仓避险 |

**历史验证**:
- RSRS在2018、2022年熊市底部信号准确率达70%
- 成交量加权版比原始版提升夏普比率约0.15

**项目应用**:
- ETF动量轮动策略已验证RSRS择时效果
- 当前RFScore组合暂未引入,可作为进阶优化

### 2.4 NH-NL净新高占比 (行业指数顶底)

**定义**: 行业指数中创52周新高与新低之差占比

**计算方法**:
```python
# NH-NL% = (新高股数 - 新低股数) / 行业内上市>1年股数
nhnl_pct = (new_high_count - new_low_count) / total_stocks
```

**阈值划分**:
| NH-NL值 | 状态 | 信号 |
|---------|------|------|
| **≥30%** | 贪婪 | 顶部预警,做空信号 |
| **20-30%** | 乐观 | 关注顶部 |
| **-20-20%** | 正常 | 维持 |
| **-30-(-20%)** | 悲观 | 关注底部 |
| **≤-30%** | 恐惧 | 底部信号,做多信号 |

**历史验证**:
- 科技板块(电子、通信、计算机、传媒)顶底识别准确率75%
- 反转策略年化收益约30%,适合指数增强

**项目应用**:
- 可用于行业轮动辅助判断
- 当前暂未引入,可作为扩展模块

### 2.5 拥挤率 (资金集中度)

**定义**: 成交额前5%股票的资金集中度

**计算方法**:
```python
# 拥挤率 = Top5%成交额 / 全市场成交额
crowding = top_5pct_money / total_money * 100
```

**阈值划分**:
| 拥挤率 | 状态 | 风险提示 |
|--------|------|----------|
| **<15%** | 资金分散 | 正常 |
| **15-25%** | 轻度拥挤 | 关注 |
| **25-35%** | 中度拥挤 | 预警 |
| **>35%** | 重度拥挤 | 过热风险 |

**历史验证**:
- 2021-02 拥挤率达42%,触发顶部信号
- 2022-04 拥挤率降至12%,底部特征

**项目应用**:
- 可结合市场宽度使用
- 当前暂未引入,可作为风险预警模块

### 2.6 C-VIX波动率 (恐慌指数)

**定义**: 基于期权隐含波动率的恐慌指数(中国版VIX)

**计算方法**:
```python
# C-VIX = 前两月期权隐含波动率加权平均
cvix = weighted_avg_iv(front_month_options)
```

**阈值划分**:
| C-VIX值 | 状态 | 市场情绪 |
|---------|------|----------|
| **<15** | 低波动 | 市场平静 |
| **15-25** | 正常 | 情绪稳定 |
| **25-35** | 高波动 | 恐慌加剧 |
| **>35** | 极端恐慌 | 底部特征 |

**历史验证**:
- 2020-03 C-VIX达48,历史极端恐慌
- 2022-04 C-VIX达32,熊市恐慌

**项目应用**:
- 可作为仓位调整的辅助指标
- 当前暂未引入,数据源需确认
