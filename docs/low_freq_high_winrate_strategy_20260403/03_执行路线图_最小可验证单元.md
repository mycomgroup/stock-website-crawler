# 执行路线图与最小可验证单元

日期：2026-04-03

## 1. 执行原则

本轮后续探索不再以“大题目”为最小单位，而以“单个可验证结论”为最小单位。

每个任务都应该满足：

1. 能独立回答一个具体问题；
2. 有明确输入材料；
3. 有清晰的通过 / 不通过标准；
4. 能在不依赖全部后续任务完成的情况下单独产出；
5. 即使结论是否定，也能形成可复用的淘汰清单。

## 2. 波次安排

### 第 1 波：口径统一与候选缩表

建议并行任务：

- 01 主仓底座现状对齐与冲突表
- 02 状态输入统一与阈值冻结
- 05 RSRS 改进版本候选筛选
- 08 ML 逻辑回归基线复核与泄露检查
- 13 低频 ETF / 宏观原型映射与白名单

目标：先把“比较对象”和“指标口径”固定下来。

### 第 2 波：最小增量验证

建议并行任务：

- 03 主仓动态映射候选表轻量比较
- 06 RSRS 复合过滤增量验证
- 09 ML 十三因子候选筛选与缩表
- 11 FFScore 对 RFScore 增量验证
- 12 筹码 / STR / 行业量价预筛白名单

目标：只验证“是否值得继续”，不做全量工程化。

### 第 3 波：接口与规格收敛

建议并行任务：

- 04 主仓动态路由 V1 规格书
- 07 RSRS 模块挂接规则与接口定义
- 10 ML 调仓频率与持仓数轻量比较

目标：把前两波的结果收敛成可开发、可编码的下一轮规格。

### 第 4 波：统一收口

建议单独任务：

- 14 下一轮开发顺序与 Agent 排班表

目标：形成最终执行清单，决定哪些进开发，哪些继续观察，哪些淘汰。

## 4. 推荐的 agent 分工方式

### A 类：整编 / 审计型 agent

适合任务：

- 01
- 02
- 05
- 08
- 12
- 13

特点：

- 以阅读、归纳、冲突解释为主；
- 重点是统一口径和缩表；
- 原则上不需要新代码。

### B 类：轻验证 / 对比型 agent

适合任务：

- 03
- 06
- 09
- 10
- 11

特点：

- 允许做小范围验证；
- 重点回答“增量价值是否存在”；
- 只允许比较少数候选，不允许大规模参数搜索。

### C 类：收口 / 规格型 agent

适合任务：

- 04
- 07
- 14

特点：

- 不负责重测；
- 负责把前置任务的结论整理成规格书、接口和下一轮排班表。

## 5. 每波结束后的检查点

### 第 1 波检查点

必须回答：

1. 主仓到底有哪些底座？
2. 状态口径是否已经统一？
3. RSRS 候选是否已缩小到少数版本？
4. ML 结果是否可信？
5. ETF / 宏观原型是否已列出白名单？

### 第 2 波检查点

必须回答：

1. 动态主仓映射是否优于静态映射，至少在哪些场景更优？
2. RSRS 叠加过滤是否真有增量？
3. ML 是否值得进入下一轮扩展？
4. FFScore 是否值得成为 RFScore 增强线？
5. 其他候选增强因子中，哪些只值得保留观察？

### 第 3 波检查点

必须回答：

1. 主仓动态路由 V1 是否已经可以写成明确规格？
2. RSRS 是总开关、确认器，还是只做 veto？
3. ML 的操作频率与持仓数是否已收敛？

### 第 4 波检查点

必须回答：

1. 哪些任务进入开发？
2. 哪些方向继续观察？
3. 哪些方向停止继续投入？
