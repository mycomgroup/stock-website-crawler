# 任务 06：RSRS 复合过滤增量验证

你正在做“RSRS 叠加过滤”的增量验证，不是在重做一套完整指数择时系统。

硬约束：
1. 只能比较以下 4 档：
   - 无过滤
   - RSRS 单独过滤
   - RSRS + 宽度
   - RSRS + 宽度 + 情绪
2. 必须使用任务 02 冻结后的口径。
3. 必须保持同一基础资产或同一基础策略，不允许中途换底座。
4. 不允许把 RSRS 任务扩展成新的复杂组合优化。

技术方案：
- 输入材料：
  - `docs/low_freq_high_winrate_strategy_20260403/results_followup_20260403/result_02_状态输入统一与阈值冻结.md`
  - `docs/low_freq_high_winrate_strategy_20260403/results_followup_20260403/result_05_RSRS改进版本候选筛选.md`
  - `docs/parallel_strategy_tasks_20260328/09_regime_router_v2_final_report_2026-03-31.md`

  - `strategies/quantsplaybook_validation/strategies/05_rsrs_optimized.py`
- 方法：
  1. 选定一个统一底座；
  2. 对比 4 档过滤方案的信号数、收益、回撤、稳定性；
  3. 重点看是否出现“信号显著减少但质量提高”的增量价值；
  4. 若需验证脚本，写到 `docs/low_freq_high_winrate_strategy_20260403/results_followup_20260403/scripts/`。
- 验证标准：
  1. 必须给出 4 档对比；
  2. 必须回答 RSRS 是否真的提供增量价值；
  3. 必须明确最适合的接法是单独过滤还是复合过滤。
- 输出文件：
  - 主结果：`docs/low_freq_high_winrate_strategy_20260403/results_followup_20260403/result_06_RSRS复合过滤增量验证.md`

  - 状态
  - 统一底座
  - 4 档对比结果
  - 主推荐接法
  - 不推荐接法

任务提示词：
请只验证 RSRS 在复合过滤中的增量价值，不要把任务扩展成新的完整择时框架。你需要在同一底座上比较无过滤、RSRS 单独过滤、RSRS+宽度、RSRS+宽度+情绪四档方案，重点回答：RSRS 到底有没有带来额外价值，它最适合单独使用还是与宽度、情绪叠加使用。
