# 任务 08：ML 逻辑回归基线复核与泄露检查

你正在做“ML 基线审计”任务，不是在直接扩展模型复杂度。

硬约束：
1. 先审计现有逻辑回归结果，再决定是否扩因子。
2. 必须检查时间切分、标签定义、因子滞后、股票池稳定性、未来信息泄露风险。
3. 不允许跳过方法审计直接进入新模型。
4. 不修改正式策略代码。

技术方案：
- 输入材料：
  - `docs/ml_multifactor_validation_20260401/README.md`
  - `docs/ml_multifactor_validation_20260401/docs/07_ML_Results_Summary.md`
  - `docs/ml_multifactor_validation_20260401/docs/07_ML_MultiFactor_Validation_Report_REAL.md`

  - `docs/ml_multifactor_validation_20260401/scripts/ml_ultra_quick.py`
  - `docs/ml_multifactor_validation_20260401/scripts/STRATEGY_CODE.md`
- 方法：
  1. 审核样本切分与 walk-forward 逻辑；
  2. 审核因子是否按可获得时点滞后；
  3. 审核股票池是否存在样本穿越问题；
  4. 审核评价指标是否足够支持“低频主仓”定位；
  5. 输出可信度结论与风险等级。
- 验证标准：
  1. 必须给出泄露检查表；
  2. 必须明确当前逻辑回归结论是 `Go / Watch / No-Go`；
  3. 必须指出若继续扩因子，应保留哪些方法学约束。

任务提示词：
请先对现有 ML 逻辑回归主仓结果做一次方法学审计，不要急着加模型复杂度。你需要重点检查时间切分、标签定义、因子滞后、股票池稳定性以及未来信息泄露风险，并明确回答：当前逻辑回归结果是否足够可信到可以继续扩因子。如果有风险，也要说明风险在哪里、严重程度多高、后续该如何约束。
