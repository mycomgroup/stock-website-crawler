## 四、过滤器候选

以下原型**只适合作为过滤器或仓位开关，不建议作为独立主仓**：

### 过滤器 A：大盘PE标准差模块（#53）

| 维度 | 评估 |
|------|------|
| **作为过滤器的价值** | 高 — PE分位数/标准差可作为全市场估值温度计，直接决定仓位的加/减 |
| **与现有系统整合方式** | 将`QuantTiming.std()`输出的`stdpeH/stdpeL`高低估线，作为RFScore7仓位的上限调节器（高PE时降低选股仓位，低PE时提升仓位） |
| **优点** | 逻辑简单（标准差区间），有明确的高/低估判断 |
| **缺点** | 需维护PE历史数据；与PB10%有一定重叠 |

**建议**: 以`stdpeH/stdpeL`作为**仓位调节过滤器**接入，当PE > 高估线时，将RFScore7仓位压缩10-15%；当PE < 低估线时，可超额配置RFScore7。

### 过滤器 B：技术信号风险开关（#18）

| 维度 | 评估 |
|------|------|
| **作为过滤器的价值** | 中 — 32日价量结构判断牛熊，可作为市场风险开关 |
| **与现有系统整合方式** | 当`callsignal == 0`（看空）时，清仓股票类仓位，转入国债ETF/货基 |
| **优点** | 代码极简（20行），信号明确 |
| **缺点** | 信号逻辑基于日频价量，对市场噪音敏感；与现有ETF动量基线功能重叠；实盘信号频率可能过高 |
| **结论** | **暂不推进** — 功能与ETF动量基线重叠，且信号频率偏高（可能每周切换），不符合"低频"约束 |

## 六、转译顺序建议

### 第一阶段（立即可转译，1-2周）

**① 稳健型ETF策略（#85）→ 主仓补充线**
- 理由：月频多资产（股/纳指/红利/黄金/国债）轮动，逻辑清晰，与现有RFScore7+国债固收+形成互补
- 转译步骤：
  1. 提取MACD月线判断函数`get_macd_M`
  2. 改写ETF选择逻辑，去除聚宽依赖
  3. 接入当前调仓框架，月频触发
- 预期效果：降低对单一A股市场的依赖，外盘纳指+黄金提供分散

### 第二阶段（需要数据接入，2-4周）

**② 宏观PMI策略（#48/#21）→ 主仓补充线**
- 理由：提供经济周期视角，填补现有主仓无宏观择时的空白
- 转译步骤：
  1. 确认宏观数据来源（PMI库或替代指标，如财新PMI、CPI/PPI等代理）
  2. 简化模式判断逻辑（mode=0/1/2）
  3. 与国债固收+整合，当mode=0时直接使用现有国债固收+
- 预期效果：基于PMI复苏确认后才进入股票仓位，安全性高

### 第三阶段（作为过滤器接入）

**③ PE标准差过滤器（#53）→ 仓位调节过滤器**
- 理由：作为仓位上限/下限的辅助调节，不独立跑策略
- 转译步骤：
  1. 提取`QuantTiming`类的`std()`方法获取高低估线
  2. 将高估线作为RFScore7仓位的上限信号
  3. 设定阈值（如PE > 高估线时RFScore7上限压缩10%）
