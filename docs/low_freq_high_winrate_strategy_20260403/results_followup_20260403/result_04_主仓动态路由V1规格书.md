## 1. 状态

本规格书基于以下三份前置结论收口：

| 前置任务 | 核心结论 |
|----------|----------|
| 任务 01：主仓底座对齐 | 静态 60/40 仍为最强基线；动态版尚未稳定优于静态版；`rfscore_defensive_dynamic_hedge.py` 是最适合作为动态主仓底座 |
| 任务 02：状态输入冻结 | 宽度、FED、RSRS、情绪四类指标口径已统一，后续不得新增第五个核心状态输入；两套阈值体系保留但明确边界（资产配置用 V2 路由器阈值，短线交易用择时集成阈值） |
| 任务 03：动态映射比较 | 偏均衡版（B 表）最适合作为 V1 候选；动态版预期年化 7.5% 低于静态 60/40 的 10.8%，但回撤显著更小（-5~-7% vs -15~-20%）；定位为风控层而非收益增强层 |

**V1 定位**：不是收益增强层，而是风控层。核心价值在于极端行情下主动降仓，而非在牛市中超越静态 60/40。

## 2. V1 总体目标

| 维度 | 目标 |
|------|------|
| **首要目标** | 在极端弱势/高估场景下，回撤控制优于静态 60/40 |
| **次要目标** | 在正常/偏强场景下，收益不低于静态 60/40 的 70%（接受收益牺牲换取回撤控制） |
| **约束** | 不引入新指标、新资产类别、新策略方向 |
| **交付物** | 一个可独立运行的 Python 策略文件，基于 `rfscore_defensive_dynamic_hedge.py` 底座改造 |
| **运行模式** | 与静态 60/40 并行运行，不替代；满足"转正条件"后才考虑切换 |

## 3. 状态与仓位映射

### 3.1 状态定义（四状态，来自任务 02 冻结口径 + 任务 03 确认）

**状态判定规则（冻结）**：

```
if 宽度 > 70% OR FED < 0:
    → "高估防守"
elif 宽度 > 50% AND RSRS Z-Score > 0.5:
    → "趋势进攻"
elif 宽度 >= 30%:
    → "震荡轮动"
else:
    → "底部试错"
```

| 状态 | 判定条件 | 说明 |
|------|----------|------|
| **底部试错** | 宽度 < 30% | 市场极度低迷，多数股票在 MA20 之下 |
| **震荡轮动** | 宽度 30%–50% | 常态区间，市场无明显方向 |
| **趋势进攻** | 宽度 > 50% 且 RSRS Z-Score > 0.5 | 市场处于上升通道 |
| **高估防守** | 宽度 > 70% 或 FED < 0 | 市场过热或估值偏高 |

**状态判定输入指标（仅 4 个，不得新增）**：

| 指标 | 计算口径 | 样本池 | 更新频率 | 决策频率 | 阈值来源 |
|------|----------|--------|----------|----------|----------|
| 宽度 | 沪深300成分股 `close > MA20` 比例 | 沪深300（排除ST/新股/科创/北交所） | 日频计算 | 周频决策 | Doc 09 V2 |
| FED | 100/PE中位数 − 10年期国债收益率（动态值） | 沪深300 PE中位数 + 10Y国债 | 日频计算 | 周频决策 | Doc 09/11 |
| RSRS Z-Score | 滚动 OLS 斜率标准化（沪深300 high vs low） | 沪深300指数日K | 日频计算 | 周频决策 | Doc 09 |
| 情绪 | 涨停家数（涨幅 >= 9.5%） | 全市场A股（排除ST/科创/北交所/创业板） | 日频计算 | 日频决策 | 择时集成V1.0 |

**辅助阈值（来自任务 02 冻结）**：

| 指标 | 阈值 | 含义 |
|------|------|------|
| RSRS Z-Score | > 0.5 | 趋势向上 |
| RSRS Z-Score | -0.5 ~ 0.5 | 震荡 |
| RSRS Z-Score | < -0.5 | 趋势向下 |
| FED | > 0 | 低估（股票相对债券有吸引力） |
| FED | -1 ~ 0 | 中性 |
| FED | < -1 | 高估 |

### 3.4 股票层选股规则

- **股票池**：沪深300 + 中证500 并集，剔除 ST、*ST、退市、科创板（688）、上市 < 180 日、停牌
- **评分**：RFScore（7 因子）+ PB 分组（前 10%）+ 综合分（质量 50% + 估值 30% + 成长 20%）
- **选股优先级**：
  1. RFScore == 7 且 PB 分组 <= 1
  2. RFScore >= 6 且 PB 分组 <= 2（补位）
- **行业分散**：单一申万一级行业不超过总持股数的 30%
- **持股数**：按状态动态计算，`target_hold_num = int(base_hold_num × stock_weight / max_offensive_weight)`，其中 `base_hold_num = 15`，`max_offensive_weight = 0.40`
  - 底部试错：`int(15 × 0.25 / 0.40) = 9` 只
  - 震荡轮动：`int(15 × 0.35 / 0.40) = 13` 只
  - 趋势进攻：`int(15 × 0.40 / 0.40) = 15` 只
  - 高估防守：`int(15 × 0.15 / 0.40) = 5` 只

## 4. 执行节奏

| 操作 | 频率 | 时间 | 说明 |
|------|------|------|------|
| 状态指标计算 | 日频 | 收盘后 | 宽度、RSRS、情绪每日计算；FED 每日计算但周频决策 |
| 状态判定与转换 | 周频 | 每周五收盘后 | 使用当周最新指标判定状态 |
| 状态转换确认 | 连续 2 周 | — | 新状态需连续 2 周相同才触发切换（滞后机制） |
| 股票层调仓 | 月频 | 每月第 1 个交易日 14:50 | RFScore 选股 + 行业分散 + 涨跌停过滤 |
| 防守层调仓 | 月频 | 每月第 1 个交易日 14:55 | ETF 权重再平衡 |
| 防守层偏离检查 | 日频 | 收盘后 | 任一 ETF 偏离目标权重 > 15% 时触发再平衡 |
| 月度复核 | 月频 | 每月末 | 输出状态日志、仓位日志、与静态 60/40 的累计差异 |
| 回退条件检查 | 周频 | 每周五收盘后 | 检查是否触发回退到静态 60/40 或保守基线 |

### 4.1 状态转换伪代码

```python
# 每周五收盘后
today_state = calc_state(breadth, fed, rsrs_zscore)

# 连续 2 周确认机制
if today_state == last_week_state and today_state != current_regime:
    if regime_hold_weeks >= 1:
        current_regime = today_state
        regime_hold_weeks = 0
        log_regime_change(today_state)
    else:
        regime_hold_weeks += 1
else:
    regime_hold_weeks = 0
```

## 5. 风控约束

### 5.1 最小风控卡

| 风控项 | 上限/下限 | 动作 |
|--------|-----------|------|
| 单只股票仓位 | ≤ 总资金 5% | 超过则等权分配 |
| 单一行业集中度 | ≤ 持股数 30% | 超过则跳过该行业后续股票 |
| 股票层总仓位 | 15%–40% | 由状态映射表决定，不得手动突破 |
| 防守层总仓位 | 固定 30% | 内部权重按 75/10/8/4/3 分配 |
| 现金底线 | ≥ 30.9%（趋势进攻时最低） | 任何状态下不得跌破 |
| 黄金单月跌幅 | > 10% | 黄金权重降至 5%，次月恢复 |
| FED 指标 | < −1 | 纳指 ETF 清仓，权重转国债 |
| 宽度 | < 15% | 强制进入"底部试错"状态（覆盖其他信号） |
| 连续下跌 | 近 10 日中下跌 ≥ 7 日 | 股票仓位减半（临时风控，不改变状态） |
| 组合月度回撤 | > 5% | 触发月度复盘，检查状态映射是否失效 |

### 5.2 涨跌停与执行过滤

- 不买入接近涨停（last_price >= high_limit × 0.98）的股票
- 不买入接近跌停（last_price <= low_limit × 1.02）的股票
- 不买入大幅低开（last_price < yesterday_close × 0.97）的股票
- 不买入大幅高开（last_price > yesterday_close × 1.03）的股票

### 5.3 防守层再平衡规则

- 触发条件：任一 ETF 实际权重偏离目标权重 > 15%（相对偏离）
- 执行：在下一个交易日 14:55 进行再平衡
- 特殊规则：黄金单月跌幅 > 10% 时，权重降至 5%，次月恢复原权重

## 6. 回退条件

### 6.1 回退到静态 60/40 的触发条件

满足以下 **任一** 条件时，策略必须回退到静态 60/40（即 `v3_combo_static_60_40.py` 的行为：RFScore 60% + 红利小盘 40%，月频调仓，无状态路由）：

| 编号 | 条件 | 持续时间 | 回退动作 |
|------|------|----------|----------|
| R1 | 动态路由累计收益落后静态 60/40 超过 3% | 连续 3 个月 | 切换为静态 60/40，停止状态路由 |
| R2 | 状态路由器连续 6 个月处于同一状态 | 6 个月 | 说明状态机失去区分度，回退静态 |
| R3 | 组合月度回撤 > 8%（单次） | 1 次 | 立即回退静态 60/40，次月可重新评估 |
| R4 | 宽度指标数据缺失或连续 5 日无法计算 | 5 个交易日 | 回退静态 60/40，数据恢复后重新评估 |
| R5 | RSRS 计算失败或连续 5 日无法计算 | 5 个交易日 | 回退静态 60/40，修复后重新评估 |

### 6.2 回退到保守基线的触发条件

满足以下 **任一** 条件时，策略切换为保守基线（股票降至 10%，现金提升至 60%，防守层 30%）：

| 编号 | 条件 | 持续时间 | 回退动作 |
|------|------|----------|----------|
| C1 | 宽度 < 10% | 连续 3 日 | 股票降至 10%，现金提至 60% |
| C2 | 情绪涨停数 < 15 只 | 连续 5 日 | 股票降至 10%，现金提至 60% |
| C3 | RSRS Z-Score < −1.5 | 连续 3 日 | 股票降至 10%，现金提至 60% |

### 6.3 回退后的恢复条件

| 回退类型 | 恢复条件 | 恢复动作 |
|----------|----------|----------|
| R1（收益落后） | 动态路由假设重新启用后，连续 2 个月风险调整后收益优于静态 | 恢复动态路由 |
| R2（状态僵化） | 宽度波动恢复至 20%–60% 区间 | 恢复动态路由 |
| R3（大回撤） | 组合净值恢复至回撤前水平的 95% 以上 | 恢复动态路由 |
| R4/R5（数据/计算失败） | 数据源恢复且连续 5 日正常 | 恢复动态路由 |
| C1–C3（保守基线） | 宽度回升至 20% 以上 且 情绪回升至 30 只以上 且 RSRS Z-Score > -1.0 | 恢复正常状态判定 |

### 6.4 回退执行伪代码

```python
def check_fallback(context):
    """每周五收盘后检查回退条件"""
    
    # R1: 累计收益落后静态 60/40 超过 3%，连续 3 个月
    if dynamic_underperform_static_3m > 0.03:
        fallback_to_static_6040()
        return
    
    # R2: 状态僵化
    if same_regime_months >= 6:
        fallback_to_static_6040()
        return
    
    # R3: 单次月度回撤 > 8%
    if monthly_drawdown > 0.08:
        fallback_to_static_6040()
        return
    
    # C1: 极端弱势
    if breadth < 0.10 for 3_consecutive_days:
        enter_conservative_baseline()
        return
    
    # C2: 情绪冰点
    if limit_up_count < 15 for 5_consecutive_days:
        enter_conservative_baseline()
        return
    
    # C3: 趋势极度向下
    if rsrs_zscore < -1.5 for 3_consecutive_days:
        enter_conservative_baseline()
        return
    
    # R4/R5: 数据缺失
    if breadth_data_missing_days >= 5 or rsrs_calc_failed_days >= 5:
        fallback_to_static_6040()
        return
```
