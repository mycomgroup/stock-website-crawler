## 二、统一底座

### 2.1 底座选择

**RFScore7 PB10 Optimized**（`strategies/enhanced_v2/rfscore7_pb10_optimized.py`）

选择理由：
- 已在总纲（00_总纲）中明确为"主仓低频化"的首选底座之一
- 月频调仓、强过滤（RFScore=7 + PB 最低 10%）、行业分散
- 自带基础广度过滤（breadth_stop=0.15, breadth_reduce=0.25, breadth_normal=0.35）
- 与 RSRS 叠加时，RSRS 可作为**额外 veto 层**而非替代原有逻辑

### 2.2 底座原始行为

| 维度 | 值 |
|------|-----|
| 调仓频率 | 月频（每月第 1 个交易日 9:35） |
| 选股池 | HS300 + ZZ500，排除科创板、ST、上市不满 180 天 |
| 主选条件 | RFScore=7 且 PB 分组=1（最低 10%） |
| 备选条件 | RFScore≥6 且 PB 分组≤3 |
| 目标持仓 | 15 只（正常）/ 12 只（中性）/ 10 只（弱势）/ 0 只（停手） |
| 停手条件 | 广度 < 15% |
| 减仓条件 | 广度 < 25% 或（广度 < 35% 且趋势向下） |
| 止损 | 单票 -15% |

### 2.3 叠加方式

RSRS 作为**独立 veto 层**挂接在底座选股之前：
- 若 RSRS 右偏标准分 < -0.8（卖出信号），则该月不执行任何买入
- 若 RSRS 右偏标准分 ≥ -0.8，则正常执行底座逻辑
- 宽度、情绪作为**附加 veto**，在 RSRS 通过的基础上叠加

## 四、主推荐接法

### 4.1 推荐：RSRS 单独过滤（Tier 1）

**理由**：

1. **增量价值清晰且精准**
   - 3 年中精准拦截 2 次极端破位（2024-02 微盘股崩盘、2025-05 趋势破位）
   - 信号率 94.4%，仅减少 5.6% 的交易机会
   - 平均持仓从 5.3 降至 4.9，影响有限

2. **复杂度最低**
   - 只需挂接一个 veto 层，不改变底座原有逻辑
   - 参数来源光大研报（N=18, M=300, 阈值 ±0.8），非过拟合
   - 维护成本低，每月更新一次 RSRS 值即可

3. **与底座互补**
   - 底座已有广度过滤（breadth_stop=0.15），与 RSRS 维度不同
   - 底座关注"选股质量"，RSRS 关注"大盘趋势结构"
   - 两者叠加是**正交增强**而非冗余叠加

4. **真实拦截案例验证**
   - 2024-02：RSRS=-0.83，成功拦截春节前大跌
   - 2025-05：RSRS=-0.84，成功拦截趋势破位

### 4.2 具体挂接方案

```python
# 在 rebalance() 函数开头增加 RSRS veto
def rebalance(context):
    watch_date = context.previous_date

    # RSRS veto（新增）
    rsrs_allowed = check_rsrs_filter(watch_date)
    if not rsrs_allowed:
        # RSRS 发出卖出信号，清仓或保持空仓
        log.info("RSRS veto: 右偏标准分 < -0.8，本月不交易")
        clear_all_positions(context)
        return

    # 原有底座逻辑继续执行
    market_state = calc_market_state(watch_date)
    ...
```

## 五、不推荐接法

### 5.1 强烈不推荐：RSRS + 宽度 + 情绪三重过滤（Tier 3）

**理由**：

1. **几乎完全停手**
   - 36 个月中仅 1 个月允许交易（2024-10，情绪=46）
   - 信号率仅 2.8%，等于放弃整个策略
   - 平均持仓 0.2，零持仓月 35 个

2. **情绪阈值与月频策略不匹配**
   - 情绪阈值 ≥30 是为日频/周频短线策略设计的
   - 月频调仓下，绝大多数月份收盘时涨停数 < 30
   - 36 个月中仅 1 个月情绪 ≥30，说明该阈值对月频策略完全不适配

3. **与底座逻辑冲突**
   - 底座本身就是"低频高胜率"选股策略
   - 三重 veto 等于把底座变成"几乎不交易"

### 5.2 不推荐：RSRS + 宽度双重过滤（Tier 2）作为首选

**理由**：
- 相比 Tier 1 仅额外拦截 1 个月（2025-01）
- 增量价值有限（-2.7% 信号率）
- 但底座已有 breadth_stop=0.15 停手，与 Tier 2 的宽度 veto 高度重叠
- 如需要可保留，但不作为首选推荐

### 5.3 不推荐：用 RSRS 替代底座原有广度过滤

**理由**：
- 底座广度过滤已验证有效（breadth_stop=0.15 时停手）
- RSRS 与广度维度不同：RSRS 看趋势结构，广度看横截面强度
- 两者应**叠加**而非**替代**

## 六、关键发现与结论

### 6.1 RSRS 是否真的提供增量价值？

**是，增量价值有限但精准。**

| 场景 | RSRS 增量价值 | 说明 |
|------|-------------|------|
| **极端破位（大跌）** | ★★★★★ | 3 年拦截 2 次（2024-02, 2025-05），均为真破位 |
| **震荡市** | ★★★☆☆ | RSRS 在震荡区间内波动，不产生误拦截 |
| **趋势向上** | ★☆☆☆☆ | RSRS 持续为正，不产生额外过滤 |
| **反弹初期** | ★★☆☆☆ | RSRS 滞后，但月频策略本身不追求抄底 |

**核心结论**：
- RSRS 的增量价值主要体现在**极端破位保护**，3 年拦截 2 次，均为有效拦截
- 信号率 94.4%，仅减少 5.6% 的交易机会，过滤效率高
- 在底座已有广度过滤的基础上，RSRS 提供的是**正交补充**而非冗余
- 叠加宽度后增量有限（仅多拦截 1 个月），叠加情绪后完全停手

### 6.2 RSRS 最适合的接法

**RSRS 单独过滤（Tier 1）** 是最优接法：
- 作为独立 veto 层挂接在底座选股之前
- 保留底座原有的广度过滤（breadth_stop=0.15）
- 不引入情绪过滤（月频策略与日频情绪不匹配）

### 6.3 RSRS 的定位

| 维度 | 定位 |
|------|------|
| 角色 | **风控 veto 层**，非 Alpha 增强层 |
| 频率 | 月频（与底座调仓频率一致） |
| 优先级 | 在底座广度过滤之上 |
| 失效场景 | 震荡市信号频繁翻转（但 3 年未出现误拦截） |
| 拦截效率 | 3 年 2 次拦截，100% 有效 |

### 6.4 情绪阈值的适配问题

本次验证暴露出一个重要问题：

**V1.0 文档中的情绪阈值 ≥30 是为日频/周频短线策略设计的，不适用于月频选股策略。**

| 策略类型 | 推荐情绪阈值 | 说明 |
|---------|------------|------|
| 日频/周频短线（二板接力） | ≥30 只 | 原 V1.0 推荐 |
| 月频选股（RFScore） | 不建议使用 | 月频下几乎永远不达标 |
| 若必须使用 | ≥15 只 或改用其他情绪指标 | 需重新验证 |
