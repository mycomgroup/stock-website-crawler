# 后续探索优先级总表

日期：2026-04-03

## 1. 目的

基于当前仓库已经完成的研究结论，明确下一阶段不应再做“从零开题”，而应围绕：

- 主仓低频化与动态路由；
- 通用择时过滤器；
- 低频多因子主仓；
- RFScore 增强因子；
- ETF / 宏观慢变量补充线；

开展更小颗粒、可验证、可并行的后续探索。

## 2. 优先级总览

| 路线 | 优先级 | 预期价值 | 实现难度 | 当前基础 | 推荐动作 |
|------|--------|----------|----------|----------|----------|
| 主仓动态路由 | P0 | 5/5 | 4/5 | 已有 `RFScore`、防守底仓、静态 `60/40`、四状态路由器 | 先做口径统一，再做轻量映射比较，最后收敛成 V1 规格书 |
| RSRS 复合择时过滤 | P1 | 4/5 | 3/5 | `QuantsPlaybook` 验证中 RSRS 是少数正结果方向 | 先缩小 RSRS 候选，再验证与宽度/情绪的叠加价值 |
| ML 低频多因子主仓 | P1 | 4/5 | 4/5 | 逻辑回归已有真实验证结果 | 先做方法复核，再扩因子，再比较月频/季频 |
| RFScore 增强因子 | P1.5 | 4/5 | 4/5 | RFScore 底座明确，FFScore / 筹码 / STR / 行业量价有资料来源 | 先做 FFScore 增量验证，再做其他候选预筛 |
| 低频 ETF / 宏观慢变量 | P2 | 3/5 | 2/5 | 聚宽原型很多，但大多尚未系统映射到当前主仓框架 | 先做原型映射和白名单，再决定是否进入开发 |

## 3. 为什么这样排序

### 3.1 主仓动态路由排第一

原因不是它“看起来最先进”，而是它最接近主账户资金管理问题：

- 现有 `RFScore7 + PB10%`、国债、防守底仓都已经形成稳定材料；
- 静态 `60/40` 已证明可行，但动态版还没证明优于静态版；
- 这说明当前最大缺口不是资产不够，而是“状态判断 -> 仓位映射”这一层还没有收敛。

因此这条线一旦做成，能直接影响主仓配置、风控上限和资金调度。

### 3.2 RSRS 复合过滤排第二

当前 `QuantsPlaybook` 一批择时策略里，真正经 10 年验证后仍有正结果的主要是 RSRS。

这意味着：

- 不值得继续大面积撒网验证更多单一技术择时指标；
- 更值得把 RSRS 当作“总开关 / 二次确认器 / 风控过滤器”来用；
- 最合理的下一步不是重写 RSRS，而是验证它与宽度、情绪结合后的增量价值。

### 3.3 ML 低频多因子排第三

逻辑回归已经跑出较好的低频结果，但：

- 当前样本期还不够长；
- 因子数偏少；
- 调仓频率和持仓数仍未定型。

这条线有潜力成为“第二主仓候选”，但需要先保证方法可靠，再谈扩展。

### 3.4 RFScore 增强因子排第四

RFScore 目前最大优点是：

- 已经能跑；
- 已有候选池和补位逻辑；
- 适合作为所有增强实验的基线。

因此最好的做法不是重做 RFScore，而是把 FFScore、筹码、STR、行业量价这类方向当成“外挂候选”，逐个验证是否真能带来低相关增益。

### 3.5 低频 ETF / 宏观慢变量排第五

这条线不是不重要，而是更像“原型库”和“备用主仓补充线”：

- 聚宽材料多；
- 实盘叙事好；
- 但目前与你现有主仓体系的接口定义最弱。

所以应该先做映射与白名单，不要直接跳到大量回测。

## 4. 推荐的执行顺序

### 第一阶段：先统一，不急着测

优先完成：

1. 主仓底座现状对齐；
2. 状态输入与阈值口径冻结；
3. RSRS 改进版本候选筛选；
4. ML 基线复核；
5. ETF / 宏观原型映射。

这一阶段的目标是先把“可比对象”讲清楚，避免后面不同 agent 各自用不同口径。

### 第二阶段：只做最小增量验证

优先完成：

1. 主仓动态映射候选表轻量比较；
2. RSRS 复合过滤增量验证；
3. ML 月频 / 季频 / 持仓数组合比较；
4. FFScore 对 RFScore 的增量验证。

这一阶段只回答“值不值得继续”，不追求一次性做成最终线上版。

### 第三阶段：统一收口

优先完成：

1. 主仓动态路由 V1 规格书；
2. RSRS 模块挂接规则；
3. RFScore 增强因子白名单；
4. 下一轮开发顺序与 agent 排班表。

## 5. 暂不优先的方向

以下方向目前不建议进入下一轮主任务：

- 再开一轮主线假弱高开定义研究；
- 再重做二板基础规则；
- 已被删除的深度低开分支；
- 大范围验证更多单一技术择时指标；
- 没有明确数据可得性的“概念型因子堆叠”。

## 6. 推荐的总原则

1. 先统一口径，再做轻量验证。
2. 先比较少数候选，再决定是否进入大回测。
3. 先做模块接口，再谈工程化集成。
4. 能做白名单 / 淘汰清单的，不要直接做“大而全组合”。
