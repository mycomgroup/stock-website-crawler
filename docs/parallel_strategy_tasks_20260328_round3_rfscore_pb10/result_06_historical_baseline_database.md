### 4.3 关键偏离分析

#### 候选数量严重偏高（4只 vs 基准1.5只）

**原因分析**:
1. 当前市场可能处于估值低点（PB压缩）
2. 近期财务数据改善（ROA、现金流等因子得分率提升）
3. 策略筛选条件相对宽松的时期

**风险提示**:
- 候选数量偏多时，需检查筛选条件是否过于宽松
- 关注是否包含异常股（PE>50、ROA<0.5%）
- 当前已检测无异常股 ✓

#### ROA均值偏低（1.26% vs 基准2.5%）

**原因分析**:
1. 当前候选可能处于盈利低谷期
2. 周期行业（钢铁）盈利能力下降
3. 筛选侧重低估值，盈利质量权重不足

**风险提示**:
- 低ROA可能影响策略收益
- 需关注ROA改善趋势（DELTA_ROA）
- 建议增加ROA>2%硬性筛选条件

## 五、基准应用场景

### 5.1 实时监控

```python
# 实时监控逻辑示例
def check_deviation(current_value, baseline_mean, baseline_std):
    z_score = (current_value - baseline_mean) / baseline_std
    
    if abs(z_score) > 2:
        return "严重偏离"
    elif abs(z_score) > 1.5:
        return "明显偏离"
    elif abs(z_score) > 1:
        return "轻微偏离"
    else:
        return "正常范围"
```

### 5.2 质量评分优化

基于历史基准优化质量评分：

```python
# 候选数量评分（基于分位数）
if count >= baseline_p90:  # >=3只
    score += 10  # 充足
elif count >= baseline_p75:  # >=2只
    score += 5   # 较充足
elif count < baseline_p25:  # <0只（实际为0）
    score -= 10  # 不足
    
# ROA均值评分（基于偏离度）
roa_ratio = roa_mean / baseline_roa_mean
if roa_ratio < 0.5:
    score -= 20  # 严重偏低
elif roa_ratio < 0.8:
    score -= 10  # 偏低
elif roa_ratio > 1.2:
    score += 5   # 偏高（好）
```
