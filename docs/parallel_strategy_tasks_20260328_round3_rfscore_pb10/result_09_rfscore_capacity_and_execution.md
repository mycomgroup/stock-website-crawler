## 六、执行要点与风险控制

### 6.1 执行要点

1. **仓位控制**: 
   - 单只股票仓位不超过其日均成交额的 **5%**
   - 总仓位不超过策略容量的 **80%**

2. **流动性监控**:
   - 每月检查候选股日均成交额
   - 如出现显著萎缩（<3000万），及时调整持仓

3. **动态调整**:
   - 市场宽度 > 0.4: 满仓35-40%
   - 市场宽度 0.25-0.4: 标准25-35%

   - 市场宽度 < 0.15: 空仓或极轻仓<15%

4. **成本优化**:
   - 优先使用低佣金渠道
   - 避免在市场波动剧烈时大额调仓
   - 考虑拆单执行，降低市场冲击

### 6.2 风险控制矩阵

### A. 容量计算详细过程

```python
# 容量计算示例（假设数据）
hold_num = 20                                    # 持仓数量
impact_limit = 0.05                              # 冲击限制5%
min_avg_amount = 3e8                             # 最小日均成交额3亿（假设）

# 理论最大容量
max_capacity = min_avg_amount * impact_limit * hold_num
             = 3e8 * 0.05 * 20
             = 3e9（3亿）

# 考虑调仓频率和保留现金
practical_capacity = max_capacity * 0.8 / 0.15   # 预留20%现金，考虑15%调仓比例
                   ≈ 16亿

# 安全边际（50%）
suggested_capacity = practical_capacity * 0.5
                   ≈ 8-10亿（单策略）
```
