## 二、核心改进点

### 2.1 多维度仓位决策

**原策略**: 仅基于宽度判断（单一维度）
```
宽度<25% → 减仓
宽度<15% → 空仓
```

**改进版**: 5维度综合决策
```
基础仓位（状态路由）
  × RSRS系数（择时确认）
  × 波动率系数（风险控制）
  × 底部保护（极端情况）
  = 最终仓位
```

### 2.2 底部保护机制

**关键改进**: 即使状态路由看空，如果底部信号强烈（≥4/5项满足），最少保持30%仓位

**底部信号**（简化5项）：
1. 市场宽度 < 15%
2. FED估值 > 2%
3. 破净占比 > 15%
4. 成交额 < 5000亿
5. 涨停家数 < 20家

**效果**: 防止在2018年底、2022年底等大底完全空仓

### 2.3 个股风控机制

**新增**: 每日14:50检查移动止盈止损
- 止损线: -10%
- 移动止盈: 从高点回落15%

**效果**: 截断个股亏损，让利润奔跑

### 2.4 换手率过滤

**已验证**（任务04结果）：
- Turnover过滤实证有效，年化+4.67%
- CGO过滤负面，年化-1.35%（已排除）

**应用**: 选股后保留换手率/波动率最低的前80%

| 维度 | 原策略 | 改进版v2.0 |

| **换手率过滤** | ❌ 无 | ✅ 已整合 |
| **个股风控** | ❌ 无 | ✅ 移动止盈止损 |
| **空仓设计** | 15%阈值空仓 | 最少5只，永不空仓 |

### 3.2 决策逻辑对比

**场景**: 宽度23%（底部区域）

**原策略**:
```
宽度23% < 25% → 减仓至10只
宽度23% > 15% → 不全仓
结果: 10只，50%仓位
```

**改进版**:
```
基础仓位 = 50% (底部防守状态)
RSRS = -0.5 (中性偏空) → ×0.7 = 35%
ATR = 6% (高波动) → ×0.5 = 17.5%
底部信号 = 4/5满足 → max(17.5%, 30%) = 30%
结果: 6只（最少5只），30%仓位
```

**差异**: 
- 改进版在RSRS看空+高波动时更保守（17.5% vs 50%）
- 但底部保护机制确保最少30%，不完全空仓

## 七、风险提示

### 7.1 新机制风险

1. **RSRS择时滞后**: 可能与基本面信号冲突
   - 缓解: 作为调整系数(0.7-1.1)，不单独使用

2. **波动率仓位频繁**: 高波动期可能频繁减仓
   - 缓解: 月度调仓时计算，不日内调整

3. **底部保护逆势**: 可能提前加仓被套
   - 缓解: 最少5只限制，不满仓，保留现金

4. **移动止盈过早**: 可能错过大牛股
   - 缓解: 15%回撤阈值，给足利润空间

### 7.2 实施风险

- 代码复杂度增加，需要充分测试
- 多机制叠加可能导致过度拟合
- 需要更长的回测周期验证效果
