## 四、RFScore专属仓位规则

### 4.1 规则设计理念

与ETF择时策略不同，RFScore作为**基本面+低估值**策略，其仓位控制应遵循以下原则:

1. **底部不空仓**: 低估值股票在熊市中具有相对优势，完全空仓会错失布局机会
2. **渐进减仓**: 从20只逐步降至10只，而非一刀切换，降低调仓冲击
3. **质量优先**: 减仓时收紧PB分组(PB10%→PB5%)，优中选优
4. **频率适配**: 底部区域降低调仓频率(月度→双月)，减少摩擦成本

### 4.2 RFScore专属四档仓位规则

```python
# RFScore PB10 专属仓位规则
# 规则名称: RFScore-Adaptive

g.base_hold_num = 20          # 正常状态: 20只，PB10%
g.defensive_hold_num = 15     # 防守状态: 15只，PB10%  
g.bottom_hold_num = 10        # 底部状态: 10只，PB5%(优中选优)
g.min_hold_num = 5            # 极端状态: 5只，PB5%

g.breadth_defensive = 0.40    # 宽度<40%进入防守状态
g.breadth_bottom = 0.25       # 宽度<25%进入底部状态
g.breadth_extreme = 0.15      # 宽度<15%进入极端状态

g.primary_pb_group = 1        # PB10% (正常/防守)g.bottom_pb_group = 0.5      # PB5% (底部/极端，代码中通过排序取前50%实现)
```

### 4.5 规则代码实现

```python
def calc_position_size(market_breadth, trend_on):
    """
    RFScore专属仓位计算
    返回: (target_hold_num, pb_group, rebalance_frequency)
    """
    if market_breadth < 0.15:
        # 极端状态: 5只，PB5%，双月
        return 5, 0.5, "bimonthly"
    elif market_breadth < 0.25:
        # 底部状态: 10只，PB5%，双月
        return 10, 0.5, "bimonthly"
    elif market_breadth < 0.40 and not trend_on:
        # 防守状态: 15只，PB10%，月度
        return 15, 1, "monthly"
    else:
        # 正常状态: 20只，PB10%，月度
        return 20, 1, "monthly"
```

## 七、结论与行动项

### 7.1 核心结论

1. **RFScore不应完全空仓**: 基本面+低估值策略在底部有相对优势，保持5-10只精选股票优于完全空仓
2. **渐进减仓更优**: 20→15→10→5的渐进式减仓，比20→10→0一刀切换更平滑
3. **收紧PB分组**: 底部时将PB10%收紧至PB5%，优中选优提升质量
4. **当前建议满仓**: 宽度38.5%处于正常区间，维持20只PB10%配置
