## 四、超时原因分析

### 4.1 可能原因

1. **RSRS指标初始化**: 需要600天历史数据初始化，可能导致首次运行缓慢
2. **底部信号计算**: 5项信号需要多次API调用
3. **每日止盈止损检查**: run_daily函数每天执行，增加计算量
4. **因子计算**: RFScore需要5期财务数据，计算复杂

### 4.2 优化建议

1. **简化RSRS初始化**: 首次运行时使用简化版，后续再完整计算
2. **减少底部信号频率**: 从每日改为每周或每月计算
3. **优化止盈止损**: 改为每周检查，而非每日
4. **缓存因子数据**: 避免重复计算
