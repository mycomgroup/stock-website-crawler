# RFScore PB10 改进版策略 v2.0 设计文档

**生成时间**: 2026-04-02  
**设计目标**: 整合通用机制库，提升策略稳健性和收益风险比  
**基准策略**: `rfscore7_pb10_final.py`

---

## 一、改进概述

### 1.1 改进动机

原RFScore PB10策略存在以下问题：
- 单一宽度判断，缺乏多维确认
- 没有个股层面的风控机制
- 极端底部时可能过度减仓
- 缺乏波动率适应性

### 1.2 改进目标

通过整合通用机制库，实现：
1. **多维度状态判断** - 不只看宽度，还看RSRS、波动率、底部信号
2. **动态仓位调整** - 根据多重信号动态调整仓位
3. **个股风控** - 添加移动止盈止损机制
4. **换手率过滤** - 任务04验证有效，纳入策略
5. **底部保护** - 极端底部时不空仓，保持一定敞口

### 1.3 改进效果预期

| 指标 | 原策略 | 改进版目标 | 改善 |
|------|--------|-----------|------|
| 最大回撤 | -25% | -20% | 降低20% |
| 夏普比率 | 0.9 | 1.1 | 提升22% |
| 胜率 | 55% | 60% | 提升9% |
| 空仓天数 | 15% | 5% | 减少66% |

## 二、通用机制整合详解

### 2.1 状态路由器（通用机制03）⭐

**原策略**: 仅基于宽度判断（<25%减仓，<15%空仓）

**改进版**: 5档状态 + 情绪确认

```python
状态档位（通用机制03标准）：
├── 极弱停手：宽度<15% 或 涨停<20家 → 0%仓位
├── 底部防守：宽度15-25% → 50%仓位（10只）
├── 震荡平衡：宽度25-35% → 75%仓位（15只）
├── 趋势正常：宽度≥35% → 75%仓位（15只）
└── 强趋势：宽度≥40% 且 涨停≥80家 → 100%仓位（20只）
```

**改进点**:
- 增加情绪指标（涨停家数）作为确认
- 底部区间（15-25%）保持50%仓位，不完全空仓
- 强趋势状态额外奖励仓位

### 2.2 RSRS择时（通用机制08）⭐

**新增机制**: 作为额外择时确认

```python
RSRS信号应用：
├── RSRS > 0.7（看多）→ 仓位 × 1.1（最多100%）
├── RSRS < -0.7（看空）→ 仓位 × 0.7
└── -0.7 ≤ RSRS ≤ 0.7（中性）→ 仓位不变
```

**原理**: 
- RSRS利用高低价线性回归斜率判断支撑阻力
- 右偏修正版（zscore × beta × R²）在A股表现优异
- 与基本面策略结合可过滤假突破

### 2.3 波动率仓位（通用机制10）⭐

**新增机制**: ATR-based动态仓位

```python
波动率调整：
├── ATR/价格 < 2%（低波动）→ 仓位 × 1.2
├── ATR/价格 > 5%（高波动）→ 仓位 × 0.5
└── 2% ≤ ATR/价格 ≤ 5%（正常）→ 仓位 × 1.0
```

**原理**:
- 波动率高时减仓，降低风险暴露
- 波动率低时加仓，捕捉稳定收益
- 桥水全天候策略的核心机制

### 2.4 底部9项信号（通用机制23）⭐

**新增机制**: 极端底部保护

```python
底部信号（简化5项核心）：
├── 市场宽度 < 15%
├── FED估值 > 2%
├── 破净占比 > 15%
├── 成交额 < 5000亿
└── 涨停家数 < 20家

底部保护逻辑：
├── 满足≥4项 → 即使状态路由看空，最少保持30%仓位
├── 满足2-3项 → 正常跟随状态路由
└── 满足0-1项 → 正常跟随状态路由
```

**原理**:
- 历史上真正大底（2018年底、2022年底）通常满足5-8项
- 防止在极端底部完全空仓，错失反弹
- 与FED估值、RSRS形成多重确认

### 2.5 换手率过滤（通用机制19 + 任务04验证）

**已验证有效**（任务04结果）：
- Turnover过滤实证有效，年化+4.67%
- CGO过滤负面，年化-1.35%（已排除）

**应用方式**:
```python
# 选股后应用换手率过滤
stocks = choose_stocks(...)
stocks = turnover_filter.filter_stocks(stocks, keep_ratio=0.8)
```

**原理**:
- 剔除换手率/波动率过高的投机股
- 保留"低换手+低波动"的稳健标的

### 2.6 移动止盈止损（通用机制17）⭐

**新增机制**: 个股层面风控

```python
风控参数：
├── 止损线：-10%（固定）
├── 移动止盈：从高点回落15%
└── 每日14:50检查

触发条件：
├── 亏损≥10% → 止损平仓
├── 盈利后从高点回落≥15% → 止盈平仓
└── 记录最高价用于计算回撤
```

**原理**:
- 截断亏损，让利润奔跑
- 避免个股黑天鹅造成大额损失
- 移动止盈可捕捉趋势利润

## 四、仓位决策逻辑

### 4.1 决策流程

```python
def calculate_position(watch_date):
    # 步骤1: 状态路由基础仓位
    base_ratio, hold_num, state = state_router.route(watch_date)
    
    # 步骤2: RSRS择时调整
    rsrs_signal, rsrs_value = rsrs.get_signal(watch_date)
    if rsrs_signal == 'BEARISH':
        base_ratio *= 0.7
    elif rsrs_signal == 'BULLISH':
        base_ratio = min(base_ratio * 1.1, 1.0)
    
    # 步骤3: 波动率调整
    vol_multiplier, _ = vol_mgr.get_volatility_multiplier('000300.XSHG', watch_date)
    final_ratio = base_ratio * vol_multiplier
    
    # 步骤4: 底部保护
    bottom_satisfied, _ = bottom_checker.check_all(watch_date)
    if bottom_satisfied >= 4 and final_ratio < 0.3:
        final_ratio = max(final_ratio, 0.3)  # 最少30%
    
    # 步骤5: 计算持仓数
    target_hold_num = int(20 * final_ratio)
    target_hold_num = max(target_hold_num, 5) if final_ratio > 0 else 0
    
    return final_ratio, target_hold_num
```

### 4.2 决策示例

**场景1: 正常市场**
```
宽度: 45%, 涨停: 85家
RSRS: 0.85 (看多)
ATR比率: 2.5% (正常)
底部信号: 1/5满足

决策: 
  基础仓位 = 100% (强趋势)
  RSRS调整 = 100% × 1.1 = 100%
  波动率调整 = 100% × 1.0 = 100%
  底部保护 = 不适用
  最终仓位 = 100% (20只)
```

**场景2: 震荡市场**
```
宽度: 32%, 涨停: 60家
RSRS: 0.2 (中性)
ATR比率: 4.0% (正常)
底部信号: 2/5满足

决策:
  基础仓位 = 75% (震荡平衡)
  RSRS调整 = 75% × 1.0 = 75%
  波动率调整 = 75% × 1.0 = 75%
  底部保护 = 不适用
  最终仓位 = 75% (15只)
```

**场景3: 底部区域**
```
宽度: 20%, 涨停: 15家
RSRS: -0.5 (中性偏空)
ATR比率: 6.0% (高波动)
底部信号: 4/5满足 (宽度<15%, FED>2%, 破净>15%, 涨停<20)

决策:
  基础仓位 = 50% (底部防守)
  RSRS调整 = 50% × 0.7 = 35%
  波动率调整 = 35% × 0.5 = 17.5%
  底部保护 = max(17.5%, 30%) = 30% (触发保护)
  最终仓位 = 30% (6只，但最少5只)
```

## 七、风险控制

### 7.1 新机制风险

| 机制 | 潜在风险 | 缓解措施 |
|------|---------|---------|
| RSRS择时 | 信号滞后 | 与宽度判断结合，不单独使用 |
| 波动率仓位 | 频繁调仓 | 月度调仓时计算，不日内调整 |

| 参数 | 默认值 | 建议范围 | 敏感性 |

| RSRS买入阈值 | 0.7 | 0.5-1.0 | 中 |
| RSRS卖出阈值 | -0.7 | -0.5至-1.0 | 中 |
