## 四、机制分析与推荐

### 4.1 连亏3停3天机制

**实施效果**：
- ✓ 回撤改善28.4% - 显著降低尾部风险
- ✓ 收益提升20.7% - 停手恰好避开后续亏损
- ✓ 卡玛提升68.6% - 风险调整收益最优
- ✓ 胜率提升1.5% - 筛选掉低胜率环境
- ✓ 触发15次，平均每次停手3天

**优势**：
1. **规则简单**：仅需记录连亏笔数，盘中可执行
2. **触发频率适中**：15次触发，年均约12次
3. **收益提升而非损失**：停手恰好避开极端回撤区间
4. **回撤控制有效**：从35.87%降至25.67%

**适用场景**：
- 大多数机会仓策略
- 需要控制尾部风险的策略
- 希望保持较高收益的策略

**实施建议**：
```
停手规则：
  1. 连亏3笔触发停手
  2. 停手期间为3个交易日
  3. 停手期间不执行任何交易信号
  4. 第4天恢复正常交易
  
盘中执行：
  1. 每笔交易收盘后记录盈亏
  2. 连亏计数器：每亏损+1，每盈利清零
  3. 达到3笔立即启动停手计时器
  4. 停手期间忽略所有信号
```

### 4.2 连亏4停5天机制

**实施效果**：
- ✓ 回撤改善27.3% - 控制效果良好
- ✓ 收益损失9.2% - 损失较小
- ✓ 卡玛提升24.8% - 风险调整收益改善
- ✓ 触发9次，停手更激进

**优势**：
1. **触发条件更严格**：需连亏4笔才停手
2. **停手时间更长**：5天停手，更充分休息
3. **适合风险厌恶型**：更保守的风控

**劣势**：
1. **收益损失大于连亏3版**：9.2%损失
2. **停手时间更长**：错过更多机会
3. **触发次数少**：可能错过连亏3的最佳停手点

**适用场景**：
- 大资金（>500万）
- 风险厌恶型投资者
- 年化目标<50%
- 最大回撤容忍<15%

### 4.3 近10笔半仓机制

**实施效果**：
- ✓ 回撤改善27.7% - 控制效果良好
- ✓ 收益损失16.8% - 损失适中
- ✓ 无休息天数 - 保持市场参与
- ✓ 触发33次 - 频繁调整仓位

**优势**：
1. **不完全停手**：保持市场参与
2. **动态调整**：根据近10笔收益自动切换
3. **避免极端错过**：半仓运行而非全停

**劣势**：
1. **收益损失较大**：16.8%
2. **触发频繁**：33次，可能过于敏感
3. **需要持续监控**：每天计算近10笔收益

**适用场景**：
- 情绪波动期或震荡市
- 不愿完全停手的投资者
- 作为连亏停手的辅助机制

## 五、组合使用建议

### 5.1 推荐组合方案

**主方案：连亏3停3天**

**辅助方案：可叠加近10笔半仓机制**

**组合效果**：
- 连亏3停3天作为第一层风控
- 近10笔半仓作为第二层风控（在停手恢复后启用）
- 双层风控进一步降低极端回撤风险

### 5.2 组合实施逻辑

```python
组合风控规则：
  1. 连亏计数器：记录连续亏损笔数
  2. 近10笔收益计数器：记录最近10笔累计收益
  
  第一层（连亏停手）：
    - 连亏3笔 → 停手3天
    - 停手期间忽略所有信号
    
  第二层（半仓调整）：
    - 非停手期间，近10笔收益转负 → 半仓运行
    - 近10笔收益转正 → 恢复正常仓位
    
  执行顺序：
    - 先检查连亏停手
    - 如未停手，再检查半仓调整
```

## 九、停手机制代码

```python
class PauseManager:
    """
    停手机制管理 - 连亏3笔停3天
    """
    def __init__(self):
        self.loss_count = 0  # 连续亏损计数
        self.pause_days = 0  # 剩余停手天数
        self.trade_history = []  # 交易历史
        self.trigger_count = 0  # 触发次数
    
    def record_trade(self, pnl_pct):
        """
        记录交易结果
        pnl_pct: 盈亏百分比（正数为盈利，负数为亏损）
        """
        self.trade_history.append(pnl_pct)
        
        # 更新连亏计数
        if pnl_pct < 0:
            self.loss_count += 1
        else:
            self.loss_count = 0
        
        # 触发停手
        if self.loss_count >= 3 and self.pause_days == 0:
            self.pause_days = 3
            self.trigger_count += 1
            print(f"触发停手：连亏{self.loss_count}笔，停手{self.pause_days}天")
    
    def can_trade(self):
        """
        判断是否可以交易
        """
        return self.pause_days == 0
    
    def daily_update(self):
        """
        每日更新（减少停手天数）
        """
        if self.pause_days > 0:
            self.pause_days -= 1
            if self.pause_days == 0:
                print("停手结束，恢复交易")
    
    def get_status(self):
        """
        获取当前状态
        """
        return {
            "loss_count": self.loss_count,
            "pause_days": self.pause_days,
            "can_trade": self.can_trade(),
            "trigger_count": self.trigger_count
        }
```
