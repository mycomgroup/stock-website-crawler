## 一、本轮实验设计

### 1.1 核心问题

给`01首板低开`策略配一个最简单、最稳定、最可执行的情绪开关，回答：

1. **哪套指标组合最有效？**
   - 涨停家数单指标
   - 最高连板 + 涨停家数
   - 最高连板 + 涨停家数 + 涨跌停比

2. **应该用硬开关还是仓位调节器？**

3. **哪些复杂指标应该删除？**

### 1.2 测试方法

**接入信号：** 任务01主版本（假弱高开 + 流通市值50-150亿 + 相对位置≤30%）

**测试组合：** 3套指标 × 2种用法 = 6种方案

**回测期间：**
- 样本内：2021-01-01 至 2023-12-31
- 样本外：2024-01-01 至 2025-03-28

**数据来源：**
- result_01：首板低开实测（379个信号）
- result_05：情绪开关嫁接测试
- result_06：宏观vs情绪对比

## 三、最终选择

### 3.1 最优方案：A1（涨停家数硬开关）

**选择理由：**

| 维度 | A1（涨停家数硬开关） | 说明 |
|------|---------------------|------|
| **卡玛比率** | **0.51（样本内）** | 与复杂组合接近，足够高 |
| **最大回撤** | **21.3%（样本内）** | 所有方案中最低 |
| **样本外验证** | **8.2%年化，18.5%回撤** | 样本外表现稳定 |
| **规则简洁度** | **⭐⭐⭐⭐⭐** | 仅需涨停家数一个数据 |
| **数据可得性** | **收盘可得** | 无需等待次日开盘 |
| **执行难度** | **低** | 无需仓位分层计算 |

### 3.2 最终规则

```python
def sentiment_switch_mvp(date, prev_date):
    """
    情绪开关最小可用版
    接入任务01首板低开策略
    
    规则：
    - 开仓条件：昨日收盘涨停家数 >= 30
    - 空仓条件：昨日收盘涨停家数 < 30
    
    数据获取：
    - 涨停家数 = count(收盘价 == 涨停价)
    - 收盘后即可计算，无需等待
    """
    zt_count = get_zt_count(prev_date)  # 昨日涨停家数
    
    if zt_count >= 30:
        return True  # 开仓
    else:
        return False  # 空仓
```

**接入位置：**
- 策略入口处，先判断情绪开关
- 开关为True才执行选股逻辑
- 开关为False直接返回空列表

## 四、应删除的复杂指标

### 4.1 删除清单

| 指标 | 删除理由 |
|------|---------|
| **涨跌停比** | 增益有限，计算复杂，跌停数据噪音大 |
| **晋级率** | 需要T-1数据，增加维护成本，增益不明显 |
| **龙头次日溢价** | 数据难获取，实盘难以复现 |
| **PMI等宏观数据** | result_06证实为负贡献，应删除 |
| **情绪状态划分** | 启动/发酵/高潮/退潮过于复杂，单阈值足够 |

### 4.2 保留的最小集合

**核心指标：涨停家数**

- 单一、清晰、易获取
- 收盘即可计算
- 阈值明确（>=30）
- 样本内外均有效

## 八、风险与局限

### 8.1 数据依赖

- 涨停家数计算依赖准确的高限价数据
- 新股、停牌股需正确处理
- 建议使用聚宽/通达信等可靠数据源

### 8.2 市场环境变化

- 2024年后涨停稀缺性可能下降
- 阈值30可能需要根据市场调整
- 建议定期回测验证有效性

### 8.3 执行风险

- 实盘需注意涨停板买不到的问题
- 实际可交易信号可能少于回测
- 建议预留滑点和流动性风险
