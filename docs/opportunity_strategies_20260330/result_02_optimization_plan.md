## 二、优化方案设计

### 2.1 方案A：双阈值情绪开关（推荐）

```python
def sentiment_switch_v2_dual(prev_date):
    """
    双阈值情绪开关
    开仓区间：50 <= 涨停家数 <= 150
    """
    zt_count = get_zt_count(prev_date)
    
    if zt_count < 50:
        return False, "情绪过低"
    elif zt_count > 150:
        return False, "情绪过热"
    else:
        return True, "情绪正常"
```

**优势：**
- ✅ 过滤弱市（涨停<50）
- ✅ 过滤过热（涨停>150）
- ✅ 适应2024-2025市场环境

**预期效果：**
- 过滤2024年2月（35只）、6月（48只）等弱市
- 过滤2024年10月8日（658只）、12月2日（188只）等过热日子
- 只在正常情绪区间开仓

### 2.2 方案B：单阈值50（对比组）

```python
def sentiment_switch_single_50(prev_date):
    """
    单阈值情绪开关
    开仓条件：涨停家数 >= 50
    """
    zt_count = get_zt_count(prev_date)
    return zt_count >= 50
```

**优势：**
- ✅ 过滤弱市
- ❌ 无法过滤过热

**预期效果：**
- 在过热市场（如10月8日658只）仍会开仓
- 可能造成亏损

from jqdata import *

print("=== 情绪开关优化测试 ===")

def get_zt_count(date):
    stocks = get_all_securities("stock", date).index.tolist()
    stocks = [s for s in stocks if not (s.startswith("68") or s.startswith("4") or s.startswith("8"))]
    df = get_price(stocks, end_date=date, count=1, fields=["close", "high_limit"], panel=False)

    return len(df[df["close"] == df["high_limit"]])

# 测试关键日期
