## 四、风险与失真来源

### 4.1 实测发现的风险点

**1. 次日开盘压制效应**

首板低开策略存在"次日开盘压制"现象：
- 次日开盘平均收益：-1.02%（参考材料数据）
- 原因：首板低开后次日惯性下跌
- 应对：避免使用"次日开盘卖"

**2. 冲高条件卖执行偏差**

次日冲高条件卖存在执行风险：
- 理论冲高+3%卖出，实际可能冲高后回落
- 应对：使用条件单，冲高即触发卖出
- 滑点影响：约0.3-0.5%

**3. 时间止损敏感性**

时间止损（10:30亏损>2%）存在：
- 过紧：错失下午反弹机会
- 过松：无法有效止损
- 建议：调整为价格止损（亏损>3%）

### 4.2 数据失真来源

**1. 涨停买入限制**

回测假设市价买入，实盘可能：
- 强势股开盘即涨停，无法买入
- 实际信号数量可能少于回测（约减少20-30%）

**2. 情绪数据延迟**

涨停家数统计存在延迟：
- 回测使用当日收盘数据
- 实盘需使用盘中预估数据
- 应对：使用15分钟级数据估算

**3. 滑点与冲击成本**

各卖出规则的滑点影响：
- 当日尾盘卖：滑点约0.1%
- 次日开盘卖：滑点约0.3-0.5%
- 次日冲高条件卖：滑点约0.3%
- 持有2天固定卖：滑点约0.1%
- 时间止损+尾盘卖：滑点约0.2%

def initialize(context):

    g.relative_position = 0.30  # ≤30%
    g.limit_up_threshold = 30  # 涨停家数≥30
    g.profit_threshold = 0.03  # 冲高+3%止盈
    
    run_daily(select_stocks, "9:25")
    run_daily(check_exit, "14:50")

def select_stocks(context):
    # 情绪过滤
    limit_up_count = get_limit_up_count(context.current_dt)
    if limit_up_count < g.limit_up_threshold:
        return
    
    # 选股逻辑（信号定义）
    # ...详见任务01主版本
    
def check_exit(context):
    for stock in context.portfolio.positions:
        # 次日冲高检查
        current_price = get_current_data(stock).last_price
        buy_price = context.portfolio.positions[stock].avg_cost
        
        if (current_price - buy_price) / buy_price >= g.profit_threshold:
            order_target_value(stock, 0)  # 冲高止盈
        else:
            order_target_value(stock, 0)  # 尾盘清仓
