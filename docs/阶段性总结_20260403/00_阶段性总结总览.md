> 日期: 2026-04-03
> 范围: `docs/` + `strategies/`
> 真实回测执行标准: `docs/backtest_guide/PROMPT.md`

## 这阶段已经收敛出来的主结论

### 1. 主仓主线已经比较清楚

- `RFScore + 防守组合` 这条线已经不是“想法验证”阶段，而是进入“上线候选收口”阶段。
- 当前最稳健的主仓基线仍然是 `V3 静态60/40`，对应 `strategies/dividend_value_quality_v1/v3_combo_static_60_40.py`。
- 当前进攻弹性最好的现成版本是 `V1 RFScore7 纯进攻`，对应 `strategies/dividend_value_quality_v1/v1_rfscore7_offensive.py`。
- `RFScore PB10 Final` 仍然是核心选股底座，但要先统一研究口径、监控口径和生产口径，再进入正式上线评审。

### 2. 动态主仓方向可以继续，但定位已经修正

- 动态主仓路由不是收益增强主线，而是风控层。
- `主仓动态路由V1` 已经有规格书，可以直接进入开发。
- `RSRS` 已经验证出价值，但最适合做 `veto / confirm` 过滤器，不适合在月频主策略上继续叠很多短线情绪过滤。

### 3. 主仓补充线已有明确白名单

- `#85 稳健型ETF策略` 是最值得先转译的低频 ETF 补充线。
- `#48 / #21 宏观 PMI` 是第二优先级的主仓补充线。
- `#53 PE 标准差` 更适合当仓位调节过滤器，不适合单独当主策略。

### 4. 几条线已经基本明确不该再当主推方向

- 当前 `ML 逻辑回归 4因子` 基线经修复后为 `No-Go`，短期不应再作为主线推进。
- `FFScore` 对 `RFScore` 的增量验证为 `No-Go`。
- 老的 `V4/V5 动态组合` 不再建议作为独立上线版本，后续由新的 `动态路由V1` 取代。

### 5. 机会策略线暂时不能进入正式上线队列

- `主线假弱高开 / 放宽版 / 主线二板组合` 目前更适合作为观察线或影子盘。
- `二板策略` 跨周期稳定性不足，不能作为正式主策略。

|------|-------------|----------|----------|

| 立即开发 | `V1 RFScore7 纯进攻` | 进攻型候选，适合卫星仓或进攻 sleeve | 进入真实平台完整回测 |
| 立即开发 | `RFScore PB10 Final` | 核心选股底座，仍是主仓候选 | 统一口径后进入真实平台完整回测 |
| 立即开发 | `主仓动态路由V1` | 规格已清晰，定位为风控层 | 立即开发，完成后进入真实回测 |
| 立即开发 | `RFScore + RSRS veto` | 接口定义已清晰，适合作为过滤层 | 立即开发，完成后进入真实回测 |
| 立即开发 | `#85 稳健型ETF` | 主仓补充线白名单第一优先 | 立即转译，完成后进入真实回测 |
| 继续探索 | `RFScore + STR` | 当前最值得继续验证的增强因子 | 做增量验证，通过后再进平台 |
| 继续探索 | `#48 / #21 宏观 PMI` | 值得做第二优先级补充线 | 先确认数据与简化逻辑，再做平台回测 |
| 继续探索 | `#53 PE 标准差过滤器` | 适合当仓位控制辅助层 | 先做辅助模块验证，不做独立主策略 |
| 继续探索 | `主线 / 二板机会策略` | 仅适合影子盘或极小仓观察 | 继续跟踪，但不进入正式上线队列 |
| 暂停 | `ML 逻辑回归当前基线` | 修复后为 No-Go | 暂停主线投入 |
| 暂停 | `FFScore 增强` | 与 RFScore 重叠度高 | 停止推进 |
| 暂停 | `筹码因子` | 数据成本和实现成本偏高 | 延后 |
| 暂停 | `老 V4 / V5 动态组合` | 已被新的路由思路替代 | 不再单独推进 |

## 这阶段最重要的判断

### 应该继续押注的方向

- `低频主仓`
- `RFScore 主仓底座`
- `静态60/40`
- `动态风控层`
- `RSRS veto`
- `ETF / 宏观补充线`

### 不应该再重押的方向

- `当前 ML 基线`
- `FFScore 对 RFScore 增量`
- `老的动态组合 V4 / V5`
- `把短线情绪阈值直接套到月频主策略`
