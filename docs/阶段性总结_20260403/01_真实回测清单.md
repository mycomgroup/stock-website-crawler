## 平台使用原则

按 `PROMPT.md` 的标准口径，当前统一使用下面这套分流：

| 阶段 | 适用对象 | 推荐平台 |
|------|----------|----------|
| 阶段2：新策略开发 | 新转译、简单因子、先做 smoke test | RiceQuant Notebook |
| 阶段3：完整回测 | 逻辑完整、需要完整风险指标 | RiceQuant 策略编辑器 |
| 阶段4：最终验证 | 成熟策略、准备上线前确认 | JoinQuant Strategy |

当前这批任务里，真正要上线评审的策略，默认都要至少走 `阶段3 -> 阶段4`。

## 提交口径建议

每次提交平台回测时，建议都附上这 4 个目标，避免 agent 只跑出一个收益数字：

1. 回测区间、调仓频率、持仓数、手续费、滑点全部写清楚。
2. 输出至少包含年化收益、最大回撤、夏普比率、换手率、胜率。
3. 给出和基线策略的对比，不要只看绝对收益。
4. 明确本次结论属于 `Go / Watch / No-Go` 哪一种。
