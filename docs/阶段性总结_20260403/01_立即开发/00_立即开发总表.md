## 每个项目的最小可验证单元

### 1. 主仓动态路由V1

1. 实现四状态输入口径，确保与冻结版定义一致。
2. 实现映射表 B 和回退条件。
3. 接到现有主仓底座，输出可运行策略文件。
4. 先做本地历史验证，确认不会明显劣于静态60/40。
5. 进入平台真实回测，重点看回撤、极端行情保护和换手。

### 2. RFScore + RSRS veto

1. 独立产出 RSRS 信号模块。
2. 明确 `confirm / veto / neutral` 的优先级和阈值。
3. 接到月频 RFScore 主仓，不叠加短线情绪阈值。
4. 先验证相对基线是否减少大回撤月份。
5. 进入平台真实回测，验证增量是否稳定。

### 3. #85 稳健型ETF策略

1. 先把聚宽原型整理成仓库内的可执行月频版本。
2. 确认 ETF 池、切换逻辑、调仓频率和 fallback 规则。
3. 先做本地回测或轻量验证。
4. 进入平台完整回测。
5. 评估它是独立稳健线，还是作为主仓补充线更合适。

## 推荐执行顺序

1. 先把 `V3 / V1 / RFScore PB10 Final` 送进真实平台完整回测。
2. 并行开发 `主仓动态路由V1`、`RFScore + RSRS veto`、`#85 稳健型ETF策略`。
3. 第二批开发完成后，再统一送平台做阶段2 / 3 / 4 验证。
