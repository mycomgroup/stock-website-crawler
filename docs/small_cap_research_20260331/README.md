## 一、研究目标

### 核心目标
- **稳健盈利**：控制回撤在25%以内
- **年化收益**：目标年化50%以上
- **策略容量**：单策略500万以上

### 技术路径
1. 国九条基础筛选层（财务质量）
2. 市值因子优化（避开微盘流动性风险）
3. 机器学习因子选择（动态权重）
4. 风控机制验证（止损、择时、空仓）

## 三、研究步骤

### Step 1: 国九条筛选有效性验证
- 目标：验证国九条筛选是否能有效过滤风险股票
- 方法：对比筛选前后股票的退市率、ST率、业绩表现
- 输出：筛选条件有效性报告

### Step 2: 因子有效性测试
- 目标：找出对小市值股票收益预测最有效的因子
- 方法：单因子IC测试、多因子相关性分析
- 输出：因子IC排序、因子相关性矩阵

### Step 3: 机器学习模型训练
- 目标：构建动态因子选择模型
- 方法：XGBoost/保形回归，滚动训练
- 输出：模型预测准确率、因子重要性

### Step 4: 策略回测验证
- 目标：验证完整策略的收益和回撤
- 方法：简化回测（Notebook方式）
- 输出：年化收益、最大回撤、夏普比率

### Step 5: 参数敏感性分析
- 目标：找出参数对策略表现的影响
- 方法：参数网格搜索
- 输出：参数敏感性矩阵、最优参数组合

## 五、RiceQuant API 适配说明

### 主要API差异

| 功能 | JoinQuant | RiceQuant |
|------|-----------|-----------|
| 获取所有股票 | `get_all_securities("stock", date)` | `get_all_securities(["stock"])` |
| 指数成分股 | `get_index_stocks(index, date)` | `index_components(index)` |
| 历史行情 | `get_price(stocks, end_date, count)` | `history_bars(stock, bar_count, frequency, fields)` |
| 财务数据 | `get_fundamentals(query, date)` | `get_factor` 或 `get_fundamentals` |
| 因子数据 | `get_factor_values` | `get_factor` |

### Notebook 运行方式

```bash
# 运行单个步骤
node run-strategy.js --strategy step1_guojutiao_filter.py

# 增加超时时间（因子测试可能需要较长时间）
node run-strategy.js --strategy step2_factor_test.py --timeout-ms 600000
```
