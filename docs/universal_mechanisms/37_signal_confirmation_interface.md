# SignalMaker 统一投票接口与再进场状态机

## 目标

这份文档把以下三件事收敛成统一接口：

1. `SignalMaker` 多模块统一投票
2. 连续两次确认
3. 止损后的再进场状态机

它解决的是：

- `30_signalmaker_filters.md` 已确认这些模块更适合作为过滤器/确认层
- `03_state_router.md` 已提出双确认与再进场
- 但两者还缺少一个统一的工程接口

---

## 适用模块

当前优先接入：

- `QRS`
- 鳄鱼线
- `AO/MACD`
- 北向资金分位
- 其他可标准化为方向信号的模块

不要求它们都单独能跑成主策略。

---

## 统一单信号输出格式

每个确认模块必须输出以下字段：

| 字段 | 含义 |
|------|------|
| `signal_name` | 模块名 |
| `signal_value` | `-1 / 0 / 1` |
| `confidence` | 0-100，可选 |
| `effective_date` | 生效日期 |
| `lookback_info` | 观察窗口，可选 |
| `reason` | 触发原因摘要 |

### `signal_value` 解释

| 值 | 含义 |
|------|------|
| `1` | 看多 / 放行 / 支持升档 |
| `0` | 中性 / 不表态 |
| `-1` | 看空 / 阻止恢复 / 支持降档 |

---

## 统一投票接口

### 1. 聚合方式

默认：

```text
vote_score = Σ signal_value
```

### 2. 默认解释

| `vote_score` | 含义 |
|------|------|
| `>= 2` | 允许升档或恢复 |
| `1` | 弱确认，只可观察，不建议直接升档 |
| `0` | 中性，维持原状态 |
| `<= -1` | 警示，不建议恢复 |
| `<= -2` | 支持降档或暂停 |

### 3. 使用原则

- 投票接口只修正主路由，不取代主路由
- 中等强度信号优先放进投票接口，而不是各自立策略
- 若模块之间高度同质，不能机械叠加票数

---

## 连续两次确认

### 目标

降低单日误触发，防止：
- 市场短暂波动导致误升档
- 止损后过早重新进场

### 默认规则

```text
第一次同向信号 = warning
连续第二次同向信号 = confirmed
中间若反向或失效 = 清空 warning
```

### 建议接法

- `03_state_router` 给出主状态
- `37` 管理 `warning / confirmed`
- 只有 `confirmed` 才允许切换到更积极状态

---

## 再进场状态机

### 状态定义

| 状态 | 含义 |
|------|------|
| `normal` | 正常持仓 |
| `warning` | 出现风险信号，但未正式切换 |
| `defensive` | 已降档 / 减仓 |
| `stopped` | 已止损 / 已停手 |
| `reentry_pending` | 主路由恢复，但仍等待二级确认 |
| `recovered` | 确认恢复，可重新进场 |

### 默认转移逻辑

```text
normal
→ warning
→ defensive / stopped
→ reentry_pending
→ recovered
→ normal
```

### 恢复条件

必须同时满足：

1. 主路由恢复到可做状态
2. 投票接口不为负值
3. 连续两次确认通过
4. 若有突破条件，再次突破关键位

否则只允许留在 `reentry_pending`，不能急于恢复满仓。

---

## 与主路由的关系

### 主路由负责

- 市场总状态
- 仓位大档位
- 是否允许总体开仓

### 确认接口负责

- 是否允许升档
- 是否允许止损后恢复
- 是否把单次噪声拦掉

### 风险层负责

- 最终仓位缩放
- 波动率或风险预算调整

三者不可混为一谈。

## 最低交付物

每次接入确认层，至少要输出：

1. 单模块信号表
2. 聚合投票分数
3. `warning / confirmed / reentry_pending` 状态日志
4. 升档 / 降档 / 恢复的触发原因

否则很难判断：
- 是主路由错了
- 还是确认层过度过滤了

---

## 常见错误

| 错误 | 问题 |
|------|------|
| 让确认层直接替代主路由 | 状态逻辑混乱 |
| 看到单日转强就满仓恢复 | 再进场过早 |
| 所有模块一律等权投票 | 忽略同质性与重复计票 |
| 不保留状态日志 | 无法复盘 |

## 结论

以后 `SignalMaker` 的默认角色不再是“再找一个单独择时策略”，而是：

- **统一投票接口**
- **双确认执行器**
- **再进场保护层**

这会比继续堆单个技术信号，更符合主仓工程化方向。
