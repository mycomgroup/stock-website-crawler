## 一、四状态定义

状态命名采用两套术语体系，本质相同：

| 状态编号 | 择时集成命名 | 状态路由器命名 | 判定条件 | 核心含义 |
|---------|------------|--------------|---------|---------|
| 1 | **关闭** | 底部试错 | 广度<15% 或 情绪冰点 | 市场极度低迷，不参与 |
| 2 | **防守** | 震荡轮动 | 广度15-35% 且 情绪30-80 | 均衡配置，择强信号 |
| 3 | **正常** | 趋势进攻 | 广度>35% 且 情绪50-80 | 趋势确认，积极参与 |
| 4 | **进攻** | 高估防守 | 宽度>70% 或 估值高估 | 泡沫风险，主动降仓 |

### 1.1 输入指标（固化，不新增）

**广度指标：**
- 计算：沪深300成分股 `close > MA20` 的比例
- 来源：沪深300成分股行情数据
- 更新：日频（收盘后）

**情绪指标：**
- 计算：全市场涨停家数（涨幅>=9.5%）
- 来源：全市场行情
- 更新：日频（收盘后）

**估值指标（辅助）：**
- FED指标 = 盈利收益率(100/PE) - 国债收益率
- 用途：辅助判断高估/低估

### 1.2 状态判定矩阵

```
                    情绪
              冰点(<30)  启动(30-50)  发酵(50-80)  高潮(>80)
广度   <15%    关闭        关闭         关闭         关闭
      15-25%   关闭        防守         防守         防守
      25-35%   关闭        防守         正常         正常
      35-50%   关闭        防守         正常         正常
       >50%    关闭        防守         正常         进攻
       >70%    进攻        进攻         进攻         进攻
```

## 二、转换规则

### 2.1 上行转换（状态编号增大）

| 转换路径 | 触发条件 | 确认条件 |
|---------|---------|---------|
| 关闭 → 防守 | 广度回升至15%以上 | 连续2日确认 |
| 防守 → 正常 | 广度回升至35%以上且情绪>50 | 连续2日确认 |
| 正常 → 进攻 | 宽度>50%且情绪持续上升 | 连续2日确认 |
| 任意 → 进攻 | 宽度>70%或估值高估 | 立即触发 |

### 2.2 下行转换（状态编号减小）

| 转换路径 | 触发条件 | 确认条件 |
|---------|---------|---------|
| 进攻 → 正常 | 宽度回落至50%以下 | 连续2日确认 |
| 正常 → 防守 | 广度下降至35%以下 | 连续2日确认 |
| 防守 → 关闭 | 广度下降至15%以下 | 连续2日确认 |
| 进攻 → 关闭 | 宽度>70%后回落 | 连续2日确认 |

## 三、连续2日确认原则

### 3.1 核心逻辑

```
状态转换判定：
1. 每日收盘后计算广度和情绪指标
2. 判定理想状态（基于当前指标）
3. 记录最近2日的状态序列
4. 若最近2日理想状态相同，且与当前状态不同 → 执行转换
5. 否则保持当前状态
```

### 3.2 连续2日确认的意义

| 场景 | 单日波动 | 连续2日确认效果 |
|-----|---------|----------------|
| 广度突升突降 | 误判为状态切换 | 过滤，避免频繁切换 |
| 情绪数据异常 | 误判状态 | 保护，减少假信号 |
| 政策市/事件驱动 | 短期失真 | 缓冲，提高稳定性 |

### 3.3 最短持续时间

- 状态一旦确认，至少持续1个交易日
- 防止状态在确认当日即被回转

## 四、主仓映射

适用策略：RFScore7+PB20（选股策略）、高股息防守、ETF动量

### 4.1 仓位配置

| 状态 | 主仓总仓位 | 配置逻辑 |
|-----|----------|---------|
| **关闭** | 0% | 不参与，等待机会 |
| **防守** | 50% | 均衡配置，择强信号 |
| **正常** | 80-100% | 积极参与，跟随趋势 |
| **进攻** | 20-30% | 泡沫风险，降仓避险 |

### 4.2 单策略配置

| 状态 | RFScore7+PB20 | 高股息防守 | ETF动量 |
|-----|--------------|-----------|---------|
| 关闭 | 0% | 0% | 0% |
| 防守 | 20% | 20% | 10% |
| 正常 | 35-40% | 20% | 20-30% |
| 进攻 | 10% | 15% | 0% |

### 4.3 主仓行为指引

**关闭状态：**
- 不建新仓
- 持有资产逐步减仓
- 保留现金等待机会

**防守状态：**
- 仓位上限50%
- 信号只选最强（假弱高开+题材热点）
- 单票上限2.5%，总仓≤15%

**正常状态：**
- 仓位上限100%
- 正常信号筛选
- 单票上限5%，总仓≤30%

**进攻状态：**
- 仓位降至20-30%
- 持有高股息防守资产
- 警惕泡沫，逐步减仓

## 五、机会仓映射

适用策略：二板接力、题材炒作、事件驱动

### 5.1 仓位配置

| 状态 | 机会仓总仓位 | 配置逻辑 |
|-----|------------|---------|
| **关闭** | 0% | 不参与 |
| **防守** | 20% | 仅最强信号 |
| **正常** | 50% | 正常参与 |
| **进攻** | 30% | 过热信号，减少参与 |

### 5.2 信号筛选

| 状态 | 信号标准 | 涨停家数要求 |
|-----|---------|-------------|
| 关闭 | 无信号 | <30只 |
| 防守 | 只选最强 | >=30只，连续2日 |
| 正常 | 正常筛选 | >=50只 |
| 进攻 | 适度放宽 | >=80只可放宽 |

### 5.3 机会仓行为指引

**关闭状态：**
- 完全不参与机会仓
- 专注主仓防守

**防守状态：**
- 仅参与最强信号
- 单票仓位≤1%
- 总机会仓≤20%

**正常状态：**
- 正常参与机会仓
- 单票仓位≤2.5%
- 总机会仓≤50%

**进攻状态：**
- 市场过热，减少参与
- 单票仓位≤2%
- 总机会仓≤30%
- 警惕高位接盘风险

## 六、二次进场模块

### 6.1 模块定位

二次进场模块是状态切换后的增强机制：
- 主仓和机会仓在状态切换后可能错过最佳时机
- 二次进场提供状态确认后的补充入场机会
- 核心原则：状态确认后的第一个回调低点

### 6.2 二次进场触发条件

| 状态转换 | 是否触发二次进场 | 进场时机 |
|---------|----------------|---------|
| 关闭 → 防守 | 是 | 连续2日确认后，首个回调日 |
| 防守 → 正常 | 是 | 连续2日确认后，首个回调日 |
| 正常 → 进攻 | 否 | 进攻状态已是高位，不追 |
| 进攻 → 防守 | 是 | 连续2日确认后，反弹高点减仓 |
| 防守 → 关闭 | 否 | 关闭状态空仓，不进场 |
| 任意 → 进攻（宽度>70%） | 否 | 高估信号明确，不进场 |

### 6.3 二次进场仓位

```
触发二次进场时：
- 主仓：追加原目标仓位的20%
- 机会仓：不追加（防守状态除外）
- 最大总仓位不超过状态上限
```

### 6.4 二次进场止损

```
二次进场后若反向：
- 主仓：止损5%，或状态再次切换
- 机会仓：止损3%
- 连续2日确认状态回转，立即退出
```
