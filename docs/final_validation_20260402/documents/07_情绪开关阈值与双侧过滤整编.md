## 三、双侧过滤规则

### 3.1 核心理念

**"不过冷也不过热"**

- **过冷（<30）**：买什么都亏的系统性风险，必须空仓
- **过热（>80）**：高潮次日必退潮，2025年Q1验证了过热市场的危险性

### 3.2 仓位状态机

```
涨停<30        → 空仓（过冷不开）
涨停30-80      → 正常开仓（全仓）
涨停>80        → 半仓（过热降频）
跌停>30        → 空仓（退潮确认）
最高连板<2     → 空仓（龙头断板）
```

### 3.3 五阶段模型对应操作

| 阶段 | 涨停家数 | 状态 | 操作 |
|------|----------|------|------|
| 冰点期 | <10 | 绝望 | 空仓 |
| 启动期 | 10-30 | 复苏试探 | 半仓 |
| 发酵期 | 30-50 | 主升 | 全仓 |
| 高潮期 | 50-80 | 过热警戒 | 半仓或观望 |
| 退潮期 | <20 | 崩塌 | **空仓** |

## 四、恢复分级回仓

### 4.1 恢复信号清单

| 信号 | 触发条件 | 说明 |
|------|----------|------|
| 涨停回升 | 涨停家数连续2天>30 | 情绪回暖 |
| 跌停下降 | 跌停家数连续2天<15 | 风险释放 |
| 新龙头出现 | 最高连板回升至>=3 | 新周期启动 |
| 晋级率回升 | 晋级率连续2天>25% | 接力恢复 |

### 4.2 恢复节奏

```
退潮/冰点后恢复节奏：

空仓观望
    ↓ 连续2天涨停>30
恢复首日 → 仓位50%
    ↓ 次日
恢复次日 → 仓位75%
    ↓ 再次日
恢复第三日 → 仓位100%
```

## 六、风险提示

1. **阈值30在2025年Q1完全失效**（涨停均值77.9只），必须配合高情绪过滤
2. **涨停>80为过热警戒**，非绝对空仓，但需降频50%
3. **单一涨停家数不足**，需配合跌停家数和连板数综合判断
4. **使用T-1日数据**，盘中实时数据存在延迟风险
5. **硬开关优于仓位调节器**：回撤更低（23.55% vs 32.99%），夏普更高（0.34 vs 0.29）

## 七、最终推荐口径（可直接复用）

### 7.1 核心开关函数

```python
def sentiment_switch_unified(prev_date):
    """
    情绪开关统一总开关 v1.0
    适用：主仓 + 机会仓
    目标：低频高胜率化

    返回值：
        'NO_POSITION'     -> 空仓（过冷/退潮/龙头断板）
        'HALF_POSITION'   -> 半仓（过热警戒）
        'FULL_POSITION'   -> 全仓（正常区间）
    """
    zt = get_zt_count(prev_date)           # 涨停家数
    dt = get_dt_count(prev_date)           # 跌停家数
    max_board = get_max_board(prev_date)   # 最高连板数

    # 过冷 → 空仓
    if zt < 30:
        return 'NO_POSITION'

    # 退潮确认 → 空仓
    if dt > 30 or max_board < 2:
        return 'NO_POSITION'

    # 过热 → 半仓
    if zt > 80:
        return 'HALF_POSITION'

    # 正常区间 → 全仓
    return 'FULL_POSITION'
```

### 7.2 恢复分级函数

```python
def sentiment_recovery(prev_date, drawback_days):
    """
    退潮恢复分级回仓
    适用于NO_POSITION状态后的恢复节奏控制

    参数：
        prev_date: T-1日期
        drawback_days: 连续空仓天数（从1开始计数）

    返回值：
        0.0  -> 继续空仓（未恢复）
        0.5  -> 半仓（恢复首日）
        0.75 -> 75%仓（恢复次日）
        1.0  -> 全仓（恢复正常）
    """
    zt = get_zt_count(prev_date)

    # 恢复信号：涨停连续>30
    if zt > 30:
        if drawback_days == 1:
            return 0.5   # 首日50%
        elif drawback_days == 2:
            return 0.75  # 次日75%
        else:
            return 1.0  # 第三日起100%

    return 0.0  # 未恢复，继续空仓
```

### 7.3 仓位上限计算示例

```python
def calculate_position_limit(context, switch_result, recovery_ratio=1.0):
    """
    根据情绪开关结果计算仓位上限

    参数：
        context: 回测context
        switch_result: sentiment_switch_unified返回值
        recovery_ratio: sentiment_recovery返回值（仅在NO_POSITION后使用）

    返回值：
        可用仓位比例（0.0 - 1.0）
    """
    if switch_result == 'NO_POSITION':
        return 0.0

    if switch_result == 'HALF_POSITION':
        return 0.5 * recovery_ratio

    # FULL_POSITION
    return 1.0 * recovery_ratio
```
