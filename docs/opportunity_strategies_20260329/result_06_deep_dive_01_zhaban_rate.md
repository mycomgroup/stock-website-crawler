## 一、炸板率定义和数据获取方法

### 1.1 炸板率定义

**炸板率** = 当日炸板股数量 / 当日曾涨停股数量

**具体定义：**
- **曾涨停股**：当日最高价触及涨停价的股票（最高价 ≥ 涨停价 × 0.995）
- **炸板股**：曾涨停但收盘未封住涨停的股票（收盘价 < 涨停价 × 0.995）
- **炸板率**：炸板股数量 / 曾涨停股数量

**意义：**
- 炸板率反映市场情绪的稳定性
- 炸板率高 = 涨停股容易打开，情绪不稳定，风险大
- 炸板率低 = 涨停股封板牢固，情绪稳定，机会多

### 1.2 数据获取方法（JoinQuant）

```python
def get_zhaban_rate(date):
    """
    计算炸板率
    """
    all_stocks = get_all_securities("stock", date).index.tolist()
    # 过滤科创板、创业板、北交所
    all_stocks = [s for s in all_stocks 
                  if not (s.startswith("68") or s.startswith("4") or s.startswith("8"))]
    
    # 获取当日行情数据
    df = get_price(
        all_stocks,
        end_date=date,
        count=1,
        fields=["high", "close", "high_limit"],
        panel=False,
        fill_paused=False,
    )
    df = df.dropna()
    
    # 曾涨停股（最高价触及涨停价）
    touched = df[df["high"] >= df["high_limit"] * 0.995]
    touched_count = len(touched)
    
    if touched_count == 0:
        return 0.0
    
    # 炸板股（收盘未封住涨停）
    zhaban = touched[touched["close"] < touched["high_limit"] * 0.995]
    zhaban_count = len(zhaban)
    
    return zhaban_count / touched_count
```

## 五、技术细节和注意事项

### 5.1 炸板率计算的技术细节

**精度要求：**
- 涨停价判断：收盘价 ≥ 涨停价 × 0.995（容忍0.5%误差）
- 曾涨停判断：最高价 ≥ 涨停价 × 0.995（容忍0.5%误差）
- 原因：考虑数据精度、价格波动、系统延迟等因素

**股票过滤：**
- 科创板（68开头）：剔除（涨跌幅20%，涨停价计算不同）
- 创业板（300开头）：保留（涨跌幅10%，与主板一致）
- 北交所（4、8开头）：剔除（涨跌幅30%，涨停价计算不同）

### 5.2 炸板率的局限性

**局限性1：采样误差**
- 全市场股票约5000只，批量获取耗时长
- 采样100只股票可能导致炸板率估算不准确
- 建议：分层采样或使用指数成分股

**局限性2：时效性**
- 炸板率需要当日收盘后才能计算
- 无法用于盘中实时决策
- 建议：结合盘中涨停打开统计（需要分钟数据）

**局限性3：极端值**
- 炸板率可能为0%（所有涨停股都封住）或100%（所有涨停股都打开）
- 极端值可能代表特殊情况（市场极端情绪）
- 建议：设置炸板率范围过滤（如10% ~ 90%）
