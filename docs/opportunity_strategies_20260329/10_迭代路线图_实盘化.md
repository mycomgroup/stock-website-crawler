# 任务 10：机会仓迭代路线图与实盘化研究

## 设计文档

### 任务定位

- 主线：`从研究走向实盘`
- 目标：在不拘泥现有代码的前提下，给出一条最短的迭代路径，把高爆发机会仓从研究假设推进到真实可执行系统。
- 角色：不是再发明新策略，而是决定先做什么、后做什么、砍掉什么。

### 当前已知问题

- 你明确说可以不拘泥于现有代码，先有方向再沿路迭代，但原稿没有把这件事拆成清晰的阶段任务。
- 目前最怕的是同时推进太多方向，最后每条线都浅尝辄止。
- 机会仓的核心目标是短期快速收益，所以路线图必须优先保障验证速度，而不是系统工程完整性。
- 研究到实盘之间最大的鸿沟是数据可得性、回测偏差、执行滑点和信号衰减。

### 本轮要回答的问题

1. 十个任务里哪些必须先做，哪些可以后做，哪些如果结果差就应该直接砍掉。
2. 最短闭环是 `宏观开关 -> 两个高频入口 -> 风控 -> 组合`，还是别的路径。
3. 什么时候该从 `joinquant_nookbook` 研究阶段切到 `joinquant_strategy` 正式策略回测阶段。
4. 哪些验证结论足以支持小资金试盘。

### 研究假设与重点变量

重点拆解：

- 阶段优先级：总开关、最强单点、组合、风控、因子、实盘迁移

- 决策门槛：保留、观察、砍掉、延后
- 实盘门槛：最近样本外、滑点敏感性、交易频率、容量约束、连续亏损承受度
