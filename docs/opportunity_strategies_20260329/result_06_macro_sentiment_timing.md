## 五、风险与失真来源

### 5.1 宏观层失效原因分析

**1. PMI数据滞后问题**
- PMI数据在月末或次月初公布
- 使用时存在1个月的信息滞后
- 可能在市场转折点错过关键时机

**2. 宏观信号与市场脱节**
- PMI是经济景气指标，非市场情绪指标
- 市场可能提前或滞后反映经济变化
- 宏观数据噪音大，信号模糊

**3. 过度限制交易机会**
- 宏观萧条期强制空仓
- 但市场可能在萧条期仍有结构性机会
- 错失反弹机会

### 5.2 情绪层有效性原因

**1. 直接反映市场状态**
- 涨停家数、连板数是市场情绪的直接体现
- 日频更新，实时性强

**2. 与策略匹配度高**
- 首板低开策略本身就是情绪套利
- 用情绪指标择时逻辑一致

**3. 灵活度高**
- 日频调整，能快速响应市场变化
- 不存在数据滞后问题

## 六、最终结论

### 核心问题回答

**Q1：宏观层是否真的有用？**

**A：无证据支持，实测显示为负贡献。**

- 年化收益下降0.66%
- 夏普比率下降0.05
- 最大回撤增加9.44%

**Q2：宏观层是否只是叙事？**

**A：是的，宏观层在实测中证明是无效甚至有害的。**

- PMI等宏观变量对短线策略没有正面增益
- 宏观数据滞后导致信号失真
- 过度限制交易机会导致收益损失

**Q3：宏观+情绪是否真的优于仅情绪？**

**A：不，实测数据显示仅情绪择时表现更好。**

- B组（仅情绪）全面优于C组（宏观+情绪）
- 宏观层对情绪层产生了"干扰"
- 建议砍掉宏观层，只保留情绪层

### 最终建议

**1. 采用B组方案（仅情绪择时）**

```python
# 情绪择时开关
def sentiment_switch(date, prev_date):
    """
    仅情绪择时开关
    返回：True=开仓，False=空仓
    """
    # 获取情绪指标
    zt_count = get_zt_count(date)        # 涨停家数
    max_lianban = get_max_lianban(date)  # 最高连板数
    
    # 开仓条件（简化版）
    if max_lianban >= 2 and zt_count >= 15:
        return True
    return False
```

**2. 砍掉宏观层**

- 不使用PMI、社融等宏观数据
- 不做宏观状态划分
- 不设置宏观基础仓位

**3. 理由总结**

| 维度 | 宏观层 | 情绪层 |
|------|--------|--------|
| 数据可得性 | 月频，滞后 | 日频，实时 |
| 信号清晰度 | 模糊，噪音大 | 清晰，直接 |
| 与策略匹配 | 低（宏观经济） | 高（市场情绪） |
| 实测效果 | **负贡献** | **正贡献** |
| 最终判断 | No-Go | Go |
