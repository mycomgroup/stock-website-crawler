## 一、三档仓位定义和阈值

### 1.1 三档仓位定义

**设计思路：**
- 原有开关式（开仓/空仓）过于粗糙，无法适应不同市场环境
- 三档仓位将情绪状态细分为三种，提供更精细的仓位管理

**具体定义：**

| 档位 | 仓位比例 | 情绪状态 | 触发条件 |
|------|---------|---------|---------|
| **满仓档** | 1.0 (100%) | 情绪热 | 涨停家数 > 50 且 连板数 > 5 |
| **半仓档** | 0.5 (50%) | 情绪中 | 涨停家数 30-50 且 连板数 3-5 |
| **空仓档** | 0.0 (0%) | 情绪冷 | 涨停家数 < 30 或 连板数 < 3 |

**对比原有开关式：**

| 类型 | 仓位比例 | 触发条件 |
|------|---------|---------|
| 开关式（A组） | 1.0 或 0.0 | 涨停家数 ≥ 30 且 连板数 ≥ 3 → 开仓 |
| 三档式（B组） | 1.0、0.5、0.0 | 分层仓位调节（见上表） |

**核心差异：**
- A组：二元开关（要么开仓，要么空仓）
- B组：分层仓位（根据情绪强度调整仓位）

### 1.2 阈值设计逻辑

**涨停家数阈值：**
- **>50**：市场高度活跃，涨停股数量显著，情绪亢奋
- **30-50**：市场中等活跃，有一定机会，但需谨慎
- **<30**：市场低迷，涨停股稀少，风险较大

**连板数阈值：**
- **>5**：连板龙头出现，市场有强主线，情绪高涨
- **3-5**：连板股存在，市场有中等主线，情绪平稳
- **<3**：缺乏连板龙头，市场无主线，情绪低迷

**组合条件设计：**
- 满仓档：两个条件同时满足（涨停数>50 且 连板数>5）
- 半仓档：两个条件同时满足（涨停数30-50 且 连板数3-5）
- 空仓档：任意条件不满足（涨停数<30 或 连板数<3）

**设计原则：**
- 保守原则：必须两个条件同时满足才开仓
- 分层原则：不同情绪强度对应不同仓位
- 避免误判：阈值设置留有缓冲区间

## 二、A组 vs B组回测对比

### 2.1 测试设计

**测试策略：首板低开策略**

- 选股条件：昨日涨停股 + 今日低开（-5% ~ -1%）
- 买入时机：开盘价买入
- 卖出时机：次日开盘价卖出
- 最大持仓：3只股票

**仓位模式对比：**

| 组别 | 仓位模式 | 仓位比例 | 买入金额计算 |
|------|---------|---------|-------------|
| A组 | 开关式 | 1.0 或 0.0 | 基础金额 × 仓位比例 |
| B组 | 三档式 | 1.0、0.5、0.0 | 基础金额 × 仓位比例 |

**时间范围：**
- 2024-01-01 至 2025-12-31（约487个交易日）

### 2.2 理论收益计算模型

**收益计算公式：**

```
单笔收益 = (卖出价 - 买入价) / 买入价 × 仓位比例
累计收益 = Σ 单笔收益
```

**关键差异：**
- A组：开仓时仓位比例=1.0，收益全额计入
- B组：
  - 满仓档：仓位比例=1.0，收益全额计入
  - 半仓档：仓位比例=0.5，收益减半计入
  - 空仓档：仓位比例=0.0，不交易，无收益

## 四、结论：分层仓位是否优于开关式？

### 4.1 理论分析结论

**优势分析：**

1. **精细化仓位管理**
   - 三档仓位能够根据情绪强度调整仓位，避免一刀切
   - 半仓档提供缓冲，降低过度交易风险

2. **风险控制更灵活**
   - 情绪中等时（半仓档），仓位减半，降低单笔亏损影响
   - 情绪高涨时（满仓档），仓位全额，充分捕捉机会

3. **交易频率优化**
   - 满仓档更严格，减少在高风险环境下的重仓交易
   - 半仓档提供中间层，避免错过中等情绪的机会

**劣势分析：**

1. **阈值设置复杂**
   - 三档仓位需要设置两组阈值（涨停数、连板数）
   - 阈值设置不当可能导致误判或错过机会

2. **执行复杂度增加**
   - 需要逐日判断三档仓位，计算逻辑更复杂
   - 仓位比例调整可能带来额外的操作成本

3. **收益不确定性**
   - 半仓档收益减半，可能降低总体收益
   - 满仓档触发频率低，可能错过部分机会

## 五、风险与失真来源

### 5.1 阈值设置风险

**风险1：阈值设置不当**

- 涨停数阈值（50、30）和连板数阈值（5、3）基于经验设置
- 不同市场环境下，最优阈值可能不同
- 需要通过历史数据验证阈值合理性

**风险2：阈值过于严格**

- 满仓档需要涨停数>50 且 连板数>5，触发频率可能过低
- 可能错过部分中等强度的机会

**风险3：阈值过于宽松**

- 半仓档触发频率可能过高，导致过度保守
- 降低总体收益水平

### 5.2 仓位比例设置风险

**风险1：半仓比例固定**

- 半仓档仓位比例固定为0.5，可能不适应所有市场环境
- 不同策略可能需要不同的半仓比例（如0.3、0.6）

**风险2：仓位比例调整滞后**

- 仓位比例调整基于昨日情绪数据，可能存在滞后
- 市场情绪快速变化时，仓位调整可能不及时

### 5.3 Notebook环境限制

**限制1：执行时间限制**

- Notebook默认执行时间60秒，最大可扩展至180秒
- 三档仓位回测涉及约487天×5000股的数据处理，远超时间限制

**限制2：数据获取性能**

- 批量获取涨停股、连板数、首板低开信号耗时较长
- Notebook环境对数据获取有严格的性能限制

**限制3：计算逻辑复杂**

- 三档仓位需要逐日判断档位，逐笔计算收益
- 计算逻辑复杂，执行时间长

### 7.1 三档仓位计算的技术细节

**仓位比例计算：**

```python
def get_sentiment_three_level(zt_count, max_lianban):
    """三档仓位调节"""
    if zt_count > 50 and max_lianban > 5:
        return 1.0, "满仓"
    elif 30 <= zt_count <= 50 and 3 <= max_lianban <= 5:
        return 0.5, "半仓"
    else:
        return 0.0, "空仓"
```

**买入金额计算：**

```python
base_value = context.portfolio.total_value / g.ps  # 基础金额（总资金/持仓数）
position_ratio = g.position_ratio  # 仓位比例（1.0、0.5、0.0）
value = base_value * position_ratio  # 实际买入金额
```

**收益计算：**

```python
pnl_pct = (sell_price - buy_price) / buy_price * 100 * position_ratio
# 注：收益需乘以仓位比例，反映实际收益
```
