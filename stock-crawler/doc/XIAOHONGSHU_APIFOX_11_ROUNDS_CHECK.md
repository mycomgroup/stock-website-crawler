## Markdown 质量检查（结构/段落是否清晰）

### 结论（简要）

- **结构有明显混乱，段落不够清晰**。
- **存在导航菜单内容大段挤在同一行的情况**，可读性较差。
- **内容重复较多**（例如导航区先出现“分组+子链接连写”，随后又重复列出子链接）。

### 具体表现

1. `新手指南.md` 中，`## 内容` 下的很多条目是这种形态：
   - `- 开发文档[应用开发](...)[新手指南](...)...`
   - 即“父级栏目 + 多个子链接”被拼成一行，缺乏分层换行。
2. `新手指南.md` 下还出现了单独的 `## 图片`、`## 图表` 聚合块，与正文中的同一图片又重复出现，信息冗余。
3. `数据加密对接方案.md` 中正文很短，但后续 `## 列表` 中灌入了超长导航列表，主体内容占比偏低。

## 与原页面是否有缺失（本次可确认范围）

- 从已抓取结果看，页面核心标题与部分正文节点已抓到（如“新手指南”“开发者指南”“帐号注册、登录”等）。
- 但由于当前只完成了 2 个链接的有效抓取，站点级别内容仍有 **36** 条未抓，整体覆盖不完整。
- 在已抓页面内，存在“正文被导航噪音淹没”的问题，导致“是否缺失”很难直接从 Markdown 快速核对。

## 建议

1. 在内容提取阶段提高主内容权重，降低左侧目录/导航权重。
2. 对“父级栏目 + 子链接串联”的节点做二次拆分，强制按子项换行。
3. 图片与图表输出去重，避免正文中已出现后又在附录区域重复全集列出。
4. 若目标是比对完整性，建议先把 36 条 `unfetched` 跑完后再做页面对比评估。
