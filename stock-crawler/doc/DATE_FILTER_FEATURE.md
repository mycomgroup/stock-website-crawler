# 时间筛选自动处理功能

## 概述

爬虫现在支持自动检测和处理页面上的时间筛选控件，能够智能地尝试不同的时间范围来获取数据。

## 功能特性

### 1. 自动检测时间筛选控件

爬虫会自动检测页面中的日期筛选控件：

- 标准日期输入框（`<input type="date">`）
- 包含"日期"、"时间"关键字的输入框
- 开始/结束日期输入框对（通过placeholder、label、id识别）
- 支持中英文标识（开始/起始/from/start，结束/截止/to/end）

### 2. 智能时间范围尝试

当检测到时间筛选控件时，爬虫会按以下顺序尝试不同的时间范围：

1. **2000年至今** - 最大范围，获取所有历史数据
2. **最近10年** - 如果最大范围失败或无数据
3. **最近5年** - 进一步缩短范围
4. **最近3年** - 更短的范围
5. **最近1年** - 最小范围

### 3. 错误和无数据检测

爬虫会自动检测：

- **错误提示**：页面上的错误消息（"错误"、"失败"、"error"等）
- **无数据提示**：空表格或"暂无数据"、"no data"等提示
- **数据存在性**：检查表格是否有实际数据行

如果当前时间范围返回错误或无数据，自动尝试下一个更短的时间范围。

### 4. 数据追加写入

- 先保存初始页面数据（不带时间筛选）
- 时间筛选后的数据追加到同一个Markdown文件
- 每个时间范围的数据独立标记

## 工作流程

```
1. 访问页面
   ↓
2. 保存初始数据（无筛选）
   ↓
3. 检测时间筛选控件
   ├─ 未找到：结束
   └─ 找到：继续
   ↓
4. 尝试时间范围：2000年至今
   ├─ 填充开始日期：2000-01-01
   ├─ 填充结束日期：今天
   └─ 提交/触发查询
   ↓
5. 检查结果
   ├─ 有数据且无错误：提取数据 → 追加到文件 → 结束
   ├─ 有错误：尝试下一个范围（最近10年）
   └─ 无数据：尝试下一个范围（最近10年）
   ↓
6. 重复步骤4-5，直到成功或尝试完所有范围
```

## 技术实现

### 日期控件检测

```javascript
// 查找开始日期输入框
const startInputs = allInputs.filter(input => {
  const placeholder = input.placeholder || '';
  const label = input.getAttribute('aria-label') || '';
  const id = input.id || '';
  return /开始|起始|from|start/i.test(placeholder + label + id);
});

// 查找结束日期输入框
const endInputs = allInputs.filter(input => {
  const placeholder = input.placeholder || '';
  const label = input.getAttribute('aria-label') || '';
  const id = input.id || '';
  return /结束|截止|to|end/i.test(placeholder + label + id);
});
```

### 日期填充和提交

```javascript
// 填充日期
await page.locator(startSelector).fill('2000-01-01');
await page.locator(endSelector).fill(todayStr);

// 提交查询
if (hasSubmitButton) {
  await submitButton.click();
} else {
  await page.locator(endSelector).press('Enter');
}

// 等待数据加载
await page.waitForTimeout(2000);
```

### 数据和错误检测

```javascript
// 检查错误
const errorElements = document.querySelectorAll('.error, .alert-error, [role="alert"]');
const hasError = Array.from(errorElements).some(el => 
  /错误|失败|error|无数据|no data/i.test(el.textContent)

// 检查数据
const tables = document.querySelectorAll('table');

  table.querySelectorAll('tbody tr, tr').length > 1

```

## 配置

时间筛选功能是自动的，不需要额外配置。爬虫会自动检测页面上的时间筛选控件并处理。

如果需要禁用此功能，可以在parser中跳过`findAndProcessDateFilters`调用。

## 注意事项

1. **时间格式**：使用ISO格式（YYYY-MM-DD），兼容性最好
2. **等待时间**：每次提交后等待2秒，确保数据加载完成
3. **最大范围**：从2000年开始，覆盖大多数互联网数据
4. **最小范围**：最近1年，确保至少能获取部分数据
5. **数据追加**：所有时间范围的数据追加到同一文件，便于查看完整历史

## 优势

1. **自动化**：无需手动设置时间范围
2. **智能重试**：自动尝试不同范围直到成功
3. **数据完整性**：尽可能获取最长时间范围的数据
4. **容错性强**：处理各种错误和无数据情况
5. **统一存储**：所有数据保存在同一文件，便于管理

## 限制

1. **控件识别**：依赖常见的日期控件模式，特殊控件可能无法识别
2. **提交方式**：支持按钮提交和Enter键提交，其他方式可能不支持
3. **等待时间**：固定等待2秒，对于加载很慢的页面可能不够
4. **时间范围**：预设5个范围，可能不适合所有场景

## 未来改进

1. 支持更多日期控件类型（日期选择器组件、下拉框等）
2. 可配置的时间范围列表
3. 动态等待（检测加载完成而不是固定等待）
4. 支持更多提交方式
5. 时间范围优化（根据页面提示智能选择）
