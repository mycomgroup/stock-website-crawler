# Tab页和下拉框自动提取功能

## 概述

爬虫现在支持自动检测和提取页面中的Tab页和下拉框内容，并智能判断内容是否重复，只保存有变化的数据。

## 功能特性

### 1. Tab页自动检测

爬虫会自动检测页面中的Tab控件，支持多种Tab样式：

- 标准Tab组件（`[role="tab"]`）
- 自定义Tab样式（`.tab`, `.tabs button`, `.tabs a`）
- 带有`tab`关键字的按钮和链接

### 2. 下拉框自动检测

爬虫会自动检测页面中的下拉框控件：

- 标准`<select>`元素
- 带有`role="combobox"`的元素
- 自定义下拉框样式（`.dropdown select`, `[class*="select"]`）

### 3. 智能去重机制

- **文件内容比较**：将提取的内容与已保存的文件进行比较
- **只保存新内容**：只有当内容不存在于文件中时才追加写入
- **标准化比较**：去除多余空格和换行进行比较，避免格式差异导致的误判

## 工作流程

```
1. 访问页面并保存初始数据
   ↓
2. 查找所有Tab按钮
   ↓
3. 逐个点击Tab
   ↓
4. 提取当前Tab的内容
   ↓
5. 生成Markdown格式内容
   ↓
6. 与文件中已有内容比较
   ├─ 内容已存在：跳过
   └─ 内容不存在：追加到文件
   ↓
7. 查找所有下拉框
   ↓
8. 逐个选择下拉框选项
   ↓
9. 提取当前选项的内容
   ↓
10. 生成Markdown格式内容
   ↓
11. 与文件中已有内容比较
   ├─ 内容已存在：跳过
   └─ 内容不存在：追加到文件
```

```markdown
- 列表项2

| 表头1 | 表头2 |
| --- | --- |
```


### 去重逻辑

1. 点击Tab或选择下拉框选项
2. 提取页面内容（段落、列表、表格、代码块）
3. 使用`generateContentMarkdown()`生成标准化的Markdown文本

## 优势

- **避免重复数据**：通过文件内容比较，确保不会保存重复的内容
- **可靠性高**：直接比较文件内容比hash值更可靠
- **流式写入**：每个Tab/选项处理完立即写入，不占用过多内存
- **格式统一**：所有内容都以统一的Markdown格式保存

## 使用示例

爬虫会自动检测和处理Tab页和下拉框，无需额外配置。只需正常运行爬虫：

```bash
npm start
```

爬虫会自动：
1. 检测页面中的Tab和下拉框
2. 逐个点击/选择并提取内容
3. 智能去重并保存到Markdown文件

## 注意事项

- Tab按钮至少需要2个才会被识别为Tab组件
- 下拉框选项至少需要2个才会被处理
- 每次点击Tab或选择选项后会等待1秒让内容加载
- 内容比较时会标准化空格和换行，避免格式差异导致误判
