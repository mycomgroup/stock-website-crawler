# 全网网页解析器分类设计文档 (Web Page Classification Design)

## 设计背景
当前爬虫系统的网页分类（Portal, Article, Ecommerce, List Page, Search Result, Generic）粒度过粗，无法有效覆盖 Hao123 所收录的全网海量网站的结构差异。为了更精准地提取网页数据、生成高质量的 Markdown 或结构化 JSON 数据，本设计文档基于互联网网页结构化特征，梳理出 **12大类、120种细分页面类型**。

每种页面类型都具有独特的 DOM 结构、Meta 信息或特定的交互特征，系统可通过这些特征实现精准的自动化分类。

## 落地与演进方案建议
由于这 120 种类型的解析逻辑开发工作量巨大，建议：
1. **优先覆盖核心分类**：Hao123的流量大头集中在 `资讯媒体(News)`、`电商(E-commerce)` 和 `影音娱乐(Video)`。
2. **利用大模型辅助检测**：不再手写 120 个特征检测正则，而是提取页面关键 DOM 结构（如 H1, Table, Video, Form, meta 标签），交由轻量级分类模型自动映射到上述 120 种类型中。
3. **基于类型的统一抽取范式**：同大类的不同细分页面（如都是电商类）可以共享一套底层抽取逻辑（抽取 `Price`, `SKU`, `Image`），仅在 Markdown 渲染模板上做细分排版。
