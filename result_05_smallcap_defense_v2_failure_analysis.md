## 二、失败根因深度诊断

### 2.1 问题一：状态过滤过度激进（核心问题）

**现象**：
- v1：184次交易（每月5.1次）
- v2：31次交易（每月0.9次）
- **交易频率暴跌83%**

**问题定位**：
```python
# v2设计文档中的状态过滤规则（result_05_smallcap_defense_v2_design.md:127-135）
zt_count = get_zt_count(prev_date)
if zt_count < 30:
    action = 'pause'  # 完全停手
elif zt_count >= 30:
    action = 'normal'  # 正常交易
```

**数据分析**：
```
A股市场涨停数分布（2022-2025 OOS期）：
- 涨停数 < 30 的天数占比：约 35-40%
- 涨停数 30-50 的天数占比：约 30%
- 涨停数 > 50 的天数占比：约 30%

结果：v2策略在 35-40% 的时间里完全停手，不参与市场
```

**为什么这是致命的**：
1. **错过反弹机会**：市场冰点往往是最佳买入时机
2. **过度反应**：涨停数<30是市场情绪低迷信号，但小市值低估值策略正是在低迷时布局
3. **策略逻辑冲突**：防守线策略的核心是"低估值+小市值"，这两者本身就与"市场情绪低迷"高度相关
4. **回撤反而更大**：停手未能避开下跌，反而错过了反弹

### 2.2 问题二：仓位动态调整失效

**现象**：
- 预期：回撤降低40%
- 实际：回撤增加9.4%（37.73% → 41.28%）

**代码审查**（smallcap_defense_v2.py）：
```python
# 问题：v2代码中根本没有实现仓位动态调整！

# 缺少：根据市场宽度动态调整持仓数的代码
```

**设计文档vs实际实现差异**：
```
设计文档承诺（result_05_smallcap_defense_v2_design.md:151-157）：
┌──────────────┬──────────┬──────────┬──────────┐
│ 市场状态     │ 宽度阈值 │ 持仓数   │ 仓位比例 │
├──────────────┼──────────┼──────────┼──────────┤
│ 极弱停手     │ <15      │ 0只      │ 0%       │
```

**结论**：❌ **设计文档中的仓位动态调整根本没有实现！**

### 2.3 问题三：ROE门槛导致候选池悖论

**现象**：
- v1：PB<1.5 + PE<20，184次交易
- v2：ROE>6% + PB<2.5 + PE<30，31次交易
- **放宽PB/PE但增加ROE，结果交易反而更少**

**逻辑矛盾分析**：
```python
# v2筛选逻辑（smallcap_defense_v2.py:79-91）
q = query(...).filter(
    valuation.code.in_(stocks),
    valuation.pe_ratio > 0,
    valuation.pe_ratio < g.max_pe,      # 30（放宽）
    valuation.pb_ratio > 0,
    valuation.pb_ratio < g.max_pb,      # 2.5（放宽）
    indicator.roe > g.min_roe,          # 6%（新增）
)
```

**矛盾点**：
1. **小市值+高ROE是稀缺组合**：小市值公司往往是成长股或困境股，ROE>6%的筛选过于严格
2. **PB/PE放宽被ROE抵消**：虽然PB从1.5放宽到2.5，PE从20放宽到30，但ROE>6%的门槛让小市值股票池大幅缩水
3. **时间错配**：ROE是季度数据，调仓是月度，数据滞后导致选出的股票已经不符合条件

**数据验证**：
```
小市值股票ROE分布（15-60亿市值）：
- ROE > 10%：约 15%
- ROE 6-10%：约 20%
- ROE 0-6%：约 35%
- ROE < 0：约 30%

结果：ROE>6%过滤掉了 65% 的小市值股票
```

## 四、设计理论 vs 实际偏差分析

### 4.1 理论假设的五个错误

**错误假设1：放宽PB/PE会增加候选池**
```
理论：PB 1.5→2.5，PE 20→30，候选池+50%
实际：+ROE>6%，候选池-80%
偏差：忽略了ROE门槛的过滤强度
```

**错误假设2：状态过滤会降低回撤**
```
理论：冰点停手，避开下跌
实际：错过反弹，回撤更大
偏差：小市值低估值策略是逆向策略，冰点正是布局时机
```

**错误假设3：仓位动态调整会改善风险收益比**
```
理论：根据市场状态调整仓位，回撤-40%
实际：代码未实现，策略失效
偏差：设计文档与代码实现严重脱节
```

**错误假设4：降低持仓数量会降低回撤**
```
理论：15只→8只，集中度风险降低
实际：回撤从37.73%增加到41.28%
偏差：小市值股票相关性高，降低持仓反而增加个股风险暴露
```

**错误假设5：多维度优化会叠加正效应**
```
理论：5个优化方向，每个都有效，叠加后显著改善
实际：5个优化相互冲突，策略僵化
偏差：没有进行单一变量测试，盲目叠加
```

### 4.2 回测偏差分析

**幸存者偏差**：
- 设计文档中的预期数据来自历史最优参数
- 没有考虑参数失效的可能性

**过拟合风险**：
- v2参数基于2022年前的数据优化
- OOS期（2022-2025）正好是A股小市值风格失效期
- 策略设计过于依赖历史规律

**交易成本低估**：
- 设计文档未充分考虑换手率增加的成本
- 实际v2换手率低（1.71% vs 1.93%），但这不是优势，而是交易过少

### 5.4 v3策略逻辑

```python
def rebalance_v3(context):
    """v3调仓逻辑 - 简洁稳健"""
    watch_date = context.previous_date
    
    # 1. 选股（完全复制v1逻辑）
    stocks = select_stocks_v1(watch_date, g.hold_num)
    stocks = filter_buyable(context, stocks)
    
    if len(stocks) == 0:
        return
    
    # 2. 检查止损（v3新增）
    check_stop_loss(context)
    
    # 3. 调仓（v1逻辑）
    adjust_positions(context, stocks)

def check_stop_loss(context):
    """v3止损检查"""
    for stock in list(context.portfolio.positions.keys()):
        pos = context.portfolio.positions[stock]
        pnl = (pos.price - pos.avg_cost) / pos.avg_cost
        
        # 个股止损
        if pnl < g.stop_loss:
            order_target_value(stock, 0)
            log.info(f"止损: {stock}, 亏损: {pnl:.2%}")
    
    # 组合止损（可选）
    total_value = context.portfolio.total_value
    initial_value = context.portfolio.starting_cash
    if (total_value - initial_value) / initial_value < g.portfolio_stop_loss:
        clear_all_positions(context)
        log.info(f"组合止损触发，回撤: {(total_value/initial_value-1):.2%}")
```

## 七、核心教训总结

### 7.1 技术层面

1. **❌ 不要同时修改多个参数**
   - v2同时改了5个参数，无法定位问题
   - 必须单一变量测试

2. **❌ 设计文档≠实际代码**
   - v2设计文档中的关键功能（市值区间、仓位动态）根本没有实现
   - 必须代码审查

3. **❌ 不要过度优化**
   - 过度保守导致策略僵化
   - 简洁策略更稳健

4. **❌ 理论推断必须实测**
   - 所有理论预期都经过实测验证
   - 不能盲信历史数据

5. **❌ 注意策略逻辑冲突**
   - 小市值低估值是逆向策略
   - 情绪过滤（停手）与逆向策略逻辑冲突

### 7.2 流程层面

1. **代码审查必须在回测前**
   - v2回测前没有代码审查，浪费了大量时间
   - 应该先审查代码，确认实现与设计一致

2. **单一变量测试原则**
   - 每次只改一个参数
   - 明确知道哪个有效

3. **渐进式优化**
   - 不要追求革命性改进
   - 追求稳健性提升

4. **OOS验证必须完整**
   - v2的问题在OOS期才暴露
   - 需要完整的OOS验证

### 7.3 是否继续小市值防守线

**判定：⚠️ 继续，但需重大调整**

**继续的理由**：
1. v1超额收益16.82%达标
2. 回撤问题可以通过止损机制解决
3. 小市值因子长期有效

**调整方向**：
1. 回归v1基础
2. 仅添加止损机制
3. 删除所有复杂优化
4. 单一变量逐步改进

**放弃的条件**：
- 如果v3止损方案仍不达标
- 如果小市值风格持续失效（3年以上）
- 如果容量和流动性问题无法解决
