# 推荐基线版本

## 1. 推荐基线: B-v1.0

### 1.1 核心配置

| 组件 | 配置 | 理由 |
|------|------|------|
| **候选池** | v1.0 (12只 ETF) | 覆盖宽基/海外/行业/商品/固收, 低冗余 |
| **排序因子** | 20日简单收益率动量 | IC 最稳定, 半衰期匹配 |
| **持有周期** | 10日 | 匹配因子半衰期 (9-10天) |
| **持仓数量** | Top 3 | 平衡集中度和分散风险 |
| **择时** | 不开启 | 基线不叠加择时, 后续单独验证 |
| **交易成本** | 单边 0.1% | 万一手续费 + 滑点 |

### 1.2 Python 接口

```python
class ETFBaselineV1:
    """ETF 动量基线 v1.0"""
    
    # 候选池
    POOL = {
        "沪深300ETF":  "510300.XSHG",

        "医疗ETF":     "512170.XSHG",
        "消费ETF":     "159928.XSHE",
        "新能源ETF":   "516160.XSHG",
    }
    
    # 策略参数
    MOMENTUM_WINDOW = 20   # 动量计算窗口 (日)
    HOLD_DAYS = 10         # 持有周期 (日)
    TOP_N = 3              # 持仓数量
    COST = 0.001           # 单边交易成本
    
    @staticmethod
    def calc_momentum(prices: pd.DataFrame) -> pd.Series:
        """计算20日简单收益率动量"""
        return prices.iloc[-1] / prices.iloc[0] - 1
    
    @staticmethod
    def select_top(momentum: pd.Series, n: int = 3) -> list:
        """选取动量最高的 n 只 ETF"""
        return momentum.nlargest(n).index.tolist()
```

## 2. 为什么这个基线值得当母策略

### 2.1 足够简单

- 只有 2 个核心参数: 动量窗口 20日 + 持有周期 10日
- 不依赖复杂模型, 不需要训练
- 逻辑清晰, 易于解释

### 2.2 足够稳定

- 候选池覆盖 6 大类资产, 分散度高
- 20日动量 IC 在多份 notebook 中验证有效
- 10日持有匹配因子半衰期, 减少无效换手

### 2.3 可复用

- 候选池可直接用于后续择时任务
- 动量因子可作为排序层的基础信号
- 接口标准化, 便于叠加增强模块

### 2.4 成本可控

- 10日持有, 年化换手约 25 次
- 单边 0.1% 成本, 年化成本约 5%
- 净收益仍有空间
