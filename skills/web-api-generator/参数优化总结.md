### 2. 参数取值范围说明 ✅

**自动提取取值范围**：

| 情况 | 显示方式 | 示例 |
|------|---------|------|
| 样本值 ≤ 5 个 | 直接列出所有值 | `custom, my-followed, my-templates, public-templates, template-hot-and-latest` |
| 样本值 > 5 个（数字） | 显示数字范围 | `数字范围: 1-999999` |
| 样本值 > 5 个（有前缀） | 显示前缀模式 | `6888cb* (10+ 个值)` |
| 样本值 > 5 个（其他） | 显示数量 | `多个值 (50+ 个)` |

**实际示例**：

```markdown
# detail-lxr (指数详情页)
| indexCode | String | 指数代码 | 1000002, 1000004, 1000011, 1000003, 1000007 |

# user-companies (用户关注的公司列表)
| userId | String | 用户ID | 6888cb55020b3912bb08f1e6, 695f15db8b3b007457042cf0, ... |

# analytics-chart-maker (图表制作工具)
| chartType | String | 图表类型 | custom, my-followed, my-templates, public-templates, template-hot-and-latest |
```

### 3. 交互式补充参数信息 ✅

**工具**: `scripts/interactive-param-config.js`

**功能**：
- 列出所有需要配置的 API
- 显示每个参数的样本值
- 让用户输入准确的取值范围
- 自动备份原配置
- 支持选择性配置

**使用方法**：

```bash
# 启动交互式配置
node scripts/interactive-param-config.js

# 选择模式
选择模式: (1) 配置所有 (2) 选择性配置 [1/2]: 2

# 选择要配置的 API
可配置的 API 列表:
  1. detail-lxr - 指数详情页
  2. user-companies - 用户关注的公司列表
  3. analytics-chart-maker - 图表制作工具

输入要配置的 API 编号（用逗号分隔，如: 1,3,5）: 1,3

# 为每个参数输入取值范围
[1/2] API: detail-lxr
描述: 指数详情页
样本: https://www.lixinger.com/analytics/index/detail/lxr/1000002/1000002

  参数: indexCode
  说明: 指数代码
  样本值: 1000002, 1000004, 1000011, 1000003, 1000007
  取值范围 (直接回车跳过): 理杏仁指数代码，如 1000002(沪深300), 1000004(中证500)
  ✓ 已设置取值范围: 理杏仁指数代码，如 1000002(沪深300), 1000004(中证500)

# 保存配置
保存配置? [Y/n]: Y
✓ 已备份原配置到: api-configs.json.backup
✓ 已保存配置到: api-configs.json
✓ 已保存 JSONL 格式到: api-configs.jsonl
```

## 完整工作流程

### 方案 1: 自动生成（快速）

```bash
# 1. 生成文档和配置
node main.js generate-docs

# 2. 查看结果
cat output/web-api-docs/detail-lxr.md

# 3. 直接使用
node main.js call --api=detail-lxr --indexCode=1000002
```

### 方案 2: 交互式配置（精确）

```bash
# 1. 生成文档和配置
node main.js generate-docs

# 2. 交互式补充参数信息
node scripts/interactive-param-config.js

# 3. 查看更新后的文档
cat output/web-api-docs/detail-lxr.md

# 4. 使用
node main.js call --api=detail-lxr --indexCode=1000002
```

### 方案 3: 手动编辑（灵活）

```bash
# 1. 生成文档和配置
node main.js generate-docs

# 2. 手动编辑配置文件
vim output/web-api-docs/api-configs.json

# 修改 valueRange 字段：
{
  "name": "indexCode",
  "valueRange": "理杏仁指数代码，如 1000002(沪深300), 1000004(中证500)"
}

# 3. 重新生成文档
node main.js generate-docs

# 4. 使用
node main.js call --api=detail-lxr --indexCode=1000002
```

## 技术实现

### 核心方法

1. **extractValueRange()** - 提取参数取值范围
   - 从样本 URL 中提取参数值
   - 根据值的数量和类型决定显示方式
   - 支持数字范围、前缀模式、直接列举

2. **inferDefaultValue()** - 推断默认值
   - 根据参数名推断常见默认值
   - 如 page=1, limit=20, format=json

3. **generateMarkdown()** - 生成文档
   - 必选参数和可选参数分开显示
   - 必选参数显示取值范围
   - 可选参数显示默认值

### 配置文件格式

```json
{
  "api": "detail-lxr",
  "parameters": [
    {
      "name": "indexCode",
      "required": true,
      "type": "String",
      "description": "指数代码",
      "valueRange": "1000002, 1000004, 1000011, 1000003, 1000007"
    }
  ]
}
```
