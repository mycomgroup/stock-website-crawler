### 手动调整参数名称

如果自动推断的参数名不合适，有三种方式调整：

#### 方式 1: 直接编辑文档
```bash
vim temp/detail-lxr.md
# 修改参数名称和说明
```

#### 方式 2: 修改推断规则
编辑 `lib/doc-generator.js`，在 `inferParameterName()` 方法中添加规则：

```javascript
inferParameterName(pattern, rawName) {
  const path = pattern.pathTemplate.toLowerCase();
  
  // 添加自定义规则
  if (path.includes('/my-path/') && rawName === 'param5') {
    return 'myCustomName';
  }
  
  // ... 其他规则
}
```

然后重新生成：
```bash
node main.js generate-docs
```

#### 方式 3: 使用原始参数名
在调用 API 时直接使用原始参数名：
```bash
node main.js call --api=detail-lxr --param4=1000002
```

### 如何添加交互式命名

如果需要在生成时交互式确认参数名称，可以修改 `lib/doc-generator.js`：

```javascript
async generateAll(interactive = false) {
  for (const pattern of this.patterns) {
    const params = this.extractParameters(pattern);
    
    if (interactive) {
      // 提示用户确认参数名称
      for (const param of params) {
        const answer = await askUser(`参数 ${param.name} 的名称是否正确？`);
        if (!answer) {
          param.name = await askUser(`请输入正确的参数名称：`);
        }
      }
    }
    
    this.generateDoc(pattern, params);
  }
}
```
