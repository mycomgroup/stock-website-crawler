# URL Pattern Analyzer - 优化总结

## 完成的优化

### ✅ 优化1: 路径深度和段匹配严格化
**问题**: 不同结构的URL被错误归类
**解决**: 
- 要求路径深度必须完全相同
- 要求至少50%的路径段匹配
- 提高匹配段得分（10分 → 15分）

### ✅ 优化2: 半固定段智能细分
**问题**: 市场代码等半固定值未被识别和细分
**解决**:
- 识别唯一值数量有限的段（≤8个）
- 统计每个值的出现次数
- 只细分有足够大组的段（≥2个值出现≥10次）
- 添加3个可配置参数控制细分行为

### ✅ 优化3: 大簇严格细分
**问题**: 最大的簇可能仍混合不同模式
**解决**:
- 对前N个最大簇应用严格规则
- 计算每个段的固定比例
- 找出变化最大的段进行细分
- 只保留大小≥10的子簇
- 添加2个可配置参数控制严格模式

## 参数系统

### 基础参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--min-group-size` | 5 | 最小分组大小，过滤小簇 |
| `--sample-count` | 5 | 每个模式的示例URL数量 |
| `--max-patterns` | 无限制 | 最大模式数量（仅统计报告） |

### 细分控制参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--refine-max-values` | 8 | 半固定段最大唯一值数量 |
| `--refine-min-count` | 10 | 每个值最小出现次数 |
| `--refine-min-groups` | 2 | 最少需要几个大组才细分 |

### 严格模式参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--strict-top-n` | 0 | 对前N个最大簇应用严格规则（0=不启用） |
| `--strict-match-ratio` | 0.8 | 严格模式下的匹配比例阈值 |

## 调优建议

### 问题：模式太多（过度细分）
**症状**: 
- 模式数量 > 100
- 很多小簇（<20个URL）
- 相似模式重复

**解决方案**:
```bash
# 增加最小分组
--min-group-size 15

# 减少细分激进度
--refine-max-values 5
--refine-min-count 20

# 关闭严格模式
--strict-top-n 0
```

### 问题：模式太少（细分不足）
**症状**:
- 模式数量 < 20
- 大簇混合不同类型URL
- 覆盖率低

**解决方案**:
```bash
# 减少最小分组
--min-group-size 5

# 增加细分激进度
--refine-max-values 12
--refine-min-count 5

# 启用严格模式
--strict-top-n 10
--strict-match-ratio 0.9
```

### 问题：大簇仍然混合
**症状**:
- 最大簇 > 1000个URL
- 样本URL结构明显不同
- 路径模板过于泛化

**解决方案**:
```bash
# 启用严格模式
--strict-top-n 5
--strict-match-ratio 0.85

# 同时增加细分激进度
--refine-max-values 10
--refine-min-count 8
```

## 算法原理

### 1. 相似度计算
```
score = 0

IF 协议或主机不同: RETURN 0
IF 路径深度不同: RETURN 0

score += 20  # 路径深度相同

FOR 每个路径段:
  IF 段值相同:
    score += 15

IF 匹配段数 < 总段数 * 50%: RETURN 0

FOR 每个查询参数:
  IF 参数键相同:
    score += 5

RETURN score
```

### 2. 半固定段识别
```
FOR 每个路径段位置:
  统计唯一值数量
  统计每个值的出现次数
  
  IF 唯一值数量 > 1 AND 唯一值数量 <= refineMaxValues:
    大组数量 = COUNT(出现次数 >= refineMinCount)
    
    IF 大组数量 >= refineMinGroups:
      标记为半固定段
      按值分组
```

### 3. 严格细分
```
FOR 前strictTopN个最大簇:
  FOR 每个路径段:
    计算固定比例 = 1 - (唯一值数-1) / URL总数
  
  计算平均固定比例
  
  IF 平均固定比例 < strictMatchRatio:
    找出变化最大的段
    按该段的值分组
    只保留大小 >= 10 的子簇
```
