# URL Pattern Analyzer - 算法优化记录

## 优化历程

### 优化1: 路径深度和段匹配要求（2026-02-26）

**问题**: 不同层级和结构的URL被归为一类
- `/analytics/index/detail/...` 和 `/analytics/fund/detail/...` 被混在一起

**解决方案**:
1. 要求路径深度必须完全相同
2. 要求至少50%的路径段匹配
3. 每个匹配段得分从10分提高到15分

**效果**: 成功分离了不同结构的URL

---

### 优化2: 半固定段细分（2026-02-26）

**问题**: 某些变化段只有少数几个固定值（如市场代码：hk, sh, sz），应该进一步细分

**解决方案**:
1. 识别"半固定段"：唯一值数量 > 1 且 ≤ N（可配置）
2. 统计每个值的出现次数
3. 如果至少有M个值出现次数 ≥ K，则按这些值细分
4. 添加可配置参数：
   - `refineMaxValues`: 半固定段最大唯一值数量（默认8）
   - `refineMinCount`: 每个值最小出现次数（默认10）
   - `refineMinGroups`: 最少需要几个大组才细分（默认2）

**效果**: 
- 成功按市场代码分离：csi, sh, sz, nasdaq等
- 从38个模式增加到70+个模式
- 更精确的分类

---

### 优化3: 大簇严格细分（待实现）

**问题**: 最大的簇可能仍然混合了不同的模式

**解决方案**:
对于URL数量最多的前N个簇，应用更严格的规则：
1. 要求每个路径段要么全部固定，要么只有一个变量
2. 不允许一个段内有多个不同的值混合
3. 添加可配置参数：
   - `strictTopN`: 对前N个最大簇应用严格规则（默认5）
   - `strictMatchRatio`: 严格模式下的匹配比例（默认0.8，即80%段必须完全匹配）

**预期效果**:
- 进一步细分大簇
- 更精确的模式识别
- 避免"万能模式"（匹配太多不同类型的URL）

## 参数调优指南

### 基础参数

| 参数 | 默认值 | 说明 | 调优建议 |
|------|--------|------|----------|
| `minGroupSize` | 5 | 最小分组大小 | 增大=更少模式，减小=更多模式 |
| `sampleCount` | 5 | 示例URL数量 | 纯展示参数，不影响聚类 |

### 细分控制参数

| 参数 | 默认值 | 说明 | 调优建议 |
|------|--------|------|----------|
| `refineMaxValues` | 8 | 半固定段最大唯一值数量 | 增大=更激进细分，减小=更保守细分 |
| `refineMinCount` | 10 | 每个值最小出现次数 | 增大=只细分大组，减小=也细分小组 |
| `refineMinGroups` | 2 | 最少需要几个大组才细分 | 增大=更难触发细分，减小=更易触发细分 |

### 严格模式参数（待实现）

| 参数 | 默认值 | 说明 | 调优建议 |
|------|--------|------|----------|
| `strictTopN` | 5 | 对前N个最大簇应用严格规则 | 增大=更多簇被严格处理 |
| `strictMatchRatio` | 0.8 | 严格模式下的匹配比例 | 增大=更严格，减小=更宽松 |


## 算法原理

### 相似度计算

```
score = 0

1. 协议和主机必须相同，否则 score = 0
2. 路径深度必须相同，否则 score = 0
3. 路径深度相同: score += 20
4. 每个匹配的路径段: score += 15
5. 如果匹配段数 < 总段数 * 50%: score = 0
6. 每个匹配的查询参数: score += 5
```

### 聚类策略

```
1. 按URL长度排序（短的优先）
2. 对每个URL:
   - 尝试加入现有簇（相似度 > 0）
   - 如果无法加入，创建新簇
3. 后处理：细分包含"半固定段"的簇
```

### 半固定段识别

```
对于每个簇的每个路径段位置:
1. 统计该位置的唯一值数量
2. 统计每个值的出现次数
3. 如果:
   - 唯一值数量 > 1 且 ≤ refineMaxValues
   - 至少有 refineMinGroups 个值的出现次数 ≥ refineMinCount
   则该段是"半固定段"，可以细分
```

## 已知限制

1. **性能**: 对于超大规模URL集（>100k），可能需要优化
2. **查询参数**: 当前只识别参数键，不分析参数值
3. **片段标识符**: 忽略URL中的 # 片段
4. **编码**: 假设URL已正确编码
