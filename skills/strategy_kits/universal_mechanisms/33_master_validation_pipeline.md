# 主仓统一验证指标框架

## 目标

这份文档定义“任何新策略完成初版验证后，必须统一补跑一次增强版验证”的**统一报告 contract**。

核心目的只有三个：

1. 让不同策略输出同一套指标，能够横向比较
2. 让“原策略”和“增强版策略”形成一一对应的对照关系
3. 让失败时能快速定位应回改哪一层

> 以后凡是进入 `32_master_portfolio_assembly.md` 的策略，都必须按本文档输出验证结果。

---

## 适用范围

默认适用于三类：

- 股票低频主仓
- ETF / 多资产主仓
- 微盘 / 小市值条件分支

不直接适用于：

- 纯日内
- 高频
- 盘口/竞价超短

这类策略可以参考指标结构，但不能直接复用主仓门槛。

---

## 必须输出的三层报告

### 第一层：原策略基线报告

回答：
- 不做统一增强时，这条策略本身到底如何？
- 它到底是可用 alpha，还是只是一个研究想法？

### 第二层：增强版对照报告

回答：
- 接入 `32` 装配图后，收益、回撤、交易效率是否改善？
- 它更适合做主仓、过滤器，还是仅做辅助模块？

### 第三层：稳定性与归因报告

回答：
- 结果是否只来自少数年份或单一风格暴露？
- 样本外是否保留有效性？
- 数据代理偏差是否影响结论？

---

## 必要指标

### 1. 收益类

| 指标 | 说明 |
|------|------|
| 累计收益 | 全周期最终收益 |
| 年化收益 | 主收益指标 |
| Alpha | 相对基准的超额收益 |
| 超额收益 | 相对主基准的净超额 |

### 2. 风险类

| 指标 | 说明 |
|------|------|
| 最大回撤 | 必看 |
| 年化波动率 | 风险暴露水平 |
| Sharpe | 风险调整收益 |
| Calmar | 主仓优先级高于单看 Sharpe |
| Sortino | 下行风险质量 |
| 回撤恢复期 | 判断修复能力 |

### 3. 交易效率类

| 指标 | 说明 |
|------|------|
| 胜率 | 周期/交易维度都应统计 |
| 交易次数 | 过高通常意味着噪声大 |
| 平均持有期 | 判断是否偏离低频主仓定位 |
| 换手率 | 必须年化或周期化 |
| 盈亏比 | 防止高胜率低质量 |

### 4. 成本类

| 指标 | 说明 |
|------|------|
| 费前收益 | 便于研究模块原始强度 |
| 费后收益 | 实际比较口径 |
| 手续费后 Alpha | 过滤伪优势 |
| 滑点敏感性 | 标准成本 vs 压力成本 |

### 5. 稳定性类

| 指标 | 说明 |
|------|------|
| 样本内 / 样本外表现 | 不能只看总表 |
| 年度分解 | 识别是否只赚某几年 |
| 市场状态分解 | 牛市/震荡/下跌环境表现 |
| 最近 1-2 年表现 | 防止旧样本幻觉 |
| 滚动窗口结果 | 观察衰退和参数脆弱性 |

### 6. 归因类

| 指标 | 说明 |
|------|------|
| 市值归因 | 是否只是小盘暴露 |
| 行业归因 | 是否押中单一行业 |
| 风格归因 | 价值/质量/动量/规模 |
| 特异收益 | 是否真的有策略自身贡献 |

---

## 强制对照矩阵

任何策略至少要输出下面 4 版：

| 版本 | 目的 |
|------|------|
| `V0 原策略` | 原始策略，不接统一增强 |
| `V1 原策略 + 数据/成本统一口径` | 排除回测写法差异 |
| `V2 原策略 + 主仓装配增强` | 接入路由/确认/仓位/退出 |
| `V3 原策略 + 完整执行底座` | 接入目标权重与统一执行 |

如果策略本身就是组合工程策略，则至少保留：
- `基线版`
- `增强版`
- `压力成本版`
- `样本外版`

|------|------|------|------|------|------|------|------|------|

| V1 统一口径 |  |  |  |  |  |  |  |  |
| V2 装配增强 |  |  |  |  |  |  |  |  |
| V3 完整底座 |  |  |  |  |  |  |  |  |

### B. 稳定性分解

至少包含：
- 年度收益表
- 年度回撤表
- 市场状态表现表
- 样本内 / 样本外对照
- 压力成本对照

### C. 归因分解

至少回答：
- 是什么因子/风格在赚钱？
- 回撤主要来自哪类暴露？
- 增强版改善的是收益、回撤，还是只是降低交易频率？

## 诊断到模块的回改映射

| 现象 | 优先回改层 |
|------|------|
| 收益不足 | `Alpha层` |
| 回撤过大 | `03主路由` / `10仓位缩放` / `17退出层` |
| 交易过多 | `30确认层` / `37双确认与再进场` |
| 换手过高 | `34权重映射` / 调仓频率 |
| 费后崩塌 | `36成本口径` / 执行层 |
| 样本外失效 | 数据代理偏差 / 过拟合 / 路由过度复杂 |
| 只在少数年份有效 | 归因与风格暴露过于单一 |

## 默认通过标准

本文档不主张只靠单一数值阈值裁决，但至少要满足以下方向：

### 主仓候选
- 费后结果可接受
- 相比原策略，增强版没有明显破坏收益质量
- 回撤、Calmar、换手率至少有一项明确改善
- 样本外没有出现明显坍塌
- 已能输出目标权重与统一报告

### 过滤器候选
- 单独跑不一定强
- 但叠加后能改善 `回撤 / Calmar / 换手 / 胜率` 中至少一项
- 且不显著拖累长期收益

### 研究储备
- 有局部亮点
- 但证据不足，或只在少数阶段有效

### 淘汰
- 费后无优势
- 增强后仍不能改善关键指标
- 或严重依赖异常样本/代理数据

具体分档执行，见 `38_strategy_admission_oos.md`。

## 结论

如果 `32` 解决的是“流程怎么接”，那么本文解决的是：

- **结果怎么报**
- **增强前后怎么比**
- **失败后先改哪里**

以后任何新策略，不允许只给一张收益曲线或单个 Sharpe，就直接判断“有效”。
