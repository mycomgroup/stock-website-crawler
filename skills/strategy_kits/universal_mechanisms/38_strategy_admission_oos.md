# 策略准入、淘汰与 OOS 接续标准

## 目标

这份文档定义：

1. 一条策略在统一增强回放后，如何定档
2. 哪些策略可以进入主仓候选
3. 哪些策略只适合做过滤器/辅助模块
4. 哪些策略必须进入 OOS 持续观察
5. 哪些策略应直接淘汰

它是 `33~37` 的最终落点。

---

## 适用前提

只有完成以下动作后，才允许按本文定档：

- 已经过 `32_master_portfolio_assembly.md`
- 已按 `36_data_benchmark_cost_spec.md` 统一口径
- 已按 `35_enhancement_replay_checklist.md` 补跑增强流程
- 已按 `33_master_validation_pipeline.md` 输出统一报告

否则不能直接宣布“通过”。

## 五级定档

### A 档：主仓候选

满足大多数条件：
- 费后结果可接受
- 增强版相对基线有清晰提升
- 回撤、Calmar、换手率、稳定性至少有两项改善
- 样本外未明显坍塌
- 已形成标准目标权重表
- 已能接入统一执行底座

适用：
- 股票低频主仓
- ETF / 多资产主仓

### B 档：过滤器 / 确认器候选

特点：
- 单独跑不一定强
- 但叠加到主仓后，能改善回撤、Calmar、胜率或换手率
- 且不会显著破坏长期收益

典型对象：
- `QRS`
- 鳄鱼线
- 北向资金分位
- 再进场确认模块

### C 档：战略辅助模块

特点：
- 日常默认不启用
- 但在极端环境有高价值

典型对象：
- 宏观底部信号
- 估值极端开关
- 特殊风险事件过滤

### D 档：OOS 观察池

特点：
- 有一定亮点
- 但证据不足，或样本外不够稳定
- 不宜立即纳入主仓或默认过滤器

处理方式：
- 进入滚动 OOS 观察
- 定期更新表现
- 观察是否继续衰退

### E 档：淘汰 / 挂起

特点：
- 费后无优势
- 增强后仍无法改善关键指标
- 高度依赖单一窗口、代理数据或参数偶然性

处理方式：
- 暂不继续投入
- 仅保留研究记录

## 默认裁决逻辑

### 先看“增强后角色”而不是“原始标题”

同一模块可能有不同归宿：

- 有些原本看起来像主策略，最后只适合做过滤器
- 有些原本只是因子，接入底座后反而能成为主仓 alpha

因此裁决顺序应为：

```text
原策略
→ 增强回放
→ 看它在主仓体系中的最终角色
```

---

## 主仓候选的最低要求

默认至少回答清楚：

1. 它的 `Alpha层` 是什么？
2. 它的 `主路由` 是什么？
3. 它的 `确认层` 是什么？
4. 它的 `目标权重` 怎么来？
5. 它的 `费后结果` 是否可接受？
6. 它的 `样本外` 是否保留结构性有效？

任何一项答不清楚，都不建议进入 A 档。

---

## 过滤器候选的最低要求

过滤器不要求单独最强，但必须证明：

- 能改善 `回撤 / Calmar / 胜率 / 换手率` 中至少一项
- 对长期收益拖累可控
- 逻辑可以解释
- 不是仅靠单年特殊行情

这条标准对 `30_signalmaker_filters.md` 尤其重要。

---

## OOS 接续标准

### 哪些策略必须进入 OOS

以下情况建议进入 OOS 观察池：

1. 增强后看起来可用，但样本外还短
2. 收益不错，但成本/滑点敏感
3. 结果明显依赖某类风格行情
4. 数据代理误差仍未完全消除

### OOS 观察重点

至少跟踪：
- 最近 1/3/6/12 个月表现
- 费后收益
- 回撤变化
- 胜率/换手变化
- 与基线策略的相对表现

### 预警信号

若出现以下情形，优先降级：
- OOS 长期显著弱于样本内
- 成本稍上调就失效
- 回撤恶化明显
- 归因显示收益完全变形

`strategies/secondboard_oos_system` 可以作为 OOS 自动化监控的参考原型。

## 结论

以后仓库里的每条策略，最终都不该停留在“回测了一下”。

而是要被明确归入以下之一：

- **主仓候选**
- **过滤器候选**
- **战略辅助模块**
- **OOS 观察池**
- **淘汰 / 挂起**

只有完成这一步，整个 `universal_mechanisms` 才真正具备“验证 → 增强 → 定档 → 复用”的闭环。
