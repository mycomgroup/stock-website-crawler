# SignalMaker 过滤器模块库

## 概述

`QuantsPlaybook/SignalMaker` 已经不是单纯的研报示例集合，而是一组可复用的择时信号构造模块。

对“低频高胜率/高确定性”体系来说，它更适合定位为：

- **过滤器层**：决定是否允许开仓
- **确认层**：决定是否从警示升级为正式信号
- **投票层**：多个弱信号做组合决策

而不是直接把每个子模块都当作独立主策略。

---

## 模块构成

### 1. QRS：噪声抑制型趋势过滤

核心做法不是简单斜率，而是把 `zscore_beta` 与 `corr^n` 惩罚项相乘：

- 趋势强，但线性一致性差 → 信号被压低
- 趋势强，且回归相关性高 → 信号更可信

适合用途：
- 趋势策略的二次确认
- 低频主仓的减仓/空仓过滤
- 与 `03_state_router` 配合，减少单日噪声切换

### 2. 鳄鱼线 / AO / MACD：趋势确认组

这组模块适合做“趋势是否真正展开”的确认，而不是单独承担全部择时职责。

建议用法：
- 鳄鱼线：判定趋势方向
- AO：确认动量扩张/收缩
- MACD：补充趋势斜率变化

### 3. 北向资金分位信号：外部资金确认

模块里已经有统一的分位数实现：
- 60日滚动 `0.8` 分位以上 → 看多
- 60日滚动 `0.2` 分位以下 → 看空

它更适合做：
- 状态路由的修正项
- 低频主仓的加减仓开关
- ETF/指数增强的风险偏好辅助因子

### 4. HHT / NoiseArea：研究原型层

这类模块更适合保留为：
- 研究储备
- 组合投票候选
- 特定市场环境的补充信号

不建议直接视作主仓级核心开关。

---

## 为什么它应归类为“过滤器模块库”

`strategies/quantsplaybook_validation` 的10年验证结果说明：

- `QRS` 单独做主策略表现一般
- `鳄鱼线` 接近零夏普
- `北向资金` 单独做择时也不够稳

因此更合理的结论是：

1. **这些模块单独拿出来，不一定形成强主策略**
2. **但作为过滤器/确认器/投票器，工程价值很高**
3. **真正有效的，是“主仓底座 + 多过滤器”组合结构**

## 推荐接法

### 接法 A：状态路由器的二级确认层

```text
广度/情绪主路由
    ↓
SignalMaker 二级确认（QRS / 鳄鱼线 / 北向）
    ↓
确认通过才允许升档或恢复仓位
```

适合：
- `RFScore`
- `动态对冲股债`
- `指数增强`

### 接法 B：投票式过滤器

```text
QRS + 鳄鱼线 + AO/MACD + 北向资金
→ 每个模块输出 -1 / 0 / 1
→ 合成投票分数
→ 只影响仓位档位，不直接替代主策略
```

建议规则：
- 投票 ≥ 2：允许升档
- 投票 ≤ -2：降档或暂停
- 其余：维持原状态

### 接法 C：再进场确认层

在止损或停手后：
- 先由 `03_state_router` 判断市场是否修复
- 再由 `SignalMaker` 模块确认趋势是否恢复
- 最后再恢复正常仓位

这样更符合“少出手，但只做更确定的再进入场”。

## 最值得继续整编的子模块

1. **QRS 惩罚项过滤**：作为趋势噪声抑制器
2. **鳄鱼线 + AO + MACD 组合确认**：作为趋势展开确认组
3. **北向资金分位信号**：作为风险偏好辅助项
4. **SignalMaker 统一投票接口**：把多个子模块标准化成统一输出

---

## 结论

`SignalMaker` 的价值不在于“找到一个新的神级单策略”，而在于：

- 把多个中等强度信号标准化
- 让它们服务于状态路由、再进场确认和仓位升降档
- 帮低频主仓提高确定性，而不是单独追求高频博弈
