# 验证后增强回放清单

## 目标

这份文档定义“一个新策略完成初版验证后，必须再走一遍统一增强流程”的**强制回放清单**。

它对应你的最终目标：

> **新策略不是验证完就结束，而是要自动进入统一增强回放，再跑一遍，看看它在主仓体系里到底能提升到什么位置。**

---

## 适用时点

在以下时点必须触发本文：

1. 新策略完成第一版回测
2. 旧策略完成大改版
3. 新 alpha、新过滤器、新权重模块加入仓库
4. 数据口径或成本模型发生变化

---

## 回放前提

进入回放前，至少要具备：

- 原策略基线结果
- 策略分型结论
- 基础候选池或资产池定义
- 可复现的回测代码或研究脚本

如果这些都没有，说明还不到进入统一增强阶段的时候。

---

## 统一增强回放总流程

```text
原策略初版验证完成
        ↓
按 32 做策略分型
        ↓
按 36 统一数据 / 基准 / 成本口径
        ↓
按 37 补确认层 / 双确认 / 再进场
        ↓
按 34 生成目标权重表
        ↓
按 31 放入统一执行底座
        ↓
按 33 输出增强前后对照报告
        ↓
按 38 判定：主仓 / 过滤器 / OOS / 淘汰
```

## 强制回放的 8 个动作

### 1. 先分型，不允许盲目增强

先判断它属于：
- 股票低频主仓
- ETF / 多资产主仓
- 微盘 / 小市值分支
- 事件驱动 / 机会仓

只有前两类默认走全套主仓增强流程。

### 2. 锁定统一口径

必须先固定：
- 基准
- 费用
- 滑点
- 真实价格
- 是否规避未来函数
- 样本内 / 样本外切分

规则见 `36_data_benchmark_cost_spec.md`。

### 3. 映射到装配图

明确回答：
- 这条策略的 `Alpha层` 是什么？
- 它的 `主路由` 是什么？
- 它的 `二级确认` 是什么？
- 它的 `仓位缩放` 是什么？
- 它的 `退出层` 是什么？

如果答不出来，就不能进入增强回放。

### 4. 补主路由与确认层

默认检查是否需要补：
- `03_state_router.md`
- `08_rsrs_timing.md`
- `30_signalmaker_filters.md`
- `37_signal_confirmation_interface.md`

重点不是强行挂满，而是看：
- 是否能减少误交易
- 是否能降低回撤
- 是否能避免过早再进场

### 5. 把信号变成目标权重

不允许只输出：
- 买入名单
- 看多/看空标签
- 手工持仓比例

必须按 `34_alpha_weight_mapping.md` 生成：
- 候选评分表
- 目标权重表

### 6. 放入统一执行底座

必须至少做一版：
- `31_index_enhancement_base.md` 统一执行

这样才能排除：
- 执行写法差异
- 调仓频率差异
- 成本设置差异

### 7. 输出增强前后对照报告

必须输出至少 4 版：
- `V0 原策略`
- `V1 统一口径`
- `V2 装配增强`
- `V3 完整底座`

指标结构见 `33_master_validation_pipeline.md`。

### 8. 给出最终归类

最后不能只写“效果不错”。

必须归类到以下之一：
- 主仓候选
- 过滤器候选
- 战略辅助模块
- 研究储备
- 淘汰

归类规则见 `38_strategy_admission_oos.md`。

## 默认实验矩阵

建议最少补跑以下实验：

| 版本 | 必跑 | 目的 |
|------|------|------|
| 原策略基线 | 是 | 看原始强度 |
| 统一口径版 | 是 | 去掉回测框架差异 |
| `+主路由` | 是 | 看回撤与空仓控制 |
| `+确认层/再进场` | 是 | 看交易效率与误判减少 |
| `+权重映射` | 是 | 看组合表达是否稳定 |
| `+统一底座` | 是 | 看工程化后的真实可比结果 |
| `+压力成本` | 强烈建议 | 看实盘脆弱性 |
| `+样本外` | 强烈建议 | 看是否可持续 |

---

## 回放后必须回答的 6 个问题

1. 增强版到底提升了什么：收益、回撤、Calmar、换手，还是稳定性？
2. 这条策略更像 `主仓 alpha`，还是 `过滤器模块`？
3. 它最敏感的是参数、成本，还是数据代理？
4. 它的有效期是长期，还是只在局部年份成立？
5. 如果失败，最该优先回改哪一层？
6. 是否值得进入 OOS 持续跟踪？

## 常见结论模板

### 结论 A：适合升级为主仓候选

表现：
- 原策略已有一定强度
- 装配增强后回撤、Calmar、交易效率明显改善
- 能稳定输出目标权重
- OOS 不崩

### 结论 B：更适合作为过滤器/确认器

表现：
- 单独跑一般
- 但接入其他主仓后能改善回撤、换手、胜率

典型对象：
- `QRS`
- `鳄鱼线`
- `北向资金分位`

### 结论 C：保留为战略辅助

表现：
- 信号不适合日常频繁使用
- 但在极端环境有价值

典型对象：
- `20/21/22/23` 宏观底部类

### 结论 D：淘汰或挂起

表现：
- 增强后仍无优势
- 高度依赖单一窗口或代理数据
- 费后结果失真

## 最终交付物

每次增强回放，建议最少留下：

1. 一份增强前后对照结果表
2. 一份目标权重表样例
3. 一份归因摘要
4. 一份最终归类结论
5. 一份下一步动作建议

## 结论

以后“策略验证完成”不再代表研究结束。

真正的完成标准应该是：

- **原策略跑过**
- **统一增强再跑过**
- **增强前后差异可解释**
- **最终角色已归类**

只有这样，仓库里的策略才会越来越像一个可复用的主仓工厂，而不是一堆彼此孤立的回测结果。
