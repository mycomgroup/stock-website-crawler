# 指数增强主仓底座

## 概述

`QuantsPlaybook/B-因子构建类/多因子指数增强/回测模块.py` 提供了一套非常像“主仓工程底座”的实现。

它的重要性不在于某个单一因子，而在于把以下事情做成了统一框架：

- 月频调仓
- 基准对齐
- 避免未来函数
- 真实价格回测
- 手续费/滑点统一设置
- 读取目标权重并执行下单

这比“单个择时信号 notebook”更接近真正可复用的低频主仓架构。

---

## 为什么它比单一择时更像主仓底座

主仓底座最重要的不是某一个神奇信号，而是：

1. **研究层与执行层分离**
2. **权重表达统一**
3. **调仓频率稳定**
4. **成本模型清晰**
5. **能方便挂接多个 alpha 与过滤器模块**

指数增强底座天然符合这五条。

---

## 核心结构

### 1. 调度层
- 月频执行
- 参考基准为沪深300
- 适合低换手、可审计的主仓组合

### 2. 执行层
- 从研究结果文件读取目标权重
- 自动卖出不在目标权重中的持仓
- 按组合总资产执行 `order_target_value`

### 3. 回测层
- `avoid_future_data`
- `use_real_price`
- 分时期手续费设置
- 适合作为不同主仓方案的统一对比底座

---

## 在“低频高胜率”体系中的定位

### 适合作为主仓的原因

- **低频**：月度再平衡天然降低噪声交易
- **可控**：可以强制保留现金、限制单票权重、限制行业暴露
- **可拼装**：可接 `FFScore`、红利质量、RFScore、状态路由等模块
- **可扩展**：后续可以接 `EPO`、风险平价、对冲层

### 不足

- 它本身不是 alpha 来源
- 如果只做“照单下单”，并不会自动变成高胜率策略
- 仍然需要上层信号、过滤器与风险管理共同驱动

---

## 推荐架构

```text
基础股票池
    ↓
Alpha层（FFScore / RFScore / 红利质量 / 价值质量）
    ↓
过滤层（状态路由 / RSRS / SignalMaker / 北向 / 广度）
    ↓
权重层（等权 / EPO / 风险预算 / 现金缓冲）
    ↓
指数增强底座（月频执行 + 成本设置 + 目标权重下单）
```

这条链路很适合你当前追求的方向：

- **少调仓**
- **强过滤**
- **先确认再恢复仓位**
- **主仓以组合工程稳定性为先**

---

## 最推荐的三种接法

### 接法 A：FFScore / 红利质量指数增强

适合：
- 价值质量主仓
- 中大市值低频组合

组合方式：
- `24_fscore_selection`
- `28_dividend_quality_filter`
- `03_state_router`
- `31_index_enhancement_base`

### 接法 B：RFScore / PB 低估值主仓

适合：
- 股票纯度更高的月频主仓

组合方式：
- `RFScore + PB` 生成候选
- `03_state_router` 决定持仓档位
- `30_signalmaker_filters` 做二级确认
- 指数增强底座执行目标权重

### 接法 C：股债混合增强

适合：
- 更强调回撤控制的主仓

组合方式：
- 股票增强层输出权益目标权重
- 债券/黄金等防守资产补足剩余权重
- `25_epo_portfolio` 或 `27_es_risk_parity` 处理组合权重
- 指数增强底座统一执行

## 结论

如果说 `SignalMaker` 解决的是“怎么把过滤器模块化”，那么指数增强底座解决的是“怎么把主仓工程化”。

在你当前这条研究线上，它应该被正式提升为：

- **主仓底座候选**
- **低频组合执行框架**
- **后续组合化研究的统一承载层**
