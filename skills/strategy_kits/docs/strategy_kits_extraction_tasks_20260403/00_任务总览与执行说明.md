## 公共硬约束

所有子 agent 都必须遵守：

1. 目标不是“总结两个仓库”，而是**抽成能用的 kit**。
2. 每个任务都必须同时参考：
   - `/Users/fengzhi/Downloads/git/testlixingren/聚宽有价值策略558`
   - `/Users/fengzhi/Downloads/git/testlixingren/QuantsPlaybook`
3. 主结果必须写入本目录下的 `result_XX_*.md`。
4. 简版回执必须写入：`dispatch_prompts_20260403/results/`。
5. 只要边界足够清晰，就允许直接在 `strategy_kits/` 对应目录里创建最小骨架文件或 contract 文件。
6. 不允许把具体策略参数答案、具体买卖时间点、具体标的池名单，直接硬抽成公共默认值。
7. 必须明确写出：
   - 来源文件
   - 抽取后归宿
   - 最小输入
   - 最小输出
   - 不该负责什么
8. 如果发现某项更适合保留在最终策略层，必须明确写“不要抽”。

## 建议子 agent 的工作方式

每个子 agent 优先做：

1. 找源文件
2. 判边界
3. 给接口
4. 如边界清晰则补骨架
5. 写结果文档与回执

不要花大量时间写长篇背景综述。

主目标只有一个：

**让一个新策略能更少复制粘贴、更少重写底座、更快进入回测。**
