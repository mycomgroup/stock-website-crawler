# V0/V1/V2/V3 基线矩阵规范

**版本**: 1.0  
**日期**: 2026-04-03  
**状态**: 可直接用于矩阵 runner 实现

---

## 1. 版本一句话定义

| 版本 | 定义 | 核心目的 |
|------|------|----------|
| **V0 原策略** | 原始策略代码，不做任何统一增强 | 测量策略本身的原始强度 |
| **V1 统一口径** | V0 + 统一数据/基准/成本口径 | 排除回测框架差异，获得可比基线 |
| **V2 装配增强** | V1 + 主仓装配层（路由/确认/仓位/退出） | 验证装配层对策略的改善效果 |
| **V3 完整底座** | V2 + 目标权重生成 + 统一执行底座 | 工程化后的真实可比结果 |

## 2. 版本边界精确定义

### 2.1 V0 与 V1 的边界

**V0 保留（策略原貌）**:
- 策略原始信号生成逻辑
- 策略原始参数设置
- 策略原始持仓逻辑
- 策略原始选股范围

**V1 强制统一（排除干扰）**:

| 维度 | V0 可能变化 | V1 必须固定 |
|------|-------------|-------------|
| **数据口径** | 不同数据源、不同复权方式 | 统一数据源、统一复权 |
| **基准** | 可能无基准或口头基准 | 显式声明主基准（000300/000905/000852） |
| **成本** | 零成本或自定义成本 | 标准成本：佣金0.03% + 滑点0.10% |
| **样本切分** | 可能只看全样本 | 必须区分样本内/样本外 |
| **价格执行** | 可能用收盘价假设成交 | 必须用真实价格 |
| **未来函数** | 可能存在隐蔽未来函数 | 必须显式规避 |

**核心原则**: V1 只改"回测写法"，不改"策略逻辑"

### 2.2 V1 与 V2 的边界

**V2 相对 V1 最少新增的增强层**:

| 增强层 | 是否必须 | 说明 |
|--------|----------|------|
| **主路由 (03_state_router)** | 是 | 市场状态过滤，减少逆势交易 |
| **二级确认 (30_signalmaker_filters)** | 是 | 信号质量确认，降低误判 |
| **双确认/再进场 (37_signal_confirmation)** | 条件 | 高换手策略必须，低频可选 |
| **仓位缩放 (10_position_sizing)** | 是 | 基于波动率或信号的仓位调整 |
| **退出层 (17_exit_layer)** | 是 | 止损/止盈/持有期退出机制 |

**V2 不改变**:
- Alpha 信号生成核心逻辑
- 原始选股因子和权重
- 基础候选池定义

### 2.3 V2 与 V3 的边界

**V3 何时必须接完整执行底座**:

| 条件 | 是否必须 V3 |
|------|-------------|
| 策略目标是成为主仓候选 | 是 |
| 需要与其他策略做横向比较 | 是 |
| 需要输出目标权重供组合使用 | 是 |
| 需要进入 32_master_portfolio_assembly | 是 |
| 仅作为过滤器/确认器验证 | 可选 |
| 快速初筛阶段 | 否 |

**V3 相对 V2 新增**:

| 模块 | 功能 | 输出物 |
|------|------|--------|
| **目标权重生成 (34_alpha_weight_mapping)** | 信号→候选评分→目标权重 | 目标权重表 |
| **统一执行底座 (31_index_enhancement_base)** | 标准化调仓、成交、成本处理 | 标准化回测结果 |
| **完整报告输出** | 统一格式指标 | 对照报告 |

## 4. 参数冻结规则

### 4.1 版本间只允许变化的参数

以下参数在不同版本间**可以**变化:

| 参数类别 | 具体参数 | 变化范围 |
|----------|----------|----------|
| **装配层开关** | 主路由启用、确认层启用 | V0/V1→V2 可新增 |
| **权重映射方法** | 等权、信号加权、风险平价 | V2→V3 可指定 |
| **执行参数** | 调仓频率、成交时点 | V2→V3 可固定 |
| **成本参数** | 滑点水平 | V1/V2/V3 可切换标准/压力 |

### 4.2 版本间禁止同时变化的参数

以下参数在版本比较中**必须冻结**:

| 参数类别 | 具体参数 | 冻结原因 |
|----------|----------|----------|
| **Alpha 核心逻辑** | 信号生成公式、因子计算 | 这是策略本身 |
| **Alpha 核心参数** | 回看周期、阈值参数 | 避免"调参"混淆"增强效果" |
| **候选池定义** | 股票池范围、基础过滤条件 | 保证比较基准一致 |
| **数据时间范围** | 回测起止日期 | 避免时间窗口偏差 |
| **基准选择** | 主基准、比较基准 | V1 确定后全程固定 |

### 4.3 冻结规则执行检查表

```
□ V0 到 V1:
  □ 只改数据/成本/基准口径
  □ 不改信号生成逻辑
  □ 不改核心参数

□ V1 到 V2:
  □ 只增加装配层
  □ 装配层参数可独立配置
  □ Alpha 层完全复用 V1

□ V2 到 V3:
  □ 只增加权重映射和执行底座
  □ 权重映射不改变信号本身
  □ 执行底座不改变交易决策
```

## 6. 简化矩阵适用条件

### 6.1 什么策略可以只跑简化矩阵

**简化矩阵定义**: 基线版 + 增强版 + 压力成本版 + OOS版（跳过 V0→V1→V2→V3 完整链条）

| 策略类型 | 适用简化矩阵 | 原因 |
|----------|--------------|------|
| **组合工程策略** | 是 | 本身就是多模块组装，难以拆分 V0/V1/V2 |
| **纯过滤器/确认器** | 是 | 不独立产生交易，只做辅助判断 |
| **宏观择时模块** | 是 | 信号频率低，装配层影响有限 |
| **快速验证阶段** | 是 | 初步筛选，不需要完整对照 |
| **单一 Alpha 策略** | 否 | 必须跑完整 V0→V3 才能评估增强效果 |
| **主仓候选策略** | 否 | 必须完整对照以确定最终角色 |

### 6.2 简化矩阵的四个版本定义

| 版本 | 等价于完整矩阵 | 核心内容 |
|------|----------------|----------|
| **基线版** | V0 + V1 合并 | 统一口径后的原始策略表现 |
| **增强版** | V2 + V3 合并 | 接入完整装配和执行底座 |
| **压力成本版** | V3 + 压力成本 | 检验实盘脆弱性 |
| **样本外版** | V3 + 纯样本外 | 检验可持续性 |

### 6.3 简化矩阵字段

简化矩阵使用相同 CSV 格式，但 `version` 字段取值不同:
- `baseline` - 基线版
- `enhanced` - 增强版
- `stress` - 压力成本版
- `oos` - 样本外版

### 7.2 失败截断规则

| 失败点 | 截断条件 | 后续动作 |
|--------|----------|----------|
| V0 跑不通 | 策略代码报错 | 返回研究者修复代码 |
| V0 费后无收益 | 费后年化 < 0 | 标记为淘汰，结束 |
| V1 差异过大 | V0→V1 收益差异 > 30% | 检查数据/成本设置 |
| V2 严重劣化 | V2 费后收益 < V1 费后收益 × 0.5 | 检查装配层，可调整参数重跑 |
| V2 无改善 | V2 在回撤/Calmar/换手均无改善 | 标记为不适合主仓装配 |
| V3 权重生成失败 | 无法输出目标权重 | 检查 34_alpha_weight_mapping 配置 |

## 10. 常见误判与防错

### 10.1 常见误判

| 误判 | 表现 | 正确做法 |
|------|------|----------|
| **把口径差异当策略差异** | V0→V1 变化大，误以为是策略不行 | 先对齐口径再比较 |
| **同时调参+增强** | V2 收益提升，但其实是参数调优 | 冻结 Alpha 参数，只动装配层 |
| **只看费前不看费后** | 策略零成本很强，费后崩塌 | 费后结果才是真实结论 |
| **忽视样本外** | 全样本很漂亮，OOS 崩溃 | 必须强制 OOS 验证 |
| **混淆装配层效果** | V2 改善来自路由还是确认？ | 分步接入，记录每步结果 |
| **V3 权重映射失真** | 目标权重与信号意图不符 | 检查权重映射逻辑 |

### 10.2 防错检查点

```
检查点 1: V0→V1 阶段
  □ Alpha 层哈希与 V0 一致
  □ 只有数据/成本/基准参数变化
  □ 信号生成代码未修改

检查点 2: V1→V2 阶段
  □ Alpha 层完全复用 V1
  □ 装配层参数独立配置
  □ 每步增强都有独立记录

检查点 3: V2→V3 阶段
  □ 目标权重可生成
  □ 权重与信号方向一致
  □ 执行底座不改变交易决策

检查点 4: 结果汇总阶段
  □ 四版时间范围一致
  □ 基准一致
  □ 成本口径正确标注
```

### 10.3 哈希校验机制

为确保参数冻结规则被执行，建议实现哈希校验:

```python
# Alpha 层配置哈希 = hash(信号公式 + 核心参数 + 候选池定义)
def compute_alpha_layer_hash(signal_config, params, universe):
    content = f"{signal_config}:{sorted(params.items())}:{universe}"
    return hashlib.md5(content.encode()).hexdigest()[:8]

# 装配层配置哈希 = hash(路由 + 确认 + 仓位 + 退出配置)
def compute_assembly_hash(assembly_config):
    content = str(sorted(assembly_config.items()))
    return hashlib.md5(content.encode()).hexdigest()[:8]
```
