### 2.2 results_registry.csv 最关键字段

| 排名 | 字段 | 类型 | 理由 |
|------|------|------|------|
| **1** | `registry_id` | string | 台账唯一标识，所有关联基础 |
| **2** | `strategy_id` | string | 策略标识，跨版本追踪 |
| **3** | `validation_version` | enum | 验证版本 v0/v1/v2/v3 |
| **4** | `grade` | enum | 定档 A/B/C/D/E，快速筛选主仓候选 |
| **5** | `core_metrics_json` | json | 核心指标打包，跨策略比较的数据基础 |
| **6** | `file_path` | string | 完整报告路径，追溯详情 |
| **7** | `status` | enum | 生命周期状态 |
| **8** | `parent_registry_id` | string | 版本血缘追踪 |

**MVP 字段**（第一版只需 8 个）：
```csv
registry_id,experiment_date,strategy_id,strategy_name,strategy_type,validation_version,grade,status,annual_return,max_drawdown,calmar,file_path
```

## 5. 关键风险/注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 前序任务字段定义变化 | 报告模板需要同步修改 | 使用 contract 文件约定字段，变更走版本控制 |
| 报告文件过大 | 查询性能下降 | 摘要信息存 CSV，详情存 Markdown，分离存储 |
| 台账并发写入 | 数据冲突 | 使用文件锁或迁移到数据库 |
| 跨策略比较口径不一致 | 结论失真 | 强制所有报告包含统一口径标识 |
