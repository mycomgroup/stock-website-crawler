# 任务 07 回执：年度 / 状态 / 风格切片

## 一、所有策略必跑切片

### 1.1 强制性年度切片

| 切片类型 | 理由 |
|---------|------|
| 年度收益切片 | 识别"只赚某几年"，防止时间依赖 |
| 年度回撤切片 | 评估历史最差场景，识别修复能力 |
| 样本内/外对照 | 验证过拟合风险 |
| 最近1-2年表现 | 防止旧样本幻觉 |

**原因**：这些切片回答最基本问题——策略是否只在过去某个时期有效，是否样本外衰减。

### 1.2 主仓策略额外必跑

| 切片类型 | 理由 |
|---------|------|
| 市场状态切片 | 主仓必须能在不同市场环境生存 |
| 基本风格暴露 | 识别策略本质是alpha还是因子暴露 |
| 压力成本对照 | 主仓必须通过成本敏感性测试 |

### 1.3 微盘策略额外必跑

| 切片类型 | 理由 |
|---------|------|
| 小盘暴露切片 | 微盘策略必须评估纯小盘效应 |
| 流动性切片 | 微盘策略必须评估流动性风险 |
| 换手率分布 | 微盘策略必须评估交易成本 |

## 二、推荐的状态切片方法

### 2.1 复用03_state_router的五档分类

| 状态 | 广度阈值 | 情绪阈值 | 标准仓位 |
|------|---------|---------|---------|
| 极弱停手 | <15 | <30 | 0% |
| 底部防守 | 15-25 | 30-50 | 30% |
| 震荡平衡 | 25-35 | 50-80 | 50% |
| 趋势正常 | 35-40 | ≥80 | 70% |
| 强趋势 | ≥40 | ≥80 | 100% |

**理由**：
1. 现有实现已经验证有效，避免重新设计
2. 与主路由一致，便于对比理论vs实际仓位
3. 五档分类覆盖牛熊震荡，粒度适中

### 2.2 计算方法

**历史回溯方式**：
- 根据历史数据计算每日应处状态
- 统计每个状态下的收益、Alpha、胜率、回撤
- 计算每个状态的收益贡献占比

**对比分析**（针对已接入主路由的策略）：
- 理论仓位 vs 实际仓位
- 理论收益 vs 实际收益
- 状态切换准确率

### 2.3 核心异常识别

| 异常类型 | 判定条件 |
|---------|---------|
| 单状态依赖 | 单状态贡献>60%总收益 |
| 防守期亏损 | 底部防守+极弱停手Alpha<0 |
| 震荡期失效 | 震荡平衡Alpha显著<0 |
| 趋势期跑输 | 趋势正常+强趋势Alpha<0 |

## 三、最关键异常信号（按严重程度排序）

### 3.1 高严重度（立即处理）

| 异常信号 | 判定条件 | 影响 |
|---------|---------|------|
| 单年依赖 | 单年贡献>50%总收益 | 强制降档至D/E档，淘汰风险 |
| 极端回撤 | 单年最大回撤>40% | 进入A档淘汰审查 |
| 主仓趋势期跑输 | 趋势正常+强趋势Alpha<0 | 强制降档，主仓必须能捕获趋势 |
| 特异收益过低 | 特异收益占比<20% | 标记为因子策略，非真alpha |

### 3.2 中严重度（优先处理）

| 异常信号 | 判定条件 | 影响 |
|---------|---------|------|
| 单状态依赖 | 单状态贡献>60%总收益 | 标记为环境依赖型，降档 |
| 小盘暴露过大 | Size beta<-1.5且显著 | 评估流动性风险，可能降档 |
| 近期失效 | 最近2年年度Alpha<0 | 强制重新验证 |
| 动量依赖 | Momentum beta>2.0且显著 | 监控动量崩塌风险 |
| 集中依赖 | 连续2年贡献>70%总收益 | 进入OOS观察 |

### 3.3 低严重度（监控观察）

| 异常信号 | 判定条件 | 影响 |
|---------|---------|------|
| 负偏度 | 负收益年份占比>40% | 标记为下行风险高 |
| 高波动 | 年度收益标准差>年度均值 | 标记为高波动策略 |
| 部分因子显著 | 显著因子数2-4个 | 正常，但需持续监控 |

## 五、核心设计决策总结

### 5.1 为什么这样设计三类切片

**年度切片**：回答"何时赚钱"问题
- 识别时间依赖、时期特殊性
- 防止"回测好看，实盘崩盘"

**状态切片**：回答"什么环境赚钱"问题
- 识别市场环境依赖
- 评估策略适应性

**风格切片**：回答"靠什么赚钱"问题
- 识别策略本质（alpha vs 因子暴露）
- 防止"伪alpha"混入主仓

### 5.2 为什么这些异常是关键的

**单年依赖**：可能是数据窥探、过拟合
**极端回撤**：主仓必须控制回撤
**主仓趋势期跑输**：主仓核心价值就是捕获趋势
**特异收益过低**：不是真alpha，只是因子暴露

### 5.3 为什么第一版只实现这些

**优先实现**：
- 年度切片（覆盖时间维度）
- 基本风格暴露（识别策略本质）

**延后实现**：
- 行业暴露（按需）
- 宏观周期（按需）
- 极端压力场景（第三版）

**原因**：先保证核心问题能被回答，再逐步扩展。

### 7.1 数据依赖风险

**风险**：因子数据可能缺失或不准确
**缓解**：第一版只使用4个基本因子，数据来源可靠

**风险**：五档分类可能不适合所有策略
**缓解**：保留扩展接口，允许自定义状态分类

### 7.3 异常阈值风险

**风险**：异常阈值（如单年依赖>50%）可能过严或过松
**缓解**：第一版使用保守阈值，根据实际验证结果调整
