# 任务06回执：成本 / 容量 / 滑点压力测试

## 一、推荐的默认压力档位

### 1.1 成本BPS档位（核心回答）

**标准档位**（所有策略必须至少测试）：
- **5 bps**（单边）：ETF主仓/理想场景基准
- **10 bps**（单边）：股票主仓标准场景基准
- **20 bps**（单边）：股票主仓压力场景
- **30 bps**（单边）：微盘/小票压力场景基准

**扩展档位**：
- **50 bps**（单边）：微盘策略极限测试（崩溃测试）

**成本构成明细**：
```
单边总成本 = 佣金(3bps固定) + 滑点(2/7/17/27/47bps) + 冲击成本(动态)
双边总成本 = 2 × 单边总成本

示例：
- 5 bps单边 = 10 bps双边（3bps佣金 + 2bps滑点）
- 10 bps单边 = 20 bps双边（3bps佣金 + 7bps滑点）
- 20 bps单边 = 40 bps双边（3bps佣金 + 17bps滑点）
- 30 bps单边 = 60 bps双边（3bps佣金 + 27bps滑点）
- 50 bps单边 = 100 bps双边（3bps佣金 + 47bps滑点）
```

### 1.2 成交率档位

**标准档位**：
- **100%**：理想成交（基线场景）
- **90%**：轻度partial fill（主仓压力场景）
- **80%**：中度partial fill（微盘压力场景）
- **60%**：重度partial fill（微盘极限场景）
- **40%**：流动性危机（微盘崩溃测试）

### 1.3 成交价口径

**必须测试**：
- **next_open**：下一交易日开盘价成交（所有策略必测）
- **vwap_like**：成交量加权平均价成交（主仓必测）
- **close_like**：当日收盘价成交（微盘/事件策略必测）

**建议测试**：
- **twap_like**：日内时间加权平均价（大资金建议测）
- **limit_fill**：限价单模拟（精细化执行，高级功能）

### 1.4 资金规模档位

| 策略类型 | 建议规模档位（亿） | 最低要求 |
|---------|------------------|---------|
| 股票主仓 | 1 / 5 / 10 / 20 / 50 | 至少4档（1/5/10/50） |
| 微盘策略 | 0.5 / 1 / 2 / 5 / 10 | 至少5档（绘制容量衰减曲线） |
| 小票事件 | 1 / 2 / 5 / 10 / 20 | 至少3档（1/5/10） |
| ETF主仓 | 10 / 50 / 100 / 200 / 500 | 至少4档（10/50/100/200） |

## 二、哪类策略必须最严格跑容量测试

### 2.1 最严格：微盘/小市值策略

**必须最严格的理由**：
1. **流动性天然受限**：小市值股票成交量小，冲击成本高
2. **成交率风险最高**：停牌、涨跌停、流动性枯竭频发
3. **容量限制最明显**：资金规模对收益影响极大，衰减曲线陡峭
4. **成本敏感度最高**：滑点和冲击成本占比大，易坍塌

**必须完成（完整压力矩阵）**：
- **成本档位**：20/30/50 bps三档（必跑全部）
- **成交价口径**：next_open + close_like + vwap_like（必跑全部三种）
- **成交率档位**：100%/90%/80%/60%/40%（必跑全部五档）
- **Partial Fill延迟**：0天/1天/2天/3天（必跑全部四档）
- **单票容量上限**：当日成交额 2%/1%/0.5%/0.2%（必跑全部四档）
- **组合调仓上限**：5000万/2000万/500万（必跑全部三档）
- **资金规模**：0.5/1/2/5/10亿（必跑全部五档，绘制容量衰减曲线）
- **Partial Fill处理逻辑**：延后/放弃两种方式对比（必跑）

**必须报告**：
- 容量衰减曲线（AUM vs Sharpe/收益）
- 成交率敏感性分析（fill_rate vs return）
- 流动性危机模拟（某日完全无法成交）
- Partial Fill两种处理逻辑对比（延后vs放弃）

### 2.2 次严格：小票事件策略

**必须完成**：
- **成本档位**：20/30/50 bps三档
- **成交率档位**：100%/90%/75%三档
- **事件集中度**：<5%/5-10%/>10%三档（必跑全部）
- **事件窗口执行**：T+0 vs T+1对比测试
- **成交价口径**：next_open（事件后首次开盘）

### 2.3 标准：股票低频主仓

**必须完成**：
- **成本档位**：5/10/20/30 bps四档（必跑）
- **成交价口径**：next_open + vwap_like（必跑两种）
- **成交率档位**：100%/90%/80%三档
- **资金规模**：至少4档（建议：1/5/10/50亿）

### 2.4 相对宽松：ETF主仓

## 三、定义的"成本后坍塌"判断方式

### 3.1 三级坍塌体系

**一级坍塌（进入OOS观察）**：

触发条件（满足任一）：
1. **收益衰减**：20 bps成本下年化收益衰减 > 30%
2. **Sharpe衰减**：20 bps成本下Sharpe衰减 > 25%
3. **成交率敏感**：90%成交率下收益衰减 > 25%
4. **口径差异**：next_open vs vwap_like收益差异 > 20%
5. **样本外恶化**：样本外成本敏感性 > 样本内1.5倍

后果：
- 进入D档（OOS观察池）
- 持续跟踪3-6个月
- 观察实盘成本是否符合假设

**二级坍塌（降档处理）**：

触发条件（满足任一）：
1. **收益坍塌**：30 bps成本下年化收益衰减 > 50%
2. **Sharpe坍塌**：30 bps成本下Sharpe衰减 > 40%
3. **成交率崩溃**：80%成交率下收益衰减 > 40%
4. **容量不足**：实际AUM > 容量上限80%
5. **Partial Fill延迟**：延迟2天收益下降 > 30%
6. **盈亏平衡过低**：成本盈亏平衡点 < 15 bps

后果：
- 降档至C档（战略辅助）或更低
- 限制使用场景或容量
- 标注"成本敏感"标签

**三级坍塌（直接淘汰）**：

触发条件（满足任一）：
1. **极端成本失效**：50 bps成本下年化收益为负
2. **成交率崩溃**：60%成交率下收益为负
3. **容量极小**：容量上限 < 5亿（主仓）或 < 2亿（微盘）
4. **盈亏平衡崩溃**：成本盈亏平衡点 < 10 bps
5. **样本外完全失效**：样本外费后收益为负

后果：
- 直接标记为E档（淘汰/挂起）
- 或降为辅助模块（过滤器）
- 不允许进入主仓候选

### 3.2 核心判断指标优先级

**最重要的判断指标**：

1. **Sharpe衰减率**（优先级最高）
   - 原因：风险调整后收益是主仓核心指标
   - 一级阈值：20 bps下 > 25%
   - 二级阈值：30 bps下 > 40%
   - 三级阈值：任意成本下 > 60%

2. **收益衰减率**（优先级高）
   - 原因：直接反映成本侵蚀程度
   - 一级阈值：20 bps下 > 30%
   - 二级阈值：30 bps下 > 50%
   - 三级阈值：任意成本下 > 70%

3. **成本盈亏平衡点**（辅助判断）
   - 原因：识别策略成本容忍边界
   - 一级阈值：< 25 bps
   - 二级阈值：< 15 bps
   - 三级阈值：< 10 bps

4. **成交率敏感性**（特定场景）
   - 原因：识别partial fill风险
   - 一级阈值：90%成交率衰减 > 25%
   - 二级阈值：80%成交率衰减 > 40%
   - 三级阈值：60%成交率收益为负

5. **容量上限利用率**（辅助判断）
   - 原因：识别策略资金承载能力
   - 一级阈值：利用率 > 80%（警告）
   - 二级阈值：利用率 > 90%（降档）
   - 三级阈值：实际AUM > 容量上限（淘汰）

### 3.3 判断流程

```text
开始压力测试
↓
成本压力测试（5/10/20/30/50 bps）
↓
判断：30 bps下Sharpe衰减 > 40%？
├─ 是 → 二级坍塌 → 降档至C档
└─ 否 ↓
判断：20 bps下Sharpe衰减 > 25%？
├─ 是 → 一级坍塌 → 进入OOS（D档）
└─ 否 ↓
判断：20 bps下收益衰减 > 30%？
├─ 是 → 一级坍塌 → 进入OOS（D档）
└─ 否 ↓
成交率测试（90%/80%/60%）
↓
判断：80%成交率下收益衰减 > 40%？
├─ 是 → 二级坍塌 → 降档
└─ 否 ↓
判断：60%成交率下收益为负？
├─ 是 → 三级坍塌 → 淘汰（E档）
└─ 否 ↓
容量测试（多规模档位）
↓
判断：实际AUM > 容量上限80%？
├─ 是 → 二级坍塌 → 降档并警告容量不足
└─ 否 ↓
成交价口径测试（next_open vs vwap_like）
↓
判断：口径差异 > 20%？
├─ 是 → 一级坍塌 → 进入OOS
└─ 否 ↓
通过压力测试 → 进入分档流程
↓
判断：30 bps下Sharpe > 1.0？
├─ 是 → A档（坚韧） → 主仓候选
├─ 否 → 判断：20 bps下Sharpe > 1.0？
    ├─ 是 → B档（可用） → 过滤器候选
    ├─ 否 → C档（脆弱） → 战略辅助
```
