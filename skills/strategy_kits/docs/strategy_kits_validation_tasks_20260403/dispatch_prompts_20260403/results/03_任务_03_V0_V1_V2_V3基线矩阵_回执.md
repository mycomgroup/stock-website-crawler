## 1. V0/V1/V2/V3 一句话定义

| 版本 | 定义 |
|------|------|
| **V0** | 原始策略代码，不做任何统一增强，测量策略本身的原始强度 |
| **V1** | V0 + 统一数据/基准/成本口径，排除回测框架差异，获得可比基线 |
| **V2** | V1 + 主仓装配层（路由/确认/仓位/退出），验证装配层对策略的改善效果 |
| **V3** | V2 + 目标权重生成 + 统一执行底座，工程化后的真实可比结果 |

---

## 2. 版本边界核心结论

### V0 与 V1 的边界
- **V0 保留**: 策略原始信号生成逻辑、参数设置、持仓逻辑、选股范围
- **V1 强制统一**: 数据口径、基准、成本、样本切分、真实价格、未来函数规避
- **原则**: V1 只改"回测写法"，不改"策略逻辑"

### V2 相对 V1 最少新增的增强层
1. **主路由 (03_state_router)** - 市场状态过滤
2. **二级确认 (30_signalmaker_filters)** - 信号质量确认
3. **仓位缩放 (10_position_sizing)** - 基于波动率或信号的仓位调整
4. **退出层 (17_exit_layer)** - 止损/止盈/持有期退出
5. **双确认/再进场 (37_signal_confirmation)** - 高换手策略必须

### V3 何时必须接完整执行底座
- 策略目标是成为主仓候选
- 需要与其他策略做横向比较
- 需要输出目标权重供组合使用
- 需要进入 32_master_portfolio_assembly

## 5. 简化矩阵适用条件

**可以只跑简化矩阵的策略**:
- 组合工程策略（本身就是多模块组装）
- 纯过滤器/确认器（不独立产生交易）
- 宏观择时模块（信号频率低）
- 快速验证阶段

**简化矩阵四版**:
- 基线版 = V0 + V1 合并
- 增强版 = V2 + V3 合并
- 压力成本版 = V3 + 压力成本
- 样本外版 = V3 + 纯样本外

---

## 6. 最小执行顺序

```
步骤 1: 检查前提条件（原策略基线、策略分型、候选池定义）
    ↓
步骤 2: 跑 V0 原策略
    ↓
步骤 3: 跑 V1 统一口径（失败则检查数据/成本设置）
    ↓
步骤 4: 跑 V2 装配增强（严重劣化则检查装配层配置）
    ↓
步骤 5: 跑 V3 完整底座
    ↓
步骤 6: 汇总与归类
```

**失败截断规则**:
- V0 费后无收益 → 标记淘汰，结束
- V0→V1 差异 > 30% → 检查数据/成本
- V2 费后收益 < V1 × 0.5 → 检查装配层，可调整重跑
- V2 无改善 → 标记不适合主仓装配
