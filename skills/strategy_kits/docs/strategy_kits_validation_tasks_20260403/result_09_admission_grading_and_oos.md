# 任务 09：准入定档与 OOS 接续

## 任务结论摘要

本任务为 `admission/` 设计统一裁决接口,实现从验证结果到 A/B/C/D/E 五级定档的自动化映射,并定义 OOS 观察池的接续规则与预警机制。

核心设计包括：
1. 五级定档的最小证据要求与量化阈值
2. 裁决结果的结构化输出 (`admission_decision.json`)
3. OOS 观察池的字段设计 (`oos_watchlist.csv`)
4. 升档/降档/维持/淘汰的动态调整规则
5. 与任务 05/06/07/10 的接口契约

---

## 一、五级定档细化标准

### 1.1 A 档：主仓候选

**定位**：可作为主仓位 alpha 使用,具备实盘价值。

**最小证据要求**：

| 证据维度 | 最低要求 | 验证来源 |
|---------|---------|---------|
| 费后收益 | 年化 > 10%, 费后 Alpha > 5% | 任务 06 压力测试 |
| 回撤控制 | MaxDD < 25%, Calmar > 0.8 | 任务 03 基线矩阵 |
| 样本外稳定性 | OOS 年化不低于样本内 70%, 回撤恶化 < 5% | 任务 05 滚动 OOS |
| 成本后存活 | 20bps 成本下仍可接受 | 任务 06 成本压力 |
| 年度稳定性 | 至少 3 年正收益, 无单一年份 > 50% 贡献 | 任务 07 年度切片 |
| 市场状态适应性 | 在牛市/震荡/下跌至少两种环境有效 | 任务 07 状态切片 |
| 目标权重输出 | 已生成标准目标权重表 | 34_alpha_weight_mapping |
| 执行底座接入 | 已通过统一执行底座验证 | 31_index_enhancement_base |

**淘汰条件**（满足任一）：
- 费后年化 < 8%
- 样本外坍塌（见任务 05 定义）
- 成本敏感性过高（20bps 成本下失效）
- 高度依赖单一年份或风格暴露

---

### 1.2 B 档：过滤器/确认器候选

**定位**：单独跑不一定强,但叠加到主仓后能改善至少一项关键指标。

**最小证据要求**：

| 证据维度 | 最低要求 | 验证来源 |
|---------|---------|---------|
| 主仓叠加改善 | 至少改善 回撤/Calmar/胜率/换手率 之一 | 任务 08 归因诊断 |
| 长期收益拖累 | 叠加后长期收益下降 < 10% | 任务 03 基线矩阵 |
| 逻辑可解释 | 叠加机制可解释,非黑箱 | 策略文档 |
| 非单年依赖 | 不只依赖单一特殊年份 | 任务 07 年度切片 |
| 交易效率 | 不显著增加换手率 > 50% | 任务 03 交易效率类指标 |

**典型对象**：
- QRS 过滤器
- 鳄鱼线确认模块
- 北向资金分位过滤
- 再进场确认模块

**淘汰条件**：
- 叠加后无改善或恶化
- 显著拖累长期收益 > 15%
- 高度依赖单一行情窗口

---

### 1.3 C 档：战略辅助模块

**定位**：日常默认不启用,但在极端环境有高价值。

**最小证据要求**：

| 证据维度 | 最低要求 | 验证来源 |
|---------|---------|---------|
| 极端场景价值 | 在特定市场状态（如底部、极端下跌）有明显贡献 | 任务 07 状态切片 |
| 启用逻辑清晰 | 定义明确的启用条件与退出条件 | 策略文档 |
| 不依赖频繁调参 | 参数设置稳定,不频繁调整 | 策略配置文档 |
| 历史触发有效性 | 至少在历史极端事件中有过有效触发记录 | 任务 07 切片分析 |

**典型对象**：
- 宏观底部信号（20/21/22/23）
- 估值极端开关
- 特殊风险事件过滤

---

### 1.4 D 档：OOS 观察池

**定位**：有一定亮点但证据不足,或样本外不够稳定,需持续观察。

**必须进入 OOS 的场景**：

1. **证据不足型**
   - 增强后看起来可用,但样本外时长 < 6 个月
   - 样本外表现略弱于样本内,但差距在可接受范围内
   - 年度切片证据不足（少于 3 年数据）

2. **成本敏感性存疑型**
   - 收益不错,但在 10-20bps 成本档位下表现不稳定
   - 滑点敏感性较高,需实盘压力验证

3. **风格依赖型**
   - 结果明显依赖某类风格行情（如小盘暴露、动量暴露）
   - 归因显示特异收益占比较低

4. **数据代理型**
   - 使用代理数据,代理误差仍未完全消除
   - 数据质量或时间跨度存在局限

**OOS 观察时长建议**：
- 默认观察期：6-12 个月
- 成本敏感型：延长至 12-18 个月
- 风格依赖型：观察至少一个完整市场周期（24 个月）

---

### 1.5 E 档：淘汰/挂起

**定位**：费后无优势,或存在根本性缺陷。

**直接淘汰条件**（满足任一）：

| 淘汰维度 | 阈值 | 判断依据 |
|---------|------|---------|
| 费后无优势 | 费后年化 < 基准 | 任务 06 压力测试 |
| 增强无效 | 增强后关键指标无改善 | 任务 03 基线矩阵 |
| 单一窗口依赖 | 超过 80% 收益来自单一年份或窗口 | 任务 07 年度切片 |
| 数据代理严重 | 高度依赖不可靠代理数据 | 数据源评估 |
| OOS 长期衰退 | OOS 观察 12 个月持续弱于样本内 | 任务 05 滚动 OOS |
| 成本后崩塌 | 10bps 成本下即失效 | 任务 06 压力测试 |

**处理方式**：
- 暂不继续投入
- 仅保留研究记录
- 6 个月后可考虑重新评估

## 三、升档/降档/维持/淘汰规则

### 3.1 升档规则

**D → C**：
- OOS 观察 6 个月, OOS 收益衰退 < 20%
- 回撤恶化 < 5%
- 成本敏感性可控（20bps 下存活）

**D → B**：
- OOS 观察 12 个月, 持续稳定
- 叠加到主仓测试证明有改善
- 通过所有压力测试

**D → A**：
- OOS 观察 18 个月, 持续稳定
- 完整市场周期验证
- 所有指标满足 A 档要求

**C → B**：
- 证明在非极端场景下也有价值
- 可作为常规过滤器使用

**B → A**：
- 证明单独运行也具备主仓强度
- 所有 A 档证据完备

---

### 3.2 降档规则

**A → B**：
- 样本外持续衰退 > 20%
- 成本敏感性显著提升
- 市场状态覆盖度下降

**A → D**：
- OOS 坍塌（见任务 05 定义）
- 成本后失效（20bps 下失效）
- 年度切片显示结构性变化

**B → D**：
- 叠加后改善消失
- 拖累长期收益 > 15%
- OOS 表现不稳定

**C → D**：
- 极端场景触发效果减弱
- 启用逻辑频繁失效

**D → E**：
- OOS 观察 12 个月持续衰退
- 成本敏感性过高无法改善
- 归因显示结构性缺陷

---

### 3.3 维持规则

**维持当前档位**：
- 在 OOS 观察期内, 各项指标在可接受范围内波动
- 未触发升档或降档阈值
- 持续观察

---

### 3.4 自动复审触发条件

| 触发条件 | 阈值 | 复审动作 |
|---------|------|---------|
| OOS 收益衰退 | > 30% | 立即复审 |
| 回撤恶化 | > 10% | 立即复审 |
| 成本后失效 | 20bps 下失效 | 降档复审 |
| 年度失效 | 最近 1 年负收益 | 年度复审 |
| 市场状态失效 | 新增市场状态下失效 | 状态复审 |

## 四、OOS 预警与复审节奏

### 4.1 OOS 预警等级

| 预警等级 | 触发条件 | 建议动作 |
|---------|---------|---------|
| 绿色 | 所有指标正常 | 继续观察 |
| 黄色 | OOS 衰退 20-30%, 或回撤恶化 5-10% | 加强监控, 准备降档评估 |
| 橙色 | OOS 衰退 30-50%, 或回撤恶化 10-15% | 立即降档评估, 考虑调整或淘汰 |
| 红色 | OOS 衰退 > 50%, 或回撤恶化 > 15%, 或成本失效 | 立即淘汰或挂起 |

---

### 4.2 OOS 观察字段与预警规则

**必须记录的字段**：

1. **收益类**
   - 最近 1/3/6/12 个月收益
   - OOS 年化 vs 样本内年化
   - 费后收益变化

2. **风险类**
   - 当前最大回撤
   - 回撤变化趋势
   - Calmar 变化

3. **效率类**
   - 胜率变化
   - 换手率变化
   - 平均持有期变化

4. **相对表现**
   - vs 基线策略
   - vs 同类策略
   - vs 市场基准

5. **压力指标**
   - 成本敏感性变化
   - 滑点敏感性变化
   - 容量变化

**预警规则示例**：

```python
def compute_alert_level(strategy_metrics):
    alert_score = 0
    
    if strategy_metrics['return_decay_ratio'] > 0.5:
        alert_score += 4
    elif strategy_metrics['return_decay_ratio'] > 0.3:
        alert_score += 3
    elif strategy_metrics['return_decay_ratio'] > 0.2:
        alert_score += 2
    
    if strategy_metrics['dd_worsening_pct'] > 0.15:
        alert_score += 3
    elif strategy_metrics['dd_worsening_pct'] > 0.10:
        alert_score += 2
    
    if not strategy_metrics['cost_survive_20bps']:
        alert_score += 3
    
    if alert_score >= 7:
        return 'RED'
    elif alert_score >= 5:
        return 'ORANGE'
    elif alert_score >= 3:
        return 'YELLOW'
    else:
        return 'GREEN'
```

---

### 4.3 复审节奏建议

**定期复审**：
- 绿色等级：每 3 个月复审一次
- 黄色等级：每 1 个月复审一次
- 橙色等级：每 2 周复审一次
- 红色等级：立即复审

**触发式复审**：
- 预警等级提升时
- 新增重要市场事件后
- 策略参数调整后
- 数据源更新后

**复审内容**：
1. 检查所有 OOS 观察字段
2. 重新评估档位
3. 更新预警等级
4. 决定升档/降档/维持/淘汰
5. 更新 `admission_decision.json`
6. 记录复审日志
