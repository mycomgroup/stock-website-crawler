## Introduction

The HTML Template Generator is a tool that automates the creation of web scraping extraction rules. Instead of manually writing XPath expressions, you provide sample URLs and the tool analyzes them to generate robust extraction templates.

### When to Use This Tool

- You have multiple similar pages to scrape (e.g., product pages, article pages)
- You want consistent extraction rules across similar pages
- You need to extract structured content (tables, lists, headings)
- You want to automate the rule creation process

### Prerequisites

- Node.js 14 or higher
- npm or yarn package manager
- url-patterns.json file (generated by url-pattern-analyzer)
- Sample URLs that represent the pages you want to scrape

## Basic Usage

### Command Structure

```bash
node run-skill.js <template-name> --input <patterns-file> --output <output-file>
```

### Parameters

| Parameter | Required | Description | Example |
|-----------|----------|-------------|---------|
| `<template-name>` | Yes | Name of template from url-patterns.json | `api-doc` |
| `--input` | Yes | Path to url-patterns.json | `./url-patterns.json` |
| `--output` | Yes | Path for output template file | `./output/template.json` |

### Example 1: Generate a Single Template

```bash
node run-skill.js api-doc \
  --input ../../stock-crawler/output/lixinger-crawler/url-patterns.json \
  --output ./output/api-doc.json
```

**What happens:**
1. Reads the "api-doc" template from url-patterns.json
2. Launches Chrome browser with persistent context
3. Fetches HTML from 5 sample URLs
4. Analyzes HTML structure
5. Generates XPath rules
6. Saves template to `./output/api-doc.json`

### Example 2: Using Relative Paths

```bash
node run-skill.js company-detail \
  --input ../url-pattern-analyzer/url-patterns.json \
  --output ./templates/company.json
```

### Example 3: Different Output Directory

```bash
# Create output directory if it doesn't exist
mkdir -p ./my-templates

node run-skill.js fund-info \
  --input ./url-patterns.json \
  --output ./my-templates/fund-info.json
```

## Advanced Usage

### Processing Multiple Templates

#### Option 1: Shell Loop

```bash
#!/bin/bash
# process-all-templates.sh

TEMPLATES=("api-doc" "company-detail" "fund-info" "stock-detail")
INPUT_FILE="../../stock-crawler/output/lixinger-crawler/url-patterns.json"
OUTPUT_DIR="./output"

for template in "${TEMPLATES[@]}"; do
  echo "Processing $template..."
  node run-skill.js "$template" \
    --input "$INPUT_FILE" \
    --output "$OUTPUT_DIR/${template}.json"
  
  if [ $? -eq 0 ]; then
    echo "✓ $template completed"
  else
    echo "✗ $template failed"
  fi
done
```

#### Option 2: Node.js Script

```javascript
// process-all.js
import { TemplateGenerator } from './main.js';
import fs from 'fs/promises';

const templates = ['api-doc', 'company-detail', 'fund-info'];
const inputFile = './url-patterns.json';
const outputDir = './output';

for (const templateName of templates) {
  console.log(`Processing ${templateName}...`);
  
  const generator = new TemplateGenerator();
  const outputFile = `${outputDir}/${templateName}.json`;
  
  try {
    await generator.generate(templateName, inputFile, outputFile);
    console.log(`✓ ${templateName} completed`);
  } catch (error) {
    console.error(`✗ ${templateName} failed:`, error.message);
  }
}
```

## Understanding Output

### Output File Structure

```json
{
  "templateName": "api-doc",
  "version": "1.0.0",
  "generatedAt": "2024-01-15T10:30:00.000Z",
  "samples": [
    "https://example.com/api/doc",
    "https://example.com/api/doc?key=fund"
  ],
  "xpaths": {
    "title": "//h1[@class='page-title']/text()",
    "sections": {
      "xpath": "//section[@class='api-section']",
      "extract": {
        "heading": ".//h2/text()",
        "description": ".//p[@class='description']/text()",
        "table": {
          "xpath": ".//table[@class='params-table']",
          "headers": ".//thead/tr/th/text()",
          "rows": ".//tbody/tr"
        },
        "codeExample": ".//pre/code/text()"
      }
    }
  },
  "filters": {
    "removeXPaths": [
      "//nav",
      "//aside",
      "//div[contains(@class, 'ad')]"
    ],
    "cleanText": true
  },
  "metadata": {
    "sampleCount": 5,
    "commonElements": {
      "title": 5,
      "sections": 5,
      "tables": 5
    }
  }
}
```

### Field Explanations

#### Top-Level Fields

- **templateName**: Name of the template (matches input)
- **version**: Template format version (currently 1.0.0)
- **generatedAt**: ISO timestamp of generation
- **samples**: URLs used for analysis

#### XPath Rules

- **Absolute XPath** (`//element`): Searches from document root
- **Relative XPath** (`.//element`): Searches within current context
- **Text Extraction** (`/text()`): Extracts text content
- **Attribute Extraction** (`/@href`): Extracts attribute value

#### Filters

- **removeXPaths**: Elements to remove before extraction
- **cleanText**: Whether to clean whitespace and formatting

#### Metadata

- **sampleCount**: Number of samples analyzed
- **commonElements**: Frequency of each element type

### Using Generated Templates

#### Example: Extract Content with jsdom

```javascript
import fs from 'fs/promises';
import { JSDOM } from 'jsdom';
import xpath from 'xpath';

// Load template
const templateData = await fs.readFile('./output/api-doc.json', 'utf-8');
const template = JSON.parse(templateData);

// Fetch and parse page
const html = await fetchPageHTML(url);
const dom = new JSDOM(html);
const doc = dom.window.document;

// Extract title
const titleNodes = xpath.select(template.xpaths.title, doc);
const title = titleNodes[0]?.data || '';
console.log('Title:', title);

// Extract sections
const sectionNodes = xpath.select(template.xpaths.sections.xpath, doc);
for (const section of sectionNodes) {
  const heading = xpath.select(template.xpaths.sections.extract.heading, section)[0]?.data;
  console.log('Section:', heading);
}
```

#### Example: Extract Tables

```javascript
// Extract table data
const tableXPath = template.xpaths.sections.extract.table.xpath;
const tables = xpath.select(tableXPath, doc);

for (const table of tables) {
  // Get headers
  const headerXPath = template.xpaths.sections.extract.table.headers;
  const headers = xpath.select(headerXPath, table).map(n => n.data);
  
  // Get rows
  const rowXPath = template.xpaths.sections.extract.table.rows;
  const rows = xpath.select(rowXPath, table);
  
  console.log('Headers:', headers);
  console.log('Row count:', rows.length);
}
```

## Troubleshooting

### Problem: Template Not Found

**Error**: `Template not found: api-doc`

**Solution**:
1. Check template name in url-patterns.json:
   ```bash
   cat url-patterns.json | grep "name"
   ```
2. Ensure exact name match (case-sensitive)
3. Verify JSON file is valid

### Problem: Browser Launch Fails

**Error**: `Failed to launch browser`

**Solutions**:
1. Install Playwright browsers:
   ```bash
   npx playwright install chrome
   ```
2. Check user data directory exists:
   ```bash
   ls -la ../../stock-crawler/chrome_user_data
   ```
3. Try headless mode:
   ```javascript
   this.headless = true;
   ```

### Problem: Page Load Timeout

**Error**: `Timeout waiting for page load`

**Solutions**:
1. Increase timeout in `lib/browser-manager.js`:
   ```javascript
   this.timeout = 60000; // 60 seconds
   ```
2. Check network connectivity
3. Verify URL is accessible

### Problem: No Common Elements Found

**Error**: `No common elements found across samples`

**Solutions**:
1. Verify samples are similar pages
2. Check if pages have consistent structure
3. Lower frequency threshold in `lib/xpath-generator.js`:
   ```javascript
   if (headings.h1 && headings.h1.frequency >= 0.6) // Was 0.8
   ```
4. Increase sample count in url-patterns.json

### Problem: XPath Not Working

**Issue**: Generated XPath doesn't extract expected content

**Solutions**:
1. Test XPath in browser DevTools:
   - Open page in Chrome
   - Press F12 → Console
   - Run: `$x("//h1[@class='title']")`
2. Check for dynamic content (JavaScript-rendered)
3. Manually adjust XPath in output file
4. Use more specific selectors

### Problem: Memory Issues

**Error**: `JavaScript heap out of memory`

**Solutions**:
1. Increase Node.js memory:
   ```bash
   NODE_OPTIONS="--max-old-space-size=4096" node run-skill.js ...
   ```
2. Process templates one at a time
3. Reduce sample count

## Best Practices

### 1. Sample Selection

- **Use 5+ samples**: More samples = more reliable patterns
- **Choose diverse samples**: Include edge cases and variations
- **Verify similarity**: Samples should have similar structure
- **Check authentication**: Ensure samples are accessible

### 2. Template Naming

- **Use descriptive names**: `api-doc`, `company-detail`, not `template1`
- **Follow conventions**: Use kebab-case or snake_case
- **Be specific**: `product-listing` vs `products`

### 3. Output Organization

```
output/
├── api-doc.json
├── company-detail.json
├── fund-info.json
└── stock-detail.json
```

### 4. Version Control

- **Commit templates**: Track changes to extraction rules
- **Document changes**: Note why XPath was modified
- **Test after changes**: Verify templates still work

### 5. Validation

Always validate generated templates:

```bash
# Test template on sample URLs
node test/test-template.js ./output/api-doc.json
```

### 6. Maintenance

- **Regenerate periodically**: Website structures change
- **Monitor failures**: Track extraction errors
- **Update samples**: Add new URL variations
- **Refine XPath**: Manually improve generated rules

### 7. Performance

- **Batch processing**: Process multiple templates in parallel
- **Cache results**: Save HTML for repeated analysis
- **Optimize XPath**: Use specific selectors over broad ones
