## General Questions

### What is the HTML Template Generator?

The HTML Template Generator is a tool that automatically creates web scraping extraction rules by analyzing sample web pages. Instead of manually writing XPath expressions, you provide sample URLs and the tool generates robust extraction templates.

### When should I use this tool?

Use this tool when:
- You need to scrape multiple similar pages (e.g., product pages, articles)
- You want consistent extraction rules across similar pages
- You need to extract structured content (tables, lists, headings)
- You want to automate the rule creation process
- You have 5+ sample URLs that represent the pages you want to scrape

### What are the prerequisites?

- Node.js 14 or higher
- npm or yarn package manager
- url-patterns.json file (generated by url-pattern-analyzer)
- Sample URLs that are accessible and have similar structure

### How does it work?

1. Reads template configuration from url-patterns.json
2. Launches a browser using Playwright
3. Fetches HTML from sample URLs
4. Analyzes HTML structure using jsdom
5. Identifies common patterns across samples
6. Generates XPath extraction rules
7. Outputs a JSON template file

### What's the difference between this and url-pattern-analyzer?

- **url-pattern-analyzer**: Identifies URL patterns and groups similar URLs
- **html-template-generator**: Creates extraction rules for content within pages
- They work together: url-pattern-analyzer → html-template-generator → web scraper

## Installation & Setup

### How do I install the tool?

```bash
cd skills/html-template-generator
npm install
npx playwright install chrome
```

### Do I need to install browsers?

Yes, Playwright requires browser binaries:
```bash
npx playwright install chrome
```

You can also install other browsers:
```bash
npx playwright install firefox
npx playwright install webkit
```

### Can I use an existing Chrome profile?

Yes! The tool uses a persistent browser context. By default, it uses:
```
../../stock-crawler/chrome_user_data
```

You can change this in `lib/browser-manager.js`:
```javascript
this.userDataDir = '/path/to/your/chrome/profile';
```

### How do I verify installation?

```bash
npm test
```

All tests should pass if installation is successful.

### What if npm install fails?

Common solutions:
1. Update Node.js: `node --version` (should be 14+)
2. Clear npm cache: `npm cache clean --force`
3. Delete node_modules: `rm -rf node_modules && npm install`
4. Check network connectivity
5. Try with sudo (Linux/Mac): `sudo npm install`

## Usage Questions

### What is the basic command?

```bash
node run-skill.js <template-name> --input <patterns-file> --output <output-file>
```

Example:
```bash
node run-skill.js api-doc \
  --input ./url-patterns.json \
  --output ./output/api-doc.json
```

### Where do I get url-patterns.json?

Generate it using url-pattern-analyzer:
```bash
cd skills/url-pattern-analyzer
node run-skill.js --input links.txt --output url-patterns.json
```

### How many sample URLs do I need?

**Recommended: 5+ samples**

- Minimum: 3 samples (less reliable)
- Optimal: 5-10 samples (good balance)
- Maximum: No limit, but 10+ may be slow

More samples = more reliable patterns, but slower processing.

### Can I process multiple templates at once?

Yes, use a shell script:
```bash
#!/bin/bash
for template in api-doc company-detail fund-info; do
  node run-skill.js $template \
    --input ./url-patterns.json \
    --output ./output/${template}.json
done
```

### How long does processing take?

Typical timing:
- 5 samples: 20-30 seconds
- 10 samples: 40-60 seconds
- Per sample: 3-5 seconds

Factors affecting speed:
- Page load time
- Network speed
- Page complexity
- Number of samples

### Can I see the browser while it runs?

Yes! Edit `lib/browser-manager.js`:
```javascript
this.headless = false;  // Change from true
```

This is useful for debugging.

### What if a sample URL fails to load?

The tool continues processing other samples. Check the console output for error messages. Common causes:
- Network timeout
- Page requires authentication
- URL is invalid or moved
- Server is down

## XPath Questions

### What is XPath?

XPath is a query language for selecting nodes from XML/HTML documents. It's more powerful than CSS selectors for data extraction.

Example:
```xpath
//h1[@class='title']/text()
```

This selects the text content of h1 elements with class 'title'.

### Why XPath instead of CSS selectors?

XPath advantages:
- ✅ Select by text content: `//h2[contains(text(), 'Parameters')]`
- ✅ Navigate to parent: `//table/../div`
- ✅ Complex logic: `//div[@class='content' and not(@class='hidden')]`
- ✅ Position-based: `//table[1]`, `//table[last()]`

CSS selectors are simpler but less powerful for data extraction.

### How do I test XPath expressions?

**In Browser DevTools:**
```javascript
// Chrome/Firefox Console
$x("//h1[@class='title']")
```

**In Node.js:**
```javascript
import { JSDOM } from 'jsdom';
import xpath from 'xpath';

const dom = new JSDOM(html);
const doc = dom.window.document;
const result = xpath.select("//h1/text()", doc);
```

### What does `.//` mean in XPath?

- `//` = Search from document root (absolute)
- `.//` = Search from current node (relative)

Example:
```json
{
  "sections": {
    "xpath": "//section",
    "extract": {
      "heading": ".//h2/text()"  // Relative to section
    }
  }
}
```

### How do I modify generated XPath?

Edit the output JSON file:
```json
{
  "xpaths": {
    "title": "//h1[@class='page-title']/text()",
    
    // Change to:
    "title": "//h1[contains(@class, 'title')]/text()"
  }
}
```

### What if XPath doesn't work?

1. Test in browser: `$x("your-xpath-here")`
2. Check for typos in class/id names
3. Verify element exists in HTML
4. Check for dynamic content (JavaScript-rendered)
5. Try more flexible selectors: `contains(@class, 'partial')`

## Troubleshooting

### Error: "Template not found"

**Cause:** Template name doesn't exist in url-patterns.json

**Solution:**
```bash
# Check available templates
cat url-patterns.json | grep "name"

# Use exact name (case-sensitive)
node run-skill.js exact-template-name --input ...
```

### Error: "Failed to launch browser"

**Cause:** Playwright browsers not installed

**Solution:**
```bash
npx playwright install chrome
```

**Alternative causes:**
- Insufficient permissions
- Corrupted browser installation
- Missing system dependencies (Linux)

**Linux solution:**
```bash
# Install dependencies
npx playwright install-deps chrome
```

### Error: "Timeout waiting for page load"

**Cause:** Page takes too long to load

**Solution 1:** Increase timeout in `lib/browser-manager.js`:
```javascript
this.timeout = 60000;  // 60 seconds (was 30000)
```

**Solution 2:** Check network connectivity:
```bash
curl -I https://example.com
```

**Solution 3:** Try headless=false to see what's happening:
```javascript
this.headless = false;
```

### Error: "No common elements found"

**Cause:** Sample pages have different structures

**Solutions:**
1. Verify samples are similar pages
2. Check if pages loaded correctly
3. Lower frequency threshold in `lib/xpath-generator.js`:
   ```javascript
   if (headings.h1 && headings.h1.frequency >= 0.6) // Was 0.8
   ```
4. Add more samples to url-patterns.json

### Error: "JavaScript heap out of memory"

**Cause:** Not enough memory for large pages

**Solution:**
```bash
NODE_OPTIONS="--max-old-space-size=4096" node run-skill.js ...
```

### Generated XPath doesn't extract content

**Debugging steps:**

1. **Test XPath in browser:**
   ```javascript
   $x("//h1[@class='title']/text()")
   ```

2. **Check HTML structure:**
   - View page source
   - Verify class/id names
   - Check for dynamic content

3. **Try simpler XPath:**
   ```xpath
   # Instead of:
   //h1[@class='page-title main-title']
   
   # Try:
   //h1[contains(@class, 'title')]
   ```

4. **Check for JavaScript rendering:**
   - Some content is added by JavaScript
   - Playwright should handle this, but check timing

### Pages require login

**Solution:** Use persistent browser context with saved login

1. **Login manually in Chrome:**
   ```bash
   # Start Chrome with user data directory
   chrome --user-data-dir=/path/to/chrome_user_data
   # Login to the website
   ```

2. **Configure tool to use that directory:**
   ```javascript
   // lib/browser-manager.js
   this.userDataDir = '/path/to/chrome_user_data';
   ```

3. **Run tool:**
   ```bash
   node run-skill.js template-name --input ... --output ...
   ```

### Output file is empty or invalid

**Causes:**
- All samples failed to load
- No common elements found
- Write permission issue

**Solutions:**
1. Check console output for errors
2. Verify output directory exists and is writable
3. Try different output path
4. Check if samples loaded successfully

## Performance & Optimization

### How can I speed up processing?

1. **Reduce sample count:**
   ```json
   {
     "samples": ["url1", "url2", "url3"]  // 3 instead of 5
   }
   ```

2. **Use faster network:**
   - Local network vs remote
   - Wired vs wireless

3. **Increase timeout for slow pages:**
   ```javascript
   this.timeout = 60000;
   ```

4. **Process templates in parallel:**
   ```bash
   node run-skill.js template1 ... &
   node run-skill.js template2 ... &
   wait
   ```

### How much memory does it use?

Typical usage:
- Small pages (< 100KB): ~200MB
- Medium pages (100KB-1MB): ~300-400MB
- Large pages (> 1MB): ~500MB+

For large pages, increase Node.js memory:
```bash
NODE_OPTIONS="--max-old-space-size=4096" node run-skill.js ...
```

### Can I cache HTML to avoid re-fetching?

Yes, modify `lib/html-fetcher.js` to save HTML:
```javascript
async fetchOne(url) {
  const cacheFile = `./cache/${encodeURIComponent(url)}.html`;
  
  // Check cache
  if (fs.existsSync(cacheFile)) {
    return {
      url,
      html: await fs.readFile(cacheFile, 'utf-8'),
      title: 'Cached',
      timestamp: new Date().toISOString()
    };
  }
  
  // Fetch and cache
  const result = await this._fetchFromBrowser(url);
  await fs.writeFile(cacheFile, result.html);
  return result;
}
```

### Should I run this on a schedule?

Yes, if website structure changes frequently:
```bash
# Cron job (daily at 2 AM)
0 2 * * * cd /path/to/html-template-generator && node run-skill.js template-name --input ... --output ...
```

## Integration Questions

### How do I use generated templates?

**Example with jsdom:**
```javascript
import fs from 'fs/promises';
import { JSDOM } from 'jsdom';
import xpath from 'xpath';

// Load template
const template = JSON.parse(
  await fs.readFile('./output/api-doc.json', 'utf-8')
);

// Fetch page
const html = await fetchPage(url);
const dom = new JSDOM(html);
const doc = dom.window.document;

// Extract using template
const title = xpath.select(template.xpaths.title, doc)[0]?.data;
const sections = xpath.select(template.xpaths.sections.xpath, doc);
```

### Can I use templates with Puppeteer?

Yes, but you need to evaluate XPath in the browser:
```javascript
const title = await page.evaluate((xpath) => {
  const result = document.evaluate(
    xpath,
    document,
    null,
    XPathResult.STRING_TYPE,
    null
  );
  return result.stringValue;
}, template.xpaths.title);
```

### Can I use templates with Playwright?

Yes, similar to Puppeteer:
```javascript
const title = await page.evaluate((xpath) => {
  const result = document.evaluate(
    xpath,
    document,
    null,
    XPathResult.STRING_TYPE,
    null
  );
  return result.stringValue;
}, template.xpaths.title);
```

### How do I integrate with stock-crawler?

1. Generate templates for each URL pattern
2. Load templates in crawler configuration
3. Use templates to extract content
4. Generate Markdown from extracted data

### Can I export templates to other formats?

Yes, write a converter:
```javascript
// Convert to CSS selectors (limited)
function xpathToCSS(xpath) {
  // Simple conversion
  return xpath
    .replace(/\/\//g, ' ')
    .replace(/\[@class='([^']+)'\]/g, '.$1')
    .replace(/\[@id='([^']+)'\]/g, '#$1');
}

// Convert to Scrapy selectors
function xpathToScrapy(xpath) {
  return `response.xpath('${xpath}').get()`;
}
```

## Advanced Topics

### Can I customize the analysis algorithm?

Yes, edit `lib/structure-analyzer.js`:
```javascript
_mergeHeadings(headingsList, sampleCount) {
  // Change frequency threshold
  const minFrequency = 0.6;  // Was 0.8
  
  // Add custom logic
  // ...

```

### Can I add custom element types?

Yes, extend `lib/structure-analyzer.js`:
```javascript
async _analyzeSingle(html) {
  return {
    mainContent: this._findMainContent(document),
    headings: this._extractHeadings(document),
    tables: this._extractTables(document),
    
    // Add custom extraction
    videos: this._extractVideos(document),
    forms: this._extractForms(document)
  };
}

_extractVideos(document) {
  const videos = document.querySelectorAll('video, iframe[src*="youtube"]');
  return Array.from(videos).map(video => ({
    src: video.src,
    xpath: this._generateXPath(video)
  }));
}
```

### Can I generate templates for JavaScript-rendered content?

Yes, Playwright handles JavaScript rendering. Ensure pages load completely:
```javascript
// lib/html-fetcher.js
await page.goto(url, { 
  waitUntil: 'networkidle',  // Wait for network to be idle
  timeout: this.timeout 
});

// Additional wait for dynamic content
await page.waitForTimeout(2000);
```

### Can I validate templates automatically?

Yes, create a validation script:
```javascript
// validate-template.js
import { JSDOM } from 'jsdom';
import xpath from 'xpath';

async function validateTemplate(template, sampleHTML) {
  const dom = new JSDOM(sampleHTML);
  const doc = dom.window.document;
  
  const results = {};
  
  // Test each XPath
  for (const [key, xpathExpr] of Object.entries(template.xpaths)) {
    try {
      const nodes = xpath.select(xpathExpr, doc);
      results[key] = {
        success: nodes.length > 0,
        count: nodes.length
      };
    } catch (error) {
      results[key] = {
        success: false,
        error: error.message
      };
    }
  }
  
  return results;
}
```

### How do I handle pagination?

Templates don't handle pagination directly. You need to:
1. Identify pagination links separately
2. Use template to extract content from each page
3. Combine results

Example:
```javascript
// Extract pagination links
const nextPageXPath = "//a[@class='next-page']/@href";
const nextPage = xpath.select(nextPageXPath, doc)[0]?.value;

// Extract content using template
const content = extractWithTemplate(template, doc);

// Recursively process next page
if (nextPage) {
  const nextContent = await processPage(nextPage, template);
  content.push(...nextContent);
}
```

Yes, configure Playwright to emulate mobile:

```javascript
const browser = await chromium.launchPersistentContext(userDataDir, {

```
