## Description

Automatically generates HTML extraction templates with XPath rules by analyzing sample web pages. This skill bridges the gap between URL pattern identification and content extraction, enabling automated creation of robust, reusable extraction rules for web scraping workflows.

## Purpose

The html-template-generator skill solves a critical problem in web scraping workflows: manually creating extraction rules is time-consuming and error-prone. This skill automates the process by:

1. **Fetching Sample Pages**: Uses Playwright to download HTML from sample URLs with support for authenticated sessions
2. **Analyzing Structure**: Employs jsdom to parse HTML and identify common structural patterns across multiple samples
3. **Identifying Patterns**: Calculates element frequency to find reliable extraction targets (elements appearing in 80%+ of samples)
4. **Generating XPath**: Creates robust XPath expressions that work across similar pages
5. **Creating Filters**: Automatically identifies and filters out noise (navigation, ads, sidebars)
6. **Outputting Templates**: Produces structured JSON templates ready for use in web scrapers

## Workflow Position

This skill is the second step in a three-stage web scraping pipeline:

```
[url-pattern-analyzer] → [html-template-generator] → [web-scraper]
     url-patterns.json         extraction-templates        content.md
```

## Input

### Required Input File

- **url-patterns.json**: Generated by url-pattern-analyzer, containing:
  - Template names and descriptions
  - URL patterns and path templates
  - Sample URLs (typically 5 per template)
  - Query parameters and URL structure

### Example Input Structure

```json
{
  "templates": [
    {
      "name": "api-doc",
      "description": "API documentation pages",
      "pathTemplate": "/open/api/doc",
      "samples": [
        "https://example.com/api/doc",
        "https://example.com/api/doc?key=fund",
        "..."
      ]
    }
  ]
}
```

## Output

### Template JSON Files

Generated templates contain:

- **Template Metadata**: Name, version, generation timestamp
- **Sample URLs**: URLs used for analysis
- **XPath Rules**: Extraction expressions for:
  - Page title
  - Section headings
  - Tables (with headers and rows)
  - Code blocks
  - Lists
  - Custom content areas
- **Filter Rules**: XPath expressions to remove unwanted content
- **Analysis Metadata**: Element frequencies and sample count

### Example Output

```json
{
  "templateName": "api-doc",
  "version": "1.0.0",
  "generatedAt": "2024-01-15T10:30:00Z",
  "samples": ["..."],
  "xpaths": {
    "title": "//h1[@class='page-title']/text()",
    "sections": {
      "xpath": "//section[@class='api-section']",
      "extract": {
        "heading": ".//h2/text()",
        "table": {
          "xpath": ".//table[@class='params']",
          "headers": ".//thead/tr/th/text()",
          "rows": ".//tbody/tr"
        }
      }
    }
  },
  "filters": {
    "removeXPaths": ["//nav", "//aside", "//div[contains(@class, 'ad')]"],
    "cleanText": true
  }
}
```

## Usage

### Basic Command

```bash
node run-skill.js <template-name> --input <patterns-file> --output <output-file>
```

### Parameters

- `<template-name>`: Name of the template to process (from url-patterns.json)
- `--input`: Path to url-patterns.json file
- `--output`: Path for the generated template file

### Common Use Cases

#### 1. Generate Single Template

```bash
node run-skill.js api-doc \
  --input ../../stock-crawler/output/lixinger-crawler/url-patterns.json \
  --output ./output/api-doc.json
```

#### 2. Process Multiple Templates

```bash
# Process each template separately
for template in api-doc company-detail fund-info; do
  node run-skill.js $template \
    --input ./url-patterns.json \
    --output ./output/${template}.json
done
```

#### 3. Custom Browser Configuration

Modify `lib/browser-manager.js` to use different browser settings or user data directories.

## Key Features

### 1. Automatic XPath Generation

- Generates stable XPath expressions based on element attributes
- Prioritizes `id` and `class` attributes over positional selectors
- Uses `contains()` for flexible class matching
- Creates relative paths for nested elements

### 2. Frequency-Based Pattern Identification

- Analyzes multiple sample pages (typically 5)
- Calculates element frequency across samples
- Selects elements appearing in 80%+ of samples
- Ensures generated rules work reliably across similar pages

### 3. Comprehensive Element Support

- **Headings**: h1-h6 with hierarchy preservation
- **Tables**: Headers, rows, cells with structure
- **Code Blocks**: Pre/code elements with language detection
- **Lists**: Ordered and unordered lists with items
- **Custom Containers**: Main content areas and sections

### 4. Intelligent Filtering

- Automatically identifies noise elements (nav, aside, ads)
- Generates removal XPath expressions
- Supports text cleaning options
- Preserves main content while removing clutter

### 5. Browser Session Reuse

- Uses Playwright's persistent context
- Reuses login sessions from stock-crawler
- Accesses authenticated pages without re-login
- Configurable user data directory

### 6. Robust Error Handling

- Continues processing if individual samples fail
- Provides detailed error messages
- Cleans up browser resources properly
- Validates input and output data

## Technical Details

### Architecture

- **Modular Design**: Six independent modules with clear responsibilities
- **Async/Await**: Modern JavaScript async patterns throughout
- **Error Boundaries**: Comprehensive error handling at each stage
- **Resource Management**: Proper cleanup of browser and file resources

### Performance

- **Processing Time**: < 30 seconds for 5 sample pages
- **Memory Usage**: < 500MB typical
- **Concurrent Fetching**: Parallel page downloads when possible
- **Efficient Parsing**: jsdom for fast HTML analysis

### XPath Strategy

- **Stability**: Prefers attribute-based selectors over positions
- **Flexibility**: Uses `contains()` for partial matches
- **Hierarchy**: Maintains parent-child relationships
- **Relative Paths**: Uses `.//` for context-aware selection

## Limitations

- Requires consistent HTML structure across sample pages
- Works best with 5+ sample URLs
- May need manual XPath adjustment for complex layouts
- Does not handle JavaScript-rendered content (use Playwright for that)
- Assumes similar page structure within a template
