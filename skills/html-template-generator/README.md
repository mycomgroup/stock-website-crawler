# HTML Template Generator

Generate HTML extraction templates using XPath from sample URLs.

## Overview

This skill analyzes sample web pages and generates XPath-based extraction templates that can be used to extract structured content from similar pages. It bridges the gap between URL pattern identification and content extraction by automatically creating reusable extraction rules.

## Features

- **Pattern Reading**: Reads URL patterns from `url-patterns.json` generated by url-pattern-analyzer
- **Smart Fetching**: Uses Playwright with persistent browser context to fetch pages (supports authenticated sessions)
- **Structure Analysis**: Analyzes HTML structure using jsdom to identify common patterns across samples
- **XPath Generation**: Automatically generates robust XPath expressions for content extraction
- **Filter Rules**: Creates rules to remove unwanted content (ads, navigation, etc.)
- **JSON Output**: Produces structured template files ready for use in web scrapers

## How It Works

1. **Read Patterns**: Loads the specified template from url-patterns.json
2. **Launch Browser**: Starts Playwright with persistent context (reuses login sessions)
3. **Fetch Samples**: Downloads HTML from all sample URLs (typically 5 pages)
4. **Analyze Structure**: Identifies common elements (headings, tables, code blocks, lists)
5. **Generate XPath**: Creates XPath expressions for elements appearing in 80%+ of samples
6. **Write Template**: Outputs a JSON file with extraction rules and metadata

## Architecture

### Module Structure

```
html-template-generator/
├── lib/
│   ├── pattern-reader.js      # Reads url-patterns.json
│   ├── browser-manager.js      # Manages Playwright browser
│   ├── html-fetcher.js         # Fetches HTML from URLs
│   ├── structure-analyzer.js   # Analyzes HTML structure
│   ├── xpath-generator.js      # Generates XPath expressions
│   └── template-writer.js      # Writes template JSON
├── main.js                     # Main orchestration
├── run-skill.js                # CLI entry point
└── docs/                       # Documentation
```

### Data Flow

```
url-patterns.json
    ↓
[PatternReader] → Template + Sample URLs
    ↓
[BrowserManager] → Playwright Browser
    ↓
[HTMLFetcher] → HTML Content
    ↓
[StructureAnalyzer] → Common Patterns
    ↓
[XPathGenerator] → XPath Rules
    ↓
[TemplateWriter] → template.json
```

## Output Format

Generated templates include:

```json
{
  "templateName": "api-doc",
  "version": "1.0.0",
  "generatedAt": "2024-01-15T10:30:00Z",
  "samples": ["https://example.com/page1", "..."],
  "xpaths": {
    "title": "//h1[@class='page-title']/text()",
    "sections": {
      "xpath": "//div[@class='main-content']",
      "extract": {
        "heading": ".//h2/text()",
        "table": {
          "xpath": ".//table[@class='params']",
          "headers": ".//thead/tr/th/text()",
          "rows": ".//tbody/tr"
        }
      }
    }
  },
  "filters": {
    "removeXPaths": ["//nav", "//aside"],
    "cleanText": true
  },
  "metadata": {
    "sampleCount": 5,
    "commonElements": {
      "title": 5,
      "sections": 5
    }
  }
}
```

## Browser Configuration

The skill uses Playwright with a persistent browser context, which allows it to:
- Reuse login sessions from stock-crawler
- Access authenticated pages without re-login
- Use the Chrome user data directory at `../../stock-crawler/chrome_user_data`

To use a different browser profile, modify the configuration in `lib/browser-manager.js`.

## Performance

- Single template processing: < 30 seconds (5 sample pages)
- Memory usage: < 500MB
- Concurrent page fetching for faster processing

## Troubleshooting

### Browser Launch Fails

Ensure Playwright browsers are installed:
```bash
npx playwright install chrome
```

### Template Not Found

Verify the template name exists in url-patterns.json:
```bash
cat url-patterns.json | grep "templateName"
```

### XPath Not Working

Check the generated XPath in the output file and test it manually using browser DevTools.

## Integration

This skill is designed to work in a pipeline:

1. **url-pattern-analyzer** → Generates url-patterns.json
2. **html-template-generator** → Generates extraction templates
3. **stock-crawler** (or custom scraper) → Uses templates to extract content
