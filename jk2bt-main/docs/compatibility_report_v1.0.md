## 5. 失败归因分析

### 5.1 失败原因分类

| 失败类型 | 数量 | 占比 | 说明 |
|----------|------|------|------|
| **超时 (timeout)** | 27 | 56.2% | 策略运行超过时间限制 |
| **NameError** | 8 | 16.7% | 未定义的函数或变量 |
| **缺失模块** | 6 | 12.5% | Python依赖包缺失 |
| **其他错误** | 3 | 6.2% | 其他运行时错误 |
| **缺失文件** | 2 | 4.2% | 外部模型/数据文件缺失 |
| **语法错误** | 2 | 4.2% | 代码语法问题 |

### 5.2 详细归因

#### 5.2.1 超时问题 (56.2%)

超时是最主要的失败原因，主要涉及以下类型策略：
- 多因子选股策略（计算复杂度高）
- 机器学习策略（模型推理耗时）
- 大市值策略（数据量大）

**建议**: 
- 用户可调整回测时间范围缩短运行时间
- 后续版本可优化计算性能

#### 5.2.2 NameError问题 (16.7%)

| 未定义名称 | 出现次数 | 说明 |
|------------|----------|------|
| get_current_data_wrapper | 5 | 策略内部函数，需在策略代码中定义 |
| Enum | 2 | Python标准库，需导入 |
| BarData | 1 | Backtrader数据类型 |

**建议**: 
- `Enum`和`BarData`可通过在策略顶部添加导入解决
- `get_current_data_wrapper`是策略自定义函数，需用户自行定义

#### 5.2.3 缺失模块问题 (12.5%)

| 缺失模块 | 出现次数 | 说明 |
|----------|----------|------|
| src | 3 | 策略内部模块引用 |
| torch | 2 | PyTorch深度学习库 |
| sqlalchemy | 1 | SQL数据库库 |

**解决方案**:
```bash
pip install torch sqlalchemy
```

#### 5.2.4 缺失文件问题 (4.2%)

| 缺失文件 | 出现次数 | 说明 |
|----------|----------|------|
| ZCSZX_0.pt | 2 | PyTorch模型文件 |

**说明**: 这些策略依赖外部模型文件，用户需自行获取模型文件并放置到正确路径。

## 8. 产品发布建议

### 8.1 可直接发布

- **核心交易API**: 100%覆盖率，支持主流策略
- **静态扫描系统**: 准确识别395个可执行策略
- **批量运行框架**: 支持并行运行，超时保护

### 8.2 需用户知晓的限制

1. **机器学习策略**: 需自行安装`torch`等依赖库
2. **外部模型文件**: 需用户自行获取并配置路径
3. **超时策略**: 建议缩短回测时间范围（如1年而非10年）
4. **研究文档**: notebook和md文件不计入支持范围

### 8.3 后续优化方向

1. 添加`log.set_level`和`finance.run_query`方法（高优先级）
2. 优化计算性能，降低超时比例（中优先级）
3. 提供常用筛选函数的实现模板（中优先级）
