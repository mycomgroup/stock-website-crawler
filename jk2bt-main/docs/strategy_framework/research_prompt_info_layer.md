# 子 Agent 调研提示词：信息层

你负责调研并产出“多策略调度框架的信息层设计方案”。

## 背景

当前项目目标不是只跑单个策略，而是构建一个可以调度大量 `.txt` 策略的统一框架。信息层是最底层输入层，但我们明确不想做“全量信息仓库”，而是要做“影响策略适用性的最小有效信息层”。

## 你的任务

请围绕以下目标展开调研和设计：

1. 定义信息层应该保留哪些信息，不应该保留哪些信息
2. 将输入分成四组：
   - 宏观/流动性
   - 市场状态
   - 板块状态
   - 微观交易状态
3. 为每组设计核心指标清单
4. 说明每个指标对“策略适用性判断”的作用
5. 设计统一的数据结构、缓存结构、质量控制方案
6. 给出第一阶段 MVP 的最小实现建议

## 必须回答的问题

- 哪些指标是真正有用的，哪些只是“看起来很丰富但没必要”
- 日线多策略调度的第一版信息层最小集合应该是什么
- 如何保证离线可回放、可缓存、可解释
- 如何处理低频宏观数据和高频市场数据之间的频率冲突
- 如何定义数据质量、缺失值和异常值处理

## 交付要求

请输出一份详细文档，至少包括：

1. 设计目标
2. 设计原则
3. 四组信息的字段建议
4. 数据模型设计
5. 标准化与特征工程建议
6. 缓存与离线方案
7. MVP 范围
8. 风险与常见误区

## 注意事项

- 不要泛泛而谈“多收集数据”
- 重点是“哪些信息能提高策略调度质量”
- 重点服务于股票 / ETF / 指数类多策略框架
- 可以提行业实践，但必须落到可实现结构

## 输出风格

- 详细
- 结构化
- 面向系统设计，而不是学术综述
- 尽量给出字段级别建议
