# 子 Agent 调研提示词：策略画像层

你负责调研并产出“多策略调度框架的策略画像层设计方案”。

## 背景

当前系统要面对大量 `.txt` 策略，这些策略异构性很强。如果没有统一画像层，就无法稳定地做：

- 场景匹配
- 策略启停
- 多策略并行分仓
- 风险控制
- 依赖管理

因此需要把每个策略抽象成统一的策略画像。

## 你的任务

请围绕以下目标调研和设计：

1. 定义一套适用于 `.txt` 策略的统一画像结构
2. 设计资产类型、频率、风格、适用场景、持仓特征、风险特征、依赖条件等标签体系
3. 说明这些标签如何从源码分析、回测统计、人工校验中得到
4. 设计策略分级接入体系，让“所有策略都能套入”但不是都按同一层级接入
5. 设计画像与调度层、风控层之间的接口

## 必须回答的问题

- 哪些画像字段是第一版必须有的
- 哪些画像字段可以后续补
- 如何区分“可加载”“可回测”“可调度”“可实盘”
- 如何识别策略之间的重复暴露和高度相似
- 如何处理依赖复杂、分钟级、期货、ML、资源文件型策略

## 交付要求

请输出一份详细设计文档，至少包括：

1. 策略画像的目标与职责
2. 画像字段结构
3. 标签体系设计
4. 画像生成流程
5. A/B/C/D 四级接入框架
6. 风险画像建议
7. 依赖管理设计
8. 与调度层的接口
9. MVP 范围
10. 常见误区

## 注意事项

- 不要停留在“给策略贴几个标签”
- 重点是让画像真正能服务于调度与组合管理
- 重点考虑大量 `.txt` 策略的异构现实
- 可以给出字段示例、JSON 结构、分层接入建议

## 输出风格

- 详细
- 结构化
- 工程可落地
- 面向后续系统实现
