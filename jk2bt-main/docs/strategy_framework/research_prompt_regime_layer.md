# 子 Agent 调研提示词：场景识别层

你负责调研并产出“多策略调度框架的场景识别层设计方案”。

## 背景

我们希望先用信息层提取一组稳定、可回放、可解释的市场特征，然后用场景识别层把市场切成少数几个 regime，供策略调度层决定“此时该启用哪些策略、给多少权重”。

我们不希望做一个主观判断模块，也不希望把它做成单纯预测涨跌模型。

## 你的任务

请围绕以下目标调研和设计：

1. 如何把市场环境切成少数几个稳定的 regime
2. 如何输出“场景分数向量”，而不是一句主观结论
3. 如何设计第一阶段规则模型
4. 如何平滑场景切换，避免频繁抖动
5. 如何为后续统计模型或机器学习模型保留扩展空间

## 建议聚焦的场景

- 趋势上行
- 震荡轮动
- 高波动防守
- 流动性收缩
- 极端风险

## 必须回答的问题

- 为什么是这几个场景，而不是更多或更少
- 每个场景对应哪些核心特征
- 如何从信息层特征映射到场景分数
- 如何做平滑、阈值切换、风险覆盖
- 如何评估场景模型是否真的改善了策略调度

## 交付要求

请输出一份详细设计文档，至少包括：

1. 场景定义
2. 输入输出结构
3. 规则打分方案
4. 平滑与切换机制
5. 统计建模/机器学习的后续扩展路线
6. 与调度层的接口设计
7. MVP 方案
8. 风险与常见错误

## 注意事项

- 不要把任务写成“预测未来收益”
- 不要把场景定义做得太碎
- 不要只说概念，尽量写出实际可实现的分数结构和状态机逻辑

## 输出风格

- 详细
- 强调工程落地
- 强调可解释性
- 强调如何服务后续策略调度
