# 信息层设计文档

## 1. 目标

信息层的职责不是“尽可能多抓数据”，而是为后续的场景识别层和策略调度层提供**少量、稳定、可解释、可复现**的输入特征。

信息层只回答一个问题：

**当前市场环境，是否适合某类策略发挥？**

因此信息层的核心原则是：

- 不求全，求有效
- 不求实时复杂，求稳定可维护
- 不追求预测市场，优先服务“策略适用性判断”
- 所有指标都必须能解释、能回放、能缓存

---

## 2. 设计原则

### 2.1 少而精

每组信息保留 5 到 15 个核心指标，不要无限扩展。

### 2.2 结构化

所有信息必须落到统一结构：

- 指标名
- 计算日期/时间
- 适用频率
- 原始值
- 标准化值
- 缺失状态
- 数据来源

### 2.3 分层

信息层只负责“提取”和“标准化”，不负责最终判断市场属于哪个场景。

### 2.4 可缓存

必须支持离线重放。任何用于场景识别的输入，都不能强依赖即时在线接口。

### 2.5 可替换

同一指标允许多个来源。例如北向资金可以优先用 AkShare，失败时回退缓存或本地表。

---

## 3. 总体结构

信息层建议拆成 4 组：

1. 宏观/流动性
2. 市场状态
3. 板块状态
4. 微观交易状态

统一输出一个 `MarketFeatureSnapshot`：

```python
{
    "timestamp": "2026-03-31 15:00:00",
    "frequency": "daily",
    "macro_liquidity": {...},
    "market_state": {...},
    "sector_state": {...},
    "micro_structure": {...},
    "quality_flags": {...}
}
```

---

## 4. 四组核心信息设计

## 4.1 宏观/流动性

这一层不是做宏观择时模型，而是判断市场处在“风险偏好扩张”还是“收缩防守”。

### 建议保留的核心字段

- 短端利率：如 DR007、Shibor 7D
- 长端利率：10 年国债收益率
- 利差：长短端利差、信用利差
- 汇率：USD/CNY 或 CNH
- 北向资金净流入
- 两市成交额
- 融资余额变化
- ETF 资金净流入

### 对策略适用性的意义

- 趋势/高 beta/小市值策略通常偏好流动性扩张
- 防守/红利/低波策略更适合流动性收缩或利率上行阶段
- 北向持续流入通常有利于大盘龙头、核心资产、宽基 ETF

### 典型衍生特征

- `north_money_5d_sum`
- `turnover_vs_20d_mean`
- `credit_spread_zscore`
- `cny_depreciation_20d`
- `margin_balance_20d_change`

### 设计注意事项

- 宏观数据频率通常较低，不要伪装成高频信号
- 同一类流动性指标不要重复堆叠
- 缺失数据可以用“最近可得值”填充，但必须有 `staleness_days`

---

## 4.2 市场状态

这是信息层里最重要的一层，用于描述“大盘行为”。

### 建议保留的核心字段

- 主要指数收益率：1D / 5D / 20D
- 均线位置：收盘价相对 MA20 / MA60 / MA120
- 趋势斜率：回归斜率或趋势强度
- 实现波动率：10D / 20D / 60D
- 振幅
- 涨跌家数比
- 创新高 / 创新低数量
- 市场宽度：站上 MA20 的个股占比
- 拥挤度：高换手集中度、抱团度

### 对策略适用性的意义

- 趋势策略依赖指数趋势强度和市场宽度
- 震荡轮动策略依赖高分化、低持续性
- 防守策略依赖波动率抬升、宽度恶化

### 建议的标准化方式

- 使用 rolling z-score
- 或对历史分位数进行映射
- 输出范围尽量统一到 `[-3, 3]` 或 `[0, 1]`

### 典型衍生特征

- `index_trend_strength`
- `breadth_ma20_ratio`
- `market_realized_vol_20d`
- `advance_decline_ratio_5d`
- `crowding_score`

---

## 4.3 板块状态

板块状态用于判断“市场是全面共振，还是板块轮动，还是局部热点”。

### 建议保留的核心字段

- 行业近 5D / 20D 相对强弱
- 行业动量离散度
- 热门行业成交额占比
- 行业轮动速度
- 行业领涨持续性
- 涨停集中行业占比
- 主题集中度

### 对策略适用性的意义

- 行业轮动型策略依赖板块强弱切换
- 低频价值策略不一定依赖热点，但需要知道是否处于高主题投机环境
- 微盘/题材策略依赖热点集中和龙头持续性

### 关键衍生指标

- `sector_rotation_speed`
- `top3_sector_turnover_share`
- `sector_dispersion_score`
- `hot_theme_concentration`
- `leader_persistence`

### 实现建议

- 板块分类建议先固定一套，如申万一级行业
- 主题板块可以作为增强模块，先不要放进第一版核心链路

---

## 4.4 微观交易状态

微观层用于补充“市场交易风格”，尤其对短线、事件、情绪型策略有价值。

### 建议保留的核心字段

- 涨停数 / 跌停数
- 连板高度
- 炸板率
- 换手率中位数
- 缩量 / 放量占比
- 高开低走占比
- 量价背离统计
- 一字板占比

### 对策略适用性的意义

- 情绪高涨时，短线打板/龙头策略可能更适合
- 炸板率高、跌停增多时，高风险策略应降权
- 换手中位数可判断市场活跃度

### 典型衍生指标

- `limit_up_count`
- `limit_down_count`
- `board_break_rate`
- `avg_turnover_median`
- `price_volume_divergence_score`

---

## 5. 数据模型设计

建议定义统一特征表。

### 5.1 特征主表

字段建议：

- `timestamp`
- `frequency`
- `feature_group`
- `feature_name`
- `raw_value`
- `normalized_value`
- `value_rank`
- `source`
- `is_imputed`
- `staleness_days`

### 5.2 快照表

每天或每个 bar 存一份聚合快照：

- `snapshot_id`
- `timestamp`
- `json_blob`
- `quality_score`

### 5.3 数据质量表

- `feature_name`
- `timestamp`
- `status`
- `error_message`
- `fallback_source`

---

## 6. 标准化与降维

信息层不能直接把几十个原始字段扔给场景识别层。建议先做一次轻量标准化。

### 推荐流程

1. 计算原始值
2. 缺失值处理
3. 截尾处理
4. rolling z-score 或分位数映射
5. 生成组内复合分

### 每组可输出两个层级

- 原子特征：保留明细
- 组级得分：用于场景识别

例如：

```python
macro_liquidity_score
market_trend_score
sector_rotation_score
micro_sentiment_score
```

---

## 7. 缺失值与异常处理

### 7.1 缺失策略

- 宏观低频数据：前值填充
- 市场日线数据：不得缺失，缺失则整天标记无效
- 分钟数据：可以降级为日线场景

### 7.2 异常值策略

- 用历史分位裁剪
- 对外保留 `is_outlier` 标记

### 7.3 数据质量评分

每天给快照一个 `quality_score`：

- 1.0 完整
- 0.8 少量回退
- 0.5 多项依赖缓存
- 0.0 不可用于调度

---

## 8. 第一阶段 MVP 建议

第一阶段只做日线，减少复杂度。

### MVP 指标集

宏观/流动性：
- 北向 5D 净流入
- 两市成交额相对 20D
- 融资余额 20D 变化

市场状态：
- 沪深 300 / 中证 500 / 创业板 20D 收益
- 市场宽度
- 20D 波动率

板块状态：
- 申万一级行业相对强弱
- 行业轮动速度

微观交易状态：
- 涨停数
- 跌停数
- 炸板率

### MVP 输出

- 10 到 20 个核心特征
- 4 个组级得分
- 1 份可缓存的日频快照

---

## 9. 不建议的做法

- 不要一开始就接太多宏观指标
- 不要把所有新闻、舆情、研报都塞进来
- 不要混合日频和分钟频而不做对齐
- 不要直接用原始值做决策
- 不要依赖大量在线接口而没有缓存方案

---

## 10. 与后续层的接口

信息层输出给场景识别层的内容建议为：

```python
{
    "timestamp": "...",
    "features": {...},
    "group_scores": {
        "macro_liquidity_score": 0.62,
        "market_state_score": 0.71,
        "sector_state_score": 0.48,
        "micro_structure_score": 0.55,
    },
    "quality_score": 0.92
}
```

场景识别层不直接访问原始数据源，只消费标准化后的快照。

---

## 11. 总结

信息层不是万能信息仓库，而是多策略系统的“市场环境传感器”。

成功标准不是指标越多越好，而是：

- 能稳定产出
- 能解释策略适用性
- 能回测复现
- 能被场景识别层稳定消费
