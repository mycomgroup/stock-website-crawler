# 任务 10：批量运行器与并发存储修复

## 任务目标

修复批量运行统计不准、异常误算成功、并发存储不稳的问题，并补一个可执行策略扫描层。

## 负责范围

- `run_strategies_parallel.py`
- `jqdata_akshare_backtrader_utility/db/duckdb_manager.py`
- 如有必要，可新增扫描器模块
- 对应测试或 smoke 脚本

## 建议写入目录

- 代码写入：
  - 仓库根目录
  - `jqdata_akshare_backtrader_utility/`
  - `tests/`
- 任务结果写入：
  - `docs/0330_result/`
  - 建议文件名：`task10_batch_runner_result.md`

## 给子 Agent 的提示词

你负责批量运行器和并发存储修复，请直接开发，不要只做统计分析。

重点要求：

- 修复“异常也算成功”的统计口径
- 区分：
  - 真成功
  - 运行异常
  - 缺失 API
  - 非策略文件
- 处理 DuckDB 并发锁或写入冲突问题
- 如果需要，可增加一个策略扫描器，在跑批前先做分类
- 输出结果要更可信，便于后续衡量兼容率
- 不要顺手改行情 API、finance、timer

## 任务验证

至少完成以下验证：

- 并行跑批时不会轻易因为锁冲突整体失败
- `summary.json` 中成功/失败统计口径真实可信
- 异常策略不会再被标记为成功

建议验证方式：

- 准备 2 到 5 个最小策略样本
- 至少包含一个成功样本、一个故意异常样本、一个缺失 API 样本
