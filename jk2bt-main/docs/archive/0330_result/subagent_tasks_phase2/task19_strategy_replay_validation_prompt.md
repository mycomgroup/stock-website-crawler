# 任务 19：策略回放正确性验证

## 任务目标

不是只让策略“能跑”，而是抽样验证结果是否像样，避免出现大量因为语义偏差造成的假运行。

## 负责范围

- 真实 txt 策略样本
- `jqdata_akshare_backtrader_utility/jq_strategy_runner.py`
- 需要时可新增验证辅助脚本

## 建议写入目录

- 代码写入：
  - 仓库根目录
  - `jqdata_akshare_backtrader_utility/`
- 结果写入：
  - `docs/0330_result/`
  - 建议文件名：`task19_strategy_replay_validation_result.md`

## 给子 Agent 的提示词

你负责做“结果正确性抽检”，目标是避免把假跑通当成真跑通。

要求：

- 从白名单或高价值策略中抽一批样本
- 检查：
  - 是否真的触发了定时器
  - 是否真的发生了交易
  - 订单语义是否合理
  - 净值曲线是否存在
  - `record` 输出是否符合预期
- 如果发现“脚本执行了但策略语义明显不对”，要单独列出来
- 产出一份“真跑通策略样本池”

## 任务验证

- 至少抽检若干真实策略
- 给出每个样本的验证依据
