# 任务 11：策略资产盘点与分层

## 任务目标

把 `txt` 策略做一次结构化盘点，输出“按类型分层的可执行清单”，为后续批量推进提供基线。

## 负责范围

- `jkcode/` 或仓库中实际存放 `txt` 策略的目录
- `jqdata_akshare_backtrader_utility/strategy_scanner.py`
- 如有需要，可新增策略盘点脚本

## 建议写入目录

- 代码写入：
  - 仓库根目录
  - `jqdata_akshare_backtrader_utility/`
- 结果写入：
  - `docs/0330_result/`
  - 建议文件名：`task11_strategy_inventory_result.md`

## 给子 Agent 的提示词

你负责把现有 txt 策略做系统盘点，不要只统计总数，要产出可用于执行的分层结果。

要求：

- 识别并分类：
  - 日线股票/ETF 策略
  - 分钟级/日内策略
  - 期货/股指策略
  - 多账户/子账户策略
  - 依赖文件资源策略
  - 依赖 talib/sklearn/xgboost/lightgbm/torch 的策略
  - 研究说明/非策略文档
- 输出结构化结果：
  - 总文件数
  - 真策略数
  - 每类数量
  - 每类代表文件样本
- 最终产出一个“推进优先级清单”，明确哪些类最值得先打通。

## 任务验证

- 至少生成一个可读 Markdown 报告
- 至少生成一个机器可读 JSON 或 CSV 清单
- 报告中的数量和样本需可复查
