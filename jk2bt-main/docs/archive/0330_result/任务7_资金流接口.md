## 实现内容

### 核心 API

```python
get_money_flow(
    security_list,    # 股票代码或列表，支持 'sh600000'、'600000.XSHG'、'600000' 等格式
    start_date,       # 开始日期（可选）
    end_date,         # 结束日期（可选）
    fields,           # 返回字段列表
    count,            # 返回最近 N 天数据
)
```

### 支持的调用方式

**位置参数（兼容原有策略）：**
```python
df = get_money_flow(stocks, None, date, ["sec_code", "change_pct"], 1)
```

**关键字参数：**
```python
df = get_money_flow(security_list=['sh600000', 'sz000001'], 
                   end_date='2025-03-28', 
                   fields=['sec_code', 'net_pct_main'], 
                   count=30)
```

### 支持的字段

| 字段 | 说明 |
|------|------|
| `sec_code` | 标准化股票代码 |
| `date` | 日期 |
| `close` | 收盘价 |
| `change_pct` | 涨跌幅 |
| `net_amount_main` | 主力净流入-净额 |
| `net_pct_main` | 主力净流入-净占比 |
| `net_amount_xl` | 超大单净流入-净额 |
| `net_pct_xl` | 超大单净流入-净占比 |
| `net_amount_l` | 大单净流入-净额 |
| `net_pct_l` | 大单净流入-净占比 |
| `net_amount_m` | 中单净流入-净额 |
| `net_pct_m` | 中单净流入-净占比 |
| `net_amount_s` | 小单净流入-净额 |
| `net_pct_s` | 小单净流入-净占比 |

### 数据源

使用 akshare 的 `stock_individual_fund_flow` 接口（东方财富网数据）。

## 已知边界

| 限制 | 说明 |
|------|------|
| 历史数据范围 | akshare 返回约 120 天历史数据 |
| 数据延迟 | 取决于 akshare 数据源更新 |
| 无本地缓存 | 每次调用实时请求（后续可参考 stock.py 的 DuckDB 缓存方案） |
| 无 end_date 时 | 返回最新数据（降序排列，最近在前） |

## 示例用法

```python
from market_data import get_money_flow

# 获取单只股票最近 5 天数据
df = get_money_flow('sh600519', count=5, fields=['sec_code', 'date', 'change_pct', 'net_pct_main'])

# 筛选主力净流入为正的股票
stocks = ['sh600519', 'sz000001', 'sz000002']
df = get_money_flow(stocks, count=1, fields=['sec_code', 'net_pct_main'])
positive = df[df['net_pct_main'] > 0]

# 按 end_date 获取历史数据
df = get_money_flow('sh600519', end_date='2025-03-28', count=30, 
                   fields=['sec_code', 'date', 'net_amount_xl', 'net_amount_l'])

# 使用日期区间
df = get_money_flow('sh600519', start_date='2025-01-01', end_date='2025-03-28',
                   fields=['sec_code', 'date', 'change_pct'])
```
