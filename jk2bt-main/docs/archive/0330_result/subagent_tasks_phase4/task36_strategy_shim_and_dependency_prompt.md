# Task 36: 策略垫片与依赖兼容层

## 任务目标

为常见的第三方依赖和聚宽环境差异补一层“策略兼容垫片”，降低因为 `kuanke`、旧 pandas、旧写法导致的非核心失败。

## 建议写入目录

- `jqdata_akshare_backtrader_utility/compat/`
- `tests/`
- `docs/0330_result/`

## 负责范围

- 兼容垫片
- 运行环境适配
- 依赖缺失时的可解释降级

## 给子 agent 的提示词

你负责策略兼容垫片层。

目标不是完整复刻聚宽私有环境，而是降低本地运行时常见的非核心失败。

重点关注：
- `kuanke` 兼容导入
- `pandas.stats` 旧接口迁移
- 旧版 Python2 风格 `print` / 编码问题的预处理入口
- 依赖缺失时的清晰错误信息

要求：
- 优先做“兼容垫片”，不要大范围魔改策略源码
- 能 shim 的尽量 shim，不能 shim 的明确标记为需人工改造

## 任务验证

- 抽检一批此前因环境/依赖问题失败的策略
- 给出“自动恢复运行”的样本数量
- 输出 `docs/0330_result/task36_strategy_shim_and_dependency_result.md`

## 任务成功总结

- 减少因运行环境差异导致的无效失败
- 提升旧策略直接运行概率
