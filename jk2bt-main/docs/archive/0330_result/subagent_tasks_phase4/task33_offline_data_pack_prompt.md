# Task 33: 离线数据包与预热缓存

## 任务目标

建立一套“可离线复现”的最小数据包与预热机制，降低网络不可用时对策略运行的影响。

## 建议写入目录

- `jqdata_akshare_backtrader_utility/market_data/`
- `scripts/`
- `docs/0330_result/`

## 负责范围

- 日线与分钟数据的预热
- 关键指数/ETF/股票样本缓存
- 可重复使用的数据准备脚本

## 给子 agent 的提示词

你负责离线数据包和预热缓存机制。

当前测试虽然通过，但日志里仍能看到真实网络请求失败告警。这说明很多接口是靠 fallback 扛住的，不足以支撑大规模 txt 跑批。

请实现：
- 一个数据预热脚本
- 一个最小离线数据包清单
- 明确哪些 symbol / 指数 / ETF / 常用股票需要预热
- 尽量覆盖日线基线样本池和分钟验证样本池

要求：
- 支持固定日期区间预热
- 支持只刷新缺失缓存
- 输出预热报告：成功、失败、跳过

## 任务验证

- 在断网或模拟断网场景下，验证一组核心 API 仍可返回数据
- 输出 `docs/0330_result/task33_offline_data_pack_result.md`

## 任务成功总结

- 缓存从“顺手存在”变成“可管理的离线能力”
- 降低网络波动对回测与跑批的影响
