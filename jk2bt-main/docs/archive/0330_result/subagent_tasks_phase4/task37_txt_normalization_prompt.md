# Task 37: txt 策略文本标准化

## 任务目标

建立 `.txt` 策略的预处理与标准化流程，自动处理编码、缩进、明显语法兼容问题，降低“文件层面失败”。

## 建议写入目录

- `jqdata_akshare_backtrader_utility/`
- `scripts/`
- `docs/0330_result/`

## 负责范围

- 编码识别
- 文本清洗
- Python 语法兼容预处理

## 给子 agent 的提示词

你负责 txt 策略的文本标准化。

目标是让更多策略至少能进入“可加载/可分析”阶段，而不是一开始就死在编码和语法层。

请实现：
- 多编码读取增强
- TAB/空格缩进修复或检测
- Python2 风格 `print`、异常语法等明显兼容问题识别
- 标准化前后差异记录

要求：
- 默认不改源文件
- 优先输出标准化后的临时文本或缓存版本
- 对危险修复要保留告警

## 任务验证

- 从失败样本中抽取 20 个“文件层失败”的 txt 策略重试加载
- 输出成功恢复数与失败原因统计
- 结果写入 `docs/0330_result/task37_txt_normalization_result.md`

## 任务成功总结

- 文件层失败显著减少
- 更多策略进入后续真实验证流程
