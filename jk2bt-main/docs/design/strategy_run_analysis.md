### 3. 主要失败原因

#### A. DuckDB并发锁冲突 (最主要原因)

当多个进程并行运行时，DuckDB数据库文件锁冲突导致数据下载失败：

```
ERROR: IO Error: Could not set lock on file "/data/market.db": 
Conflicting lock is held in another process
```

**影响**: 并行运行时，多个策略进程同时尝试访问数据库，导致锁冲突。

**解决方案**:
1. 减少并行数: `--max-workers 1` 或 `--max-workers 2`
2. 使用支持并发访问的数据库模式
3. 每个策略使用独立的数据库连接

#### B. 未实现的聚宽API

部分策略使用了框架未支持的API，导致运行失败：

| API | 使用场景 | 模拟状态 |
|-----|----------|----------|
| `get_billboard_list` | 龙虎榜数据 | ❌ 未实现 |
| `get_margfin_stocks` | 融资融券 | ❌ 未实现 |
| `get_price` 分钟级数据 | 分钟策略 | ⚠️ 部分支持 |
| `talib` | 技术指标 | ❌ 未安装 |

**示例策略**:
- "01 龙回头3.0回测速度优化版.txt" - 使用龙虎榜API
- 使用分钟级数据的策略

#### C. 编码问题
