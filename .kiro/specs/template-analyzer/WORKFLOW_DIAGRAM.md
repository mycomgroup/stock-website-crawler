## 关键设计点

### 1. 一对一映射
```
pathTemplate → TemplateConfig
/open/api/doc → api-doc配置
/analytics/*/dashboard → dashboard配置
```

### 2. 直接输出配置
Template Content Analyzer不输出中间分析结果，而是直接输出完整的TemplateConfig对象，包含：
- extractors（提取器配置）
- filters（过滤器配置）
- metadata（元数据）

### 3. 实时验证
支持提供样例URL，实时抓取并验证配置效果：
```javascript
sampleUrls: [
  "https://www.lixinger.com/open/api/doc?api-key=cn/company",
  "https://www.lixinger.com/open/api/doc?api-key=cn/index"
]
```

### 4. 配置驱动
运行时通过ConfigLoader加载配置，创建TemplateParser实例，实现配置驱动的解析。

## 示例：API文档模板分析

### 输入
- pathTemplate: `/open/api/doc`
- 163个markdown文件
- 2个样例URL

### 处理
1. 分析163个页面，发现：
   - "API文档"标题出现163次（100%）→ 噪音
   - "导航"标题出现163次（100%）→ 噪音
   - 参数表格出现150次（92%）→ 数据
   - 响应表格出现145次（89%）→ 数据

2. 生成配置：
   - extractors: title, parameters, responseData, codeBlocks
   - filters: 移除"API文档"、"导航"

3. 实时验证：
   - 抓取2个样例URL
   - 应用配置提取数据
   - 验证效果 ✓

### 输出
完整的api-doc配置对象，保存到template-rules.jsonl

## 优势

1. **清晰的职责**：每个组件职责明确
2. **直接输出**：减少中间步骤，直接生成可用配置
3. **实时验证**：确保配置质量
4. **易于调试**：配置文件易读易改
5. **AI友好**：大模型可以轻松理解和修改配置
