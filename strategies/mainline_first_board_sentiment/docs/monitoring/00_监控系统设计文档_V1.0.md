## 1. 系统概述

### 1.1 系统目标

二板接力监控与衰减报警系统的核心目标是：

**主要目标**：
- ✅ **实时监控策略表现**：实时跟踪策略收益、风险、交易执行等关键指标
- ✅ **及时发现异常情况**：通过预警机制第一时间发现策略衰退、风控触发等问题
- ✅ **自动化报警通知**：通过多渠道（邮件/短信/电话）及时通知相关人员
- ✅ **支持决策优化**：提供历史数据分析、可视化展示，辅助策略优化决策

**次要目标**：
- 支持多策略监控（可扩展）
- 支持自定义监控指标
- 支持报警规则配置
- 支持历史数据导出与分析

## 2. 监控指标体系

### 2.1 收益类指标

#### 2.1.1 实时收益指标

| 指标名称 | 计算方法 | 更新频率 | 预警阈值 | 优先级 |
|---------|---------|---------|---------|--------|
| 当日收益 | (收盘市值 - 开盘市值) / 开盘市值 × 100% | 实时(1分钟) | 无 | P1 |
| 累计收益 | (当前净值 - 初始净值) / 初始净值 × 100% | 实时(1分钟) | 无 | P1 |
| 年化收益 | (累计收益 + 1)^(252/天数) - 1 | 日频 | < 15%：黄色 | P1 |
| 滚动7日收益 | ∑(过去7日日收益) | 日频 | < -5%：黄色 | P2 |
| 滚动30日收益 | ∑(过去30日日收益) | 日频 | < -10%：红色 | P2 |

| 情绪关闭期收益 | ∑(情绪关闭期交易收益) | 日频 | < 0%：观察 | P2 |
| 早盘收益 | ∑(09:30-10:00交易收益) | 日频 | 无 | P3 |
| 午盘收益 | ∑(10:00-13:30交易收益) | 日频 | 无 | P3 |
| 尾盘收益 | ∑(13:30-15:00交易收益) | 日频 | 无 | P3 |

#### 2.1.3 基准对比指标

| 指标名称 | 计算方法 | 更新频率 | 预警阈值 | 优先级 |
|---------|---------|---------|---------|--------|
| 超额收益(沪深300) | 策略收益 - 沪深300收益 | 日频 | < -5%：黄色 | P1 |

### 2.2 风险类指标

#### 2.2.1 回撤指标

| 指标名称 | 计算方法 | 更新频率 | 预警阈值 | 优先级 |
|---------|---------|---------|---------|--------|
| 当前回撤 | (峰值净值 - 当前净值) / 峰值净值 | 实时(1分钟) | > 10%：黄色<br/>> 15%：红色 | P1 |
| 最大回撤(滚动) | max(过去N日回撤) | 日频 | > 15%：黄色<br/>> 20%：红色 | P1 |
| 回撤持续时间 | 自峰值到当前的交易日数 | 实时 | > 20日：黄色<br/>> 30日：红色 | P2 |
| 平均回撤深度 | mean(过去N日回撤) | 日频 | > 8%：黄色 | P3 |

#### 2.2.2 波动指标

| 指标名称 | 计算方法 | 更新频率 | 预警阈值 | 优先级 |
|---------|---------|---------|---------|--------|
| 日波动率(滚动20日) | std(过去20日日收益) × √252 | 日频 | > 历史均值+2σ：黄色 | P2 |
| 下行波动率 | std(负收益) × √252 | 日频 | > 历史均值+2σ：黄色 | P2 |
| 收益分布偏度 | skewness(过去60日日收益) | 日频 | < -0.5：观察 | P3 |
| 收益分布峰度 | kurtosis(过去60日日收益) | 日频 | > 3：观察 | P3 |

#### 2.2.3 风险调整收益指标

| 指标名称 | 计算方法 | 更新频率 | 预警阈值 | 优先级 |
|---------|---------|---------|---------|--------|
| 夏普比率(滚动) | (年化收益 - 无风险利率) / 年化波动率 | 日频 | < 1.0：黄色<br/>< 0.5：红色 | P1 |
| 卡玛比率(滚动) | 年化收益 / 最大回撤 | 日频 | < 1.0：黄色 | P1 |
| 索提诺比率(滚动) | (年化收益 - 无风险利率) / 下行波动率 | 日频 | < 1.5：黄色 | P2 |
| 信息比率 | 超额收益 / 跟踪误差 | 日频 | < 0.5：观察 | P3 |

#### 2.2.4 风险暴露指标

| 指标名称 | 计算方法 | 更新频率 | 预警阈值 | 优先级 |
|---------|---------|---------|---------|--------|
| 市值暴露 | 平均持仓市值 / 总市值 | 日频 | 超出范围±20%：观察 | P2 |
| 行业集中度 | max(各行业持仓占比) | 日频 | > 40%：黄色 | P2 |
| 风格暴露 | 中性/成长/价值因子暴露 | 日频 | 绝对值>1.5：观察 | P3 |
| Beta暴露 | 对市场基准的Beta | 日频 | > 1.2：观察 | P3 |

| 订单拒绝率 | 拒绝订单数 / 提交订单数 | 实时 | > 5%：红色 | P1 |

### 2.5 核心指标识别

|---------|---------|---------|---------|
| 数据延迟>10分钟 | 数据更新时间 - 当前时间 > 10分钟 | 黄色 | 24小时 |
| 数据延迟>30分钟 | 数据更新时间 - 当前时间 > 30分钟 | 红色 | 1小时 |
| 数据完整率<95% | 数据完整性 < 95% | 黄色 | 24小时 |
| 系统运行异常 | 心跳检测失败或异常日志 | 红色 | 1小时 |
| API连接失败 | API接口连续3次连接失败 | 红色 | 1小时 |

### 4.2 报警级别

| 级别 | 颜色 | 含义 | 处理时限 | 通知方式 |
|------|------|------|---------|---------|
| 绿色 | 🟢 | 正常 | - | 仅仪表盘显示 |
| 黄色 | 🟡 | 观察 | 24小时 | 仪表盘+邮件 |
| 橙色 | 🟠 | 警告 | 4小时 | 仪表盘+邮件+短信 |
| 红色 | 🔴 | 紧急 | 1小时 | 仪表盘+邮件+短信+电话 |

### 4.3 报警方式

#### 4.3.1 仪表盘显示

- 顶部状态栏：预警状态图标
- 总览页面：预警信息列表
- 弹窗提示：新报警弹窗通知

#### 4.3.2 邮件通知

**邮件模板**：包含报警详情、处理建议、处理时限

#### 4.3.3 短信通知

**短信模板**：简明扼要的报警信息摘要

#### 4.3.4 电话通知

**触发条件**：仅红色报警触发
**通知方式**：连续3次电话通知（间隔5分钟）

### 4.4 报警处理流程

#### 4.4.1 黄色报警处理流程

```
黄色报警触发 → 记录报警信息 → 发送邮件通知 → 相关人员关注(24小时) → 分析报警原因 → 采取相应措施 → 记录处理结果 → 更新报警状态为"已处理"
```

#### 4.4.2 橙色报警处理流程

```
橙色报警触发 → 记录报警信息 → 发送邮件+短信通知 → 相关人员立即响应(4小时) → 分析报警原因 → 触发风控降级机制 → 采取相应措施 → 记录处理结果 → 更新报警状态
```

#### 4.4.3 红色报警处理流程

```
红色报警触发 → 立即记录报警信息 → 立即发送邮件+短信+电话通知 → 相关人员立即响应(1小时) → 立即触发熔断机制 → 暂停所有交易 → 全面分析报警原因 → 采取紧急措施 → 记录处理结果 → 更新报警状态 → 复盘总结，更新策略
```

### 4.5 熔断机制

| 报警类型 | 熔断措施 |
|---------|---------|
| 单日亏损>5% | 立即停止当日交易 |
| 回撤>15% | 降低仓位至10%，暂停新开仓 |
| 回撤>20% | 完全停止交易，清仓观望 |
| 连续5日亏损 | 停止交易，全面复盘 |
| 胜率<60% | 触发OOS验证，暂停交易 |
| 盈亏比<0.8 | 触发OOS验证，暂停交易 |
| 滚动30日收益<-10% | 停止交易，触发熔断 |

#### 9.2.1 数据采集故障

**故障现象**：数据延迟或缺失
**处理步骤**：
1. 检查数据源连接状态
2. 检查数据采集服务状态
3. 重启数据采集服务
4. 切换备用数据源

#### 9.2.2 指标计算故障

**故障现象**：指标计算错误或延迟
**处理步骤**：
1. 检查指标计算服务状态
2. 检查数据完整性
3. 重启指标计算服务
4. 重新计算指标

#### 9.2.3 报警通知故障

**故障现象**：报警通知失败
**处理步骤**：
1. 检查邮件/短信服务状态
2. 检查报警通知配置
3. 重启报警通知服务
4. 手动发送报警通知
