## 一、系统简介

二板接力监控与衰减报警系统是为首板低开情绪择时策略设计的自动化监控系统，旨在实时监控策略表现、及时发现异常、自动化报警通知，并提供决策支持。

### 核心功能

- ✅ **实时监控**：实时跟踪策略收益、风险、交易执行等关键指标
- ✅ **衰减检测**：通过OOS验证、胜率下降等机制及时发现策略衰退
- ✅ **风控触发**：自动触发风控规则，包括停手、降仓、清仓等
- ✅ **报警通知**：多渠道报警（邮件/短信/电话），分级响应
- ✅ **可视化展示**：监控仪表盘，实时展示策略状态
- ✅ **数据分析**：历史数据查询、统计分析、报表生成

## 四、系统亮点

### 4.1 监控指标体系

**指标分类**：
- 收益类指标：12个
- 风险类指标：15个
- 交易类指标：10个
- 系统类指标：8个
- **合计：45个监控指标**

**优先级分级**：
- P1级核心指标：18个（必须监控）
- P2级重要指标：13个（建议监控）
- P3级辅助指标：14个（可选监控）

### 4.2 报警机制

**报警规则**：28个报警规则

**报警级别**：
- 🟢 绿色：正常运行
- 🟡 黄色：需要关注（24小时内处理）
- 🟠 橙色：需要处理（4小时内处理）
- 🔴 红色：立即处理（1小时内处理）

**熔断机制**：
- 回撤>15%：降低仓位至10%
- 回撤>20%：完全停止交易，清仓
- 胜率<60%：触发OOS验证，暂停交易
- 单日亏损>5%：停止当日交易

### 4.3 技术架构

**现代化技术栈**：
- 后端：Python + FastAPI + Celery
- 前端：React + Ant Design + ECharts
- 数据库：PostgreSQL + InfluxDB + Redis
- 基础设施：Docker + Kubernetes + Prometheus + Grafana

**高性能设计**：
- 实时指标延迟 < 1秒
- 历史查询响应 < 3秒
- 系统可用性 > 99.9%

## 五、核心指标速查

### 5.1 收益类核心指标

| 指标 | 计算方法 | 预警阈值 |
|------|---------|---------|
| 当日收益 | (收盘市值 - 开盘市值) / 开盘市值 | 无 |
| 累计收益 | (当前净值 - 初始净值) / 初始净值 | 无 |
| 年化收益 | (累计收益 + 1)^(252/天数) - 1 | < 15%：黄色 |
| 二板接力收益 | ∑(所有二板接力交易收益) | 无 |

### 5.2 风险类核心指标

| 指标 | 计算方法 | 预警阈值 |
|------|---------|---------|
| 当前回撤 | (峰值净值 - 当前净值) / 峰值净值 | > 10%：黄色，> 15%：红色 |
| 最大回撤 | max(过去N日回撤) | > 15%：黄色，> 20%：红色 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 | < 1.0：黄色，< 0.5：红色 |
| 卡玛比率 | 年化收益 / 最大回撤 | < 1.0：黄色 |

### 5.3 交易类核心指标

| 指标 | 计算方法 | 预警阈值 |
|------|---------|---------|
| 胜率(20日) | 获利交易次数 / 总交易次数 | < 70%：黄色，< 60%：红色 |
| 盈亏比(20日) | 平均盈利 / 平均亏损 | < 1.0：黄色，< 0.8：红色 |
| 滑点统计 | 实际成交价 - 理论价格 | > 0.3%：黄色，> 0.5%：红色 |
| 成交率 | 成交订单数 / 提交订单数 | < 90%：黄色 |

### 5.4 系统类核心指标

| 指标 | 计算方法 | 预警阈值 |
|------|---------|---------|
| 数据更新状态 | 最后更新时间 - 当前时间 | > 10分钟：黄色，> 30分钟：红色 |
| 系统运行状态 | 心跳检测 | 异常：红色 |
| 数据完整性 | 有效数据点数 / 预期数据点数 | < 95%：黄色，< 90%：红色 |
